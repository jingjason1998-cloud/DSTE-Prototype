# DSTE 可访问性（a11y）规范

DSTE 是企业级战略管理平台，必须对所有用户（包括使用键盘、屏幕阅读器、高对比度模式的用户）可用。

## 颜色对比度

- 正文文字与背景对比度 ≥ 4.5:1（WCAG AA）
- 大号文字（≥18px 或 14px bold）与背景对比度 ≥ 3:1
- 焦点指示、图标、按钮边界对比度 ≥ 3:1
- 使用工具验证：Lighthouse、Stark、axe DevTools

## 颜色使用

- 状态不能只用颜色表达，必须配合图标 + 文字
- 避免红绿色盲难以区分的组合（如仅用红/绿表示状态）
- 图表使用图案或标签辅助颜色区分

## 焦点管理

- 所有可交互元素必须有可见焦点环
- 焦点环样式：`box-shadow: 0 0 0 2px var(--color-bg-surface), 0 0 0 4px var(--color-border-focus);`
- 使用 `:focus-visible` 而非 `:focus`，避免鼠标点击时显示焦点环
- Tab 顺序符合视觉顺序

## 键盘导航

- 所有功能可通过键盘完成
- 按钮、链接可通过 Enter/Space 激活
- 弹窗/抽屉打开后焦点进入，关闭后焦点回到触发元素
- ESC 关闭弹窗、抽屉、下拉菜单
- 使用 `Tab`、`Shift+Tab`、`Arrow` 键在组件内导航

## ARIA

### 图标

- 装饰性图标：`aria-hidden="true" role="presentation"`
- 功能性图标：`aria-label` 或配合可见文字

```html
<!-- 装饰性 -->
<svg aria-hidden="true" role="presentation">...</svg>

<!-- 功能性 -->
<button aria-label="关闭">
  <svg aria-hidden="true">...</svg>
</button>
```

### 导航

- Topbar 导航使用 `<nav aria-label="主导航">`
- Sidebar 使用 `<nav aria-label="侧边导航">`
- 当前页面使用 `aria-current="page"`

### 弹窗

- 使用 `role="dialog"` 或语义化 `<dialog>`
- 使用 `aria-modal="true"`
- 标题使用 `aria-labelledby` 关联

## 减少动画

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

所有页面必须尊重用户的减少动画偏好。

## 暗色模式

- 使用语义 token 实现 light/dark 切换
- 不依赖 `prefers-color-scheme`  alone，提供手动切换
- 暗色模式下避免纯黑（`#000000`）背景，使用 `--color-gray-950`
- 暗色模式下文字不使用纯白色，使用 `--color-gray-0`（#FFFFFF）配合透明度

## 表单

- 每个输入框关联 `<label>`，使用 `for` 关联 `id`
- 必填项使用 `aria-required="true"` 或 `required`
- 错误提示使用 `aria-describedby` 关联

## 表格

- 使用 `<table>`、`<thead>`、`<tbody>`、`<th scope="col">`
- 复杂表格使用 `aria-label` 或 `aria-labelledby` 描述

## 验证清单

- [ ] 所有页面通过 Lighthouse a11y 评分 ≥ 90
- [ ] 所有图标都有正确的 aria 属性
- [ ] 所有交互元素可通过 Tab 聚焦
- [ ] 弹窗/抽屉焦点管理正确
- [ ] 颜色+图标+文字共同表达状态
- [ ] 支持 `prefers-reduced-motion`
- [ ] 支持 `prefers-color-scheme` 并提供手动切换
