> 版本：v1.0
> 更新日期：2026-06-04
> 状态：设计稿（未进入开发）

---

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 产品名称 | 经营分析会运营中枢 |
| 产品定位 | 将经营分析会从“会议信息管理工具”升级为“经营分析会运营中枢”，串联报表、报告、会议、决议、执行全链路 |
| 适用范围 | DSTE 战略管理平台 — 经营分析会模块 |

| 关键外部系统 | FR（FineReport）、BI（FineBI）、KMS（Confluence）、企业微信 |

## 2. 背景与目标

### 2.1 背景

当前 DSTE 经营分析会模块已具备会议管理、议程管理、决议中心、任务督办、材料评审、会议评估、通知体系等能力。但随着业务深入，出现以下问题：

1. **会议与数据脱节**：会议讨论依赖 FR/BI 看板和 KMS 报告，但系统内无法直接引用和追踪这些外部资产。
2. **会前准备不可控**：数据是否就绪、报告是否发布、材料是否通过评审，缺乏统一的检查机制。
3. **会中缺少结构化呈现**：会议召开时，议题、报表、报告、批注、决议分散在不同系统。
4. **会后闭环不足**：决议和行动项与会议、报告、报表之间的关联不清晰，难以复盘。
5. **AI 能力未嵌入流程**：现有材料评审能力孤立，未与会前、会中、会后流程深度结合。

### 2.2 目标

构建 **经营分析会运营中枢**，实现：

- 经营分析报表、经营分析报告、经营分析会三大对象的信息架构清晰；
- 会前、会中、会后全流程在系统内闭环；
- FR/BI 看板、KMS 报告与会议流程深度融合；
- AI 能力嵌入全流程，提升报告生成、纪要生成、决议识别、复盘分析效率；
- 决议与行动项可追踪、可复盘、可沉淀。

### 2.3 设计原则

| 原则 | 说明 |
|---|---|
| **不替代外部系统** | FR/BI 继续做数据分析和看板，KMS 继续做报告沉淀，运营中枢只做编排和关联 |
| **会议为核心对象** | 所有数据和流程围绕经营分析会展开 |
| **报表-报告-会议三层递进** | 报表提供数据，报告提供分析，会议产生决策 |
| **流程闭环** | 会前准备 → 会中驾驶舱 → 会后纪要/决议/督办 → 复盘分析 |
| **AI 是流程增强，不是替代** | AI 生成草稿、识别要点、辅助诊断，最终由人确认 |

## 3. 术语定义

| 术语 | 定义 |
|---|---|
| **经营分析报表** | 由 FR/BI 制作的数据看板/报表，面向数据呈现，回答“数据是什么” |
| **经营分析报告** | 由 KMS 承载的经营分析文档，面向经营解读，回答“为什么、怎么办” |
| **经营分析会** | 由运营中枢管理的会议对象，面向讨论和决策，回答“决定了什么、谁去做” |
| **议题（Topic）** | 经营分析会的最小讨论单元，关联指标、报表、报告、决议 |
| **决议（Decision）** | 会议讨论后形成的决策事项 |
| **行动项（Action）** | 决议落地的具体执行任务 |
| **报表平台** | 经营分析报表的统一目录、元数据管理和关联消费入口 |
| **一报一会 Pipeline** | 会前报告 → 会前评审 → 会议召开 → 纪要起草 → 纪要定稿 → 行动追踪 |

## 4. 产品架构

### 4.1 整体架构

```
┌──────────────────────────────────────────────────────────────────────┐
│                    经营分析会运营中枢                                 │
│    会议管理 │ 决议中心 │ 督办中心 │ 经营分析报表平台 │ AI 功能       │
└──────────────────────────────────────────────────────────────────────┘
                              │
           ┌──────────────────┼──────────────────┐
           ▼                  ▼                  ▼
      ┌─────────┐      ┌─────────┐        ┌─────────┐
      │  FR/BI  │      │   KMS   │        │  企业微信  │
      │ 看板平台 │      │ 报告平台 │        │  通知平台  │
      └─────────┘      └─────────┘        └─────────┘
```

### 4.2 应用边界

| 应用 | 负责 | 不负责 |
|---|---|---|
| **会议管理** | 会议全生命周期、议题、会中驾驶舱、会议纪要 | 不替代 FR/BI、KMS |
| **决议中心** | 决议产生、审批、督办、归档 | 不替代任务系统 |
| **督办中心** | 行动项执行、进展填报、逾期预警 | 不替代 OA/项目管理系统 |
| **经营分析报表平台** | 报表目录、元数据、关联关系、使用统计 | 不替代 FR/BI 制作能力 |
| **AI 功能** | 材料诊断、报告生成、纪要生成、决议识别、行动项识别、复盘分析 | 不替代人做最终决策 |

## 6. 应用设计

### 6.1 全局导航

```
┌──────────────────────────────────────────────────────────────────────┐
│  Logo  │ 会议管理 │ 决议中心 │ 督办中心 │ 报表平台 │ AI 功能 │ [通知] [个人] │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                    当前应用的工作区（二级导航 + 内容区）                │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

### 6.2 会议管理应用

定位：经营分析会的核心入口，覆盖会议全生命周期。

```
会议管理
├── 我的会议
│   ├── 会议列表
│   └── 日历视图
├── 会议筹备
│   ├── 创建会议
│   ├── 会前准备
│   └── 会议纪要
├── 会议召开
│   └── 会中驾驶舱
├── 会议复盘
│   └── 会议评估
├── 会议沟通
│   └── 通知发送
└── 配置
    └── 模板管理
```

| 页面 | 作用 | 关键功能 |
|---|---|---|
| 会议列表 | 查看所有会议 | 卡片/表格/日历视图、筛选、快捷操作 |
| 创建会议 | 新建会议 | 选择模板、填写基础信息、设置场景/层级 |
| 会议编辑 | 维护会议信息 | 基本信息、议题、报告、决议、行动项、会议链 |
| 会议详情 | 查看单场会议 | 概览、议题、决议、行动项、会议纪要、评估 |
| 会前准备 | 会前材料与数据准备 | 准备度检查清单、报表就绪检查、报告状态、材料诊断、催办 |
| 会中驾驶舱 | 会议召开实时视图 | 当前议题、报表嵌入、报告章节、批注、决议速记、计时 |
| 会议纪要 | 编写与确认纪要 | 纪要骨架、决议汇总、行动项汇总、定版、发布到 KMS |
| 会议评估 | 会议效果评估 | 过程指标、闭环率、主观评价 |
| 通知发送 | 推送通知 | 选择通知类型、预览、选择目标群/人 |
| 模板管理 | 维护会议模板 | 模板列表、新建/编辑、应用统计 |

### 6.3 决议中心应用

定位：跨会议、跨周期的决议资产库。

```
决议中心
├── 决议概览
│   └── 决议总览
├── 决议管理
│   ├── 决议列表
│   └── 决议详情
├── 决议流程
│   ├── 决议审批
│   └── 决议督办
└── 决议资产
    └── 决议归档
```

| 页面 | 作用 | 关键功能 |
|---|---|---|
| 决议总览 | 决议全景 | 总量、状态分布、闭环率、逾期决议、趋势 |
| 决议列表 | 所有决议 | 筛选、搜索、状态、负责人、截止时间、关联会议 |
| 决议详情 | 单条决议完整信息 | 内容、来源议题、关联会议、行动项、进展、审批记录 |
| 决议审批 | 决议审批流程 | 待审批、已审批、驳回、审批意见 |
| 决议督办 | 跟踪执行 | 责任人视图、逾期预警、进展填报、催办 |
| 决议归档 | 已闭环决议归档 | 历史查询、导出、知识化 |

定位：行动项执行与闭环追踪。

```
督办中心
├── 督办概览
│   └── 督办总览
```

| 页面 | 作用 | 关键功能 |
|---|---|---|
| 督办总览 | 行动项全景 | 总量、完成率、逾期率、责任人排行榜 |
| 我的待办 | 当前用户任务 | 今日待办、本周待办、逾期、已完成 |

| 督办报告 | 定期报告 | 周报/月报、按部门/责任人汇总、导出 |

### 6.5 经营分析报表平台

定位：经营分析报表的统一目录、元数据管理和关联消费入口。不替代 FR/BI，只做编排。

```
经营分析报表平台
├── 报表浏览
│   ├── 报表总览
│   ├── 报表目录
│   ├── 我的收藏
│   └── 最近查看
├── 报表详情
│   └── 单张报表详情
└── 报表管理
    ├── 报表维护
    ├── 关联管理
    └── 使用统计
```

| 页面 | 作用 | 关键功能 |
|---|---|---|
| 报表总览 | 报表全景 | 总量、分类分布、高频报表、近期更新、我负责的报表 |
| 报表目录 | 浏览报表 | 按业务域/会议场景/使用层级/更新频率筛选；搜索和标签 |
| 我的收藏 | 常用报表 | 快速访问 |
| 最近查看 | 访问记录 | 最近 30 天 |
| 报表详情 | 单张报表完整信息 | 元数据、FR/BI 预览/链接、关联指标、关联议题、关联会议、关联报告、使用统计 |
| 报表维护 | 维护元数据 | 新增/编辑/下架；名称、类型、URL、业务域、会议场景、层级、指标、更新频率、责任人 |
| 关联管理 | 批量维护关联 | 报表 ↔ 指标、报表 ↔ 议题模板、报表 ↔ 会议场景 |
| 使用统计 | 消费情况 | 查看次数、会议引用次数、报告引用次数、责任人排行、闲置预警 |

#### 报表分类设计

**按业务域**：销售、供应链、生产、财务、人力资源、市场/品牌。

**按会议场景**：战区月度经营分析会、本部月/双周会、落后战区业绩承诺会、片联季度会议、专题经营分析会。

**按使用层级**：CEO/一号位、VP/高管、部门负责人、一线执行。

**按更新频率**：实时、每日、每周、每月、每季度、每年。

### 6.6 AI 功能应用

定位：嵌入经营分析会全流程的智能辅助能力。

```
AI 功能
├── AI 概览
│   └── AI 总览
├── AI 助手
│   └── AI 助手页
├── 会前智能
│   ├── 材料诊断
│   ├── 报告生成
│   └── 指标推荐
├── 会中智能
│   ├── 纪要生成
│   ├── 决议识别
│   └── 行动项识别
├── 会后智能
│   └── 复盘分析
└── AI 配置
    └── AI 配置页
```

| 页面 | 作用 | 关键功能 |
|---|---|---|
| AI 总览 | AI 能力全景 | 使用统计、待处理任务、最近生成记录、能力地图 |

| 材料诊断 | 会前材料评审 | 上传/链接 KMS 报告、完整性诊断、数据质量诊断、决策导向诊断、上会建议 |
| 报告生成 | 自动生成报告 | 选择会议/周期、读取报表数据、生成经营分析报告草稿、人工校对、定版到 KMS |
| 指标推荐 | 推荐指标 | 输入议题主题、推荐报表平台中的指标、口径建议、数据来源建议 |
| 纪要生成 | 自动生成纪要 | 导入会议录音/转写/批注、提取关键讨论、生成纪要骨架、人工编辑、发布到 KMS |
| 决议识别 | 提取决议 | 导入会议纪要、识别决议点、推荐责任人/截止时间、一键转决议 |
| 行动项识别 | 拆解行动项 | 选择决议、AI 拆解行动项、推荐负责人、生成验收标准 |
| 复盘分析 | 跨会议洞察 | 议题重复率、决议闭环趋势、指标异常根因、风险预警 |
| AI 配置 | 能力管理 | 模型选择、Prompt 管理、能力启用开关、审核规则 |

- 支持在当前议题下快速生成决议；

### 7.3 会议纪要模块

#### 7.3.1 目标
自动生成会议纪要骨架，汇总决议和行动项，定版后发布到 KMS。

#### 7.3.2 纪要内容

- 会议基本信息
- 出席/缺席人员
- 议题摘要（每个议题的讨论要点）
- 决议清单
- 行动项清单
- 遗留问题
- 下次会议待办

#### 7.3.3 生成方式

- **AI 生成**：基于会议录音/转写/批注/报告，自动生成纪要骨架；
- **人工编辑**：在 AI 生成基础上修改；
- **模板复用**：基于历史纪要模板生成。

#### 7.3.4 定版流程

```
纪要草稿 → 人工校对 → 决议/行动项确认 → 纪要定版 → 发布到 KMS → 通知相关人员
```

### 7.4 决议管理模块

#### 7.4.1 目标
把会议产生的决策事项沉淀下来，追踪从产生到闭环的全过程。

#### 7.4.2 决议状态流转

```
草案 → 待审批 → 已通过 → 执行中 → 已闭环 → 已归档
        ↓         ↓         ↓
      被驳回    被否决    被中止
```

#### 7.4.3 决议来源

- 会中驾驶舱手动录入；
- AI 决议识别从纪要中提取；
- 批量导入。

### 7.5 督办管理模块

#### 7.5.1 目标
确保决议落地的行动项被执行和闭环。

#### 7.5.2 行动项字段

- 内容
- 责任人
- 协助人
- 截止时间
- 验收标准
- 关联决议
- 关联会议
- 验证报表
- 进展记录
- 状态

#### 7.5.3 进展填报

- 责任人定期填报进展；
- 支持附件、截图、链接；
- 支持引用报表平台中的验证报表；
- 填报后自动通知相关人。

### 7.6 报表平台模块

#### 7.6.1 目标
把现有监控执行的报表中心升级为经营分析报表平台，作为经营分析报表的统一入口。

#### 7.6.2 核心能力

| 能力 | 说明 |
|---|---|
| 报表目录 | 按业务域/会议场景/层级/更新频率分类浏览 |
| 元数据管理 | 维护报表的经营属性 |
| 关联管理 | 报表 ↔ 指标、报表 ↔ 议题、报表 ↔ 会议场景 |
| 使用统计 | 查看次数、会议引用、报告引用 |
| 报表推荐 | 基于议题/会议场景推荐相关报表 |
| 收藏/最近查看 | 个人常用报表快速访问 |

#### 7.6.3 与会议系统的集成

- 议题编辑时可从报表平台选择报表；
- 会前准备检查报表就绪状态；
- 会中驾驶舱嵌入报表；
- 行动项进展可引用报表作为验证。

### 7.7 AI 功能模块

#### 7.7.1 会前智能

**材料诊断**：
- 输入 KMS 报告 URL；
- AI 从完整性、数据质量、决策导向、风险识别等维度诊断；
- 输出诊断结果和修改建议；
- 给出"准予上会 / 需补充 / 不建议上会"结论。

**报告生成**：
- 选择会议/周期；
- AI 读取报表平台中的关联报表；
- 生成 KMS 报告草稿；
- 人工校对后发布。

**指标推荐**：
- 输入议题主题；
- AI 从报表平台推荐相关指标；
- 给出指标定义、口径、数据来源建议。

#### 7.7.2 会中智能

**纪要生成**：
- 输入会议录音/转写/批注；
- AI 提取关键讨论、决议、行动项；
- 生成纪要骨架。

**决议识别**：
- 输入会议纪要；
- AI 识别决议点；
- 推荐责任人和截止时间。

**行动项识别**：
- 输入决议内容；
- AI 拆解为具体行动项；
- 推荐负责人和验收标准。

#### 7.7.3 会后智能

**复盘分析**：
- 读取历史报表数据；
- 读取历史 KMS 会议纪要；
- 分析议题重复率、决议闭环率、指标趋势；
- 输出风险预警和改进建议。


## 8. 流程设计

### 8.1 经营分析会全流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  会前   │ →  │  会中   │ →  │  会后   │ →  │  督办   │ →  │  复盘   │
└────┬────┘    └────┬────┘    └────┬────┘    └────┬────┘    └────┬────┘
     │              │              │              │              │
     ▼              ▼              ▼              ▼              ▼
 创建会议        会中驾驶舱      会议纪要       进展填报       复盘分析
 选择模板        议题讨论        决议确认       逾期预警       效果评估
 关联报表        批注提问        行动项分解     督办报告       下一轮输入
 生成报告        决议速记        发布到 KMS                    
 会前准备        计时提醒                                     
```

### 8.2 新建会议流程

```
1. 用户进入会议管理 → 创建会议
2. 选择会议模板（可选）
   ├─ 若选择模板，自动带入预设议题结构
   └─ 若不选择模板，从空白开始
3. 填写会议基础信息（标题、时间、地点、场景、层级）
4. 编辑议题
   ├─ 新增/删除/排序议题
   ├─ 设置议题类型、时长、责任人、演讲人
   └─ 从报表平台为议题选择关联报表
5. 设置会议链（上游/下游会议）
6. 保存会议，进入会前准备阶段
```

### 8.3 会前准备流程

```
1. 系统生成准备度检查清单
2. 检查会前报告
   ├─ 若未生成 → 跳转 AI 报告生成
   └─ 若已生成 → 检查 KMS 发布状态
3. 检查议题报表就绪状态
   ├─ 每张关联报表显示最后更新时间、责任人
   └─ 未就绪则催办或跳转 BI
4. 会前报告 AI 材料诊断
   ├─ 诊断通过 → 继续
   └─ 诊断不通过 → 查看详情并修改
5. 检查参会人确认状态
   └─ 未确认则发送通知
6. 所有检查项通过后，会议进入"待召开"状态
```

### 8.4 会中驾驶舱流程

```
1. 会议组织者点击"开始会议"
2. 系统加载第一个议题
   ├─ 显示议题标题、责任人、演讲人、时长
   ├─ 嵌入议题关联的 FR/BI 报表
   └─ 显示会前报告对应章节
3. 议题讨论
   ├─ 高管可批注、提问
   └─ 系统记录批注和讨论要点
4. 议题结束时
   ├─ 若形成决议 → 录入决议或 AI 识别决议
   └─ 若需继续讨论 → 标记为遗留问题
5. 切换下一议题
6. 所有议题讨论完成后，会议结束
   ├─ 汇总决议清单
   ├─ 汇总行动项清单
   └─ 进入会议纪要编写
```

### 8.5 会后闭环流程

```
1. 会议纪要编写
   ├─ AI 生成纪要骨架
   ├─ 人工校对
   └─ 确认决议和行动项
2. 纪要定版
   ├─ 发布到 KMS
   └─ 通知相关人员
3. 决议进入决议中心
   ├─ 待审批 → 已通过
   └─ 分解为行动项
4. 行动项进入督办中心
   ├─ 责任人执行
   ├─ 定期填报进展
   └─ 完成后提交验收
5. 行动项闭环后，关联决议进入"已闭环"状态
6. 会议进入"已完成"状态，可进行评估
```

### 8.6 报表平台使用流程

```
1. 数据管理员进入报表平台 → 报表维护
2. 新增/编辑报表元数据
   ├─ 名称、类型、URL、责任人
   ├─ 业务域、会议场景、使用层级
   ├─ 关联指标
   └─ 更新频率
3. 进入关联管理
   ├─ 维护报表与议题模板的关联
   └─ 维护报表与会议场景的关联
4. 报表发布后，用户可在报表目录中浏览
5. 会议组织者编辑议题时，从报表平台选择报表
6. 系统自动统计报表使用次数
```

## 11. 非功能需求

### 11.1 性能

- 会议列表加载 ≤ 2s；
- 会中驾驶舱议题切换 ≤ 1s；
- 报表嵌入加载 ≤ 5s（依赖 FR/BI）；
- AI 生成响应 ≤ 30s。

### 11.2 可用性

- 支持 Chrome、Edge、Safari 最新两个大版本；
- 移动端支持查看和简单操作；
- 关键操作有确认和回滚机制。

### 11.3 安全性

- 与 KMS 通信使用 HTTPS；
- KMS Token 不暴露给前端，通过后端代理；
- 报表 URL 仅对授权用户可见；
- 会议敏感数据按组织权限隔离。

### 11.4 可扩展性

- 应用场景、报表类型、AI Skill 支持配置化扩展；
- 新应用可通过全局导航插件化接入；
- 指标库、会议模板可独立维护。

### 11.5 可维护性

- 巨石页面 `meetings.html` 逐步拆分为独立应用模块；
- 核心对象模型统一，避免数据冗余；
- API 设计遵循 RESTful 规范。

## 12. 实施路线图

### Phase 1：夯实基础（4-6 周）

目标：补齐一报一会闭环，建立三大对象关联。

- 会议模型增加报告、议题、报表关联；
- 会前准备页：准备度检查清单；
- 会中驾驶舱：议题切换、报表嵌入、决议速记；
- 会议纪要：生成、定版、发布到 KMS；
- 决议/行动项与会议强关联。

### Phase 2：报表平台升级（3-4 周）

目标：将现有报表中心升级为经营分析报表平台。

- 报表目录和分类；
- 报表元数据管理；
- 报表与议题/会议场景关联；
- 会议系统引用报表平台。

### Phase 3：AI 能力嵌入（4-6 周）

目标：将会前、会中、会后 AI 能力产品化。

- 材料诊断升级；
- 报告生成；
- 纪要生成；

## 13. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 巨石文件拆分成本高 | 开发周期长 | 分阶段拆分，优先提取会前/会中/会后模块 |
| KMS API 不稳定 | 报告集成受阻 | 增加本地缓存和降级机制 |
| FR/BI 报表嵌入性能差 | 会中体验差 | 支持报表截图/摘要模式，关键指标本地缓存 |
| AI 生成质量不稳定 | 用户不信任 | 所有 AI 输出需人工确认，保留修改入口 |
| 用户习惯现有流程 | 推广阻力 | 保留现有会议列表/编辑功能，新能力渐进式接入 |
