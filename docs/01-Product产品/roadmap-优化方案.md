# DSTE 开发路线图（Roadmap）页面优化方案

> 文档版本：v1.0
> 创建日期：2026-06-15
> 状态：待确认
> 关联页面：`src/cockpit.html#dashboard/roadmap`

---

## 一、当前问题诊断

### 1.1 信息层级不清晰

当前页面按顺序垂直堆叠：

```
标题 + 筛选
战略态势横幅
版本时间线轴
甘特图 + 统计卡片
版本详情卡片
开发计划看板
```

问题：
- 用户进入页面后，**第一眼看不到最核心的结论**：现在是什么版本？下一个版本什么时候？有没有延期风险？
- 重要信息（最新版本、统计数据、 upcoming 里程碑）分散在 5 个独立区块
- 版本详情卡片占用了大量垂直空间，6 个版本就要滚动一屏

### 1.2 布局不灵活

- 甘特图 + 统计卡片是固定 2fr + 1fr，但统计卡片内容单薄，右侧空间浪费
- 时间线轴是横向滚动，节点多时需要左右拖动
- 开发计划看板是 4 列等宽，卡片内容拥挤

### 1.3 交互能力弱

- 筛选只联动甘特图，不联动时间线、版本详情、看板
- 没有搜索/过滤版本的能力
- 看板只是静态展示，无法拖拽、展开详情
- 没有版本对比功能

### 1.4 视觉重点不突出

- 所有模块视觉权重相同，没有"当前焦点"
- 状态颜色使用一致，但缺少大尺寸 KPI 数字
- 进度条样式普通，缺乏视觉吸引力

---

## 二、优化目标

| 目标 | 说明 |
|------|------|
| **一眼看清** | 3 秒内获取：当前版本、下一目标、整体进度、风险项 |
| **灵活布局** | 支持 2 栏/单栏切换、模块折叠、响应式适配 |
| **信息聚焦** | 按优先级呈现信息，次要信息可折叠或放次级面板 |
| **交互增强** | 筛选全局联动、搜索过滤、看板拖拽、版本对比 |
| **视觉升级** | 使用卡片层级、KPI 数字、状态色带、微动画 |

## 三、 proposed 布局方案

### 3.1 整体结构（桌面端）

采用 **"顶部概览 + 左侧主内容 + 右侧辅助面板"** 的三区布局：

```
┌─────────────────────────────────────────────────────────────────┐
│  开发路线图 Road Map              [搜索] [视图切换] [筛选] [导出]  │
├─────────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  📊 执行摘要 Executive Summary                           │   │
│  │  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐           │   │
│  │  │ 当前版本 │ │ 下一目标 │ │ 已延期 │ │ 本月发布 │           │   │
│  │  │ v0.4.11 │ │ v0.5.0 │ │  0   │ │  2   │           │   │
│  │  │ 6/15   │ │ 6/30  │ │      │ │      │           │   │
│  │  └────────┘ └────────┘ └────────┘ └────────┘           │   │
│  └─────────────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────────┤
│  ┌──────────────────────────────┐  ┌──────────────────────┐    │
│  │ ① 版本时间线 / 里程碑        │  │ ③ 统计与目标         │    │
│  │ （横向或纵向可选）            │  │ 2×2 KPI + 目标版本   │    │
│  │                              │  │                      │    │
│  ├──────────────────────────────┤  ├──────────────────────┤    │
│  │ ② 模块进度甘特图             │  │ ④ 开发计划看板       │    │
│  │ 可折叠 / 可筛选              │  │ 4 列 Kanban          │    │
│  │                              │  │                      │    │
│  ├──────────────────────────────┤  ├──────────────────────┤    │
│  │ ⑤ 最新版本详情               │  │ ⑤ Upcoming 里程碑    │    │
│  │ 只展示最新 1-2 个版本         │  │ 未来 30/60/90 天      │    │
│  │ 点击"查看全部"展开列表        │  │                      │    │
│  └──────────────────────────────┘  └──────────────────────┘    │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 桌面端双栏 vs 单栏

**双栏模式（默认）**：
- 左栏 2/3：时间线、甘特图、版本详情
- 右栏 1/3：统计、看板、 upcoming

**单栏模式（小屏/用户偏好）**：
- 所有模块垂直堆叠
- 执行摘要保持顶部
- 右栏模块按优先级垂直排列

### 3.3 移动端

- 执行摘要横向滚动卡片
- 时间线改为纵向
- 甘特图横向滚动
- 看板改为横向滑动列

## 四、核心模块优化

### 4.1 执行摘要 Executive Summary（新增）

顶部新增一个高 120px 的摘要条，使用 4 个 KPI 卡片：

| KPI | 数据 | 交互 |
|-----|------|------|
| 当前版本 | v0.4.11 / 2026-06-15 | 点击跳转到版本详情 |
| 下一目标 | v0.5.0 / 2026-06-30 | 点击滚动到对应计划 |
| 已延期 | 0 | 红色高亮 |
| 本月发布 | 2 | 点击筛选本月版本 |

设计：
- 大字号数字（28px）
- 标签小字（12px）
- 卡片左侧 3px 色带（当前=primary，目标=warning，延期=danger，发布=success）
- 悬停轻微上浮

### 4.2 版本时间线（优化）

**方案 A：横向时间线（保留，但优化）**
- 节点改为圆形+图标
- 当前版本放大显示
- 已发布节点用实线连接，未来节点用虚线
- 节点悬停显示 tooltip

**方案 B：纵向时间线（推荐）**
- 左侧显示版本号/日期
- 右侧显示关键变更（最多 3 条）
- 当前版本高亮
- 占用更少垂直空间，信息密度更高

建议默认 **方案 B**，并提供切换按钮。

### 4.3 模块甘特图（优化）

- 表头固定
- 行悬停高亮
- 进度条使用分段动画
- 点击行展开模块详情（负责人、目标版本、历史进度）
- 增加"按负责人筛选"、"按版本筛选"下拉

### 4.4 统计与目标面板（右侧）

**2×2 KPI 卡片**：
- 已发布版本数
- 累计功能数
- Bug 修复数
- 计划中功能数

**目标版本卡片**：
- 下一战略目标 v0.5.0
- 倒计时/进度
- 关联计划数

### 4.5 开发计划看板（优化）

- 卡片增加优先级色条（高=红，中=黄，低=绿）
- 显示负责人首字母/头像
- 显示目标版本 + ETA
- 卡片可拖拽变更状态（待评审→设计中→开发中→测试中→已发布）
- 列标题显示计数

### 4.6 版本详情（优化）

- 默认只显示**最新 2 个版本**
- 每个版本做成横向卡片（版本号 + 日期 + 3 个关键变更标签）
- 点击"展开全部"显示完整历史
- 大幅节省垂直空间

### 4.7 Upcoming 里程碑（新增）

右侧新增模块，展示未来 30/60/90 天的关键计划：
- 时间轴形式
- 每个节点：日期 + 计划名 + 目标版本
- 帮助用户看清未来节奏

## 五、交互增强

### 5.1 全局筛选

顶部筛选按钮影响所有模块：
- 全部
- 已完成
- 开发中
- 战略必争
- 只看我的（新增）

### 5.2 搜索

- 搜索版本号、计划名、模块名
- 实时过滤

### 5.3 视图切换

- 紧凑视图 / 详细视图
- 双栏 / 单栏
- 横向时间线 / 纵向时间线

### 5.4 模块折叠

每个模块卡片右上角增加折叠按钮：
- 折叠后只保留标题和关键数字
- 状态持久化到 localStorage

## 六、视觉风格

### 6.1 色彩

复用现有 CSS 变量：
- `--primary`：当前版本、开发中
- `--success`：已完成、已发布
- `--warning`：开发中、目标版本
- `--danger`：延期、阻塞
- `--border-color`：未开始

### 6.2 字体层级

| 元素 | 字号 | 字重 |
|------|------|------|
| 页面标题 | 24px | 600 |
| 模块标题 | 16px | 600 |
| KPI 数字 | 28px | 700 |
| KPI 标签 | 12px | 400 |
| 正文 | 13px | 400 |
| 小字 | 11px | 400 |

### 6.3 动画

- KPI 数字：计数动画 0.8s
- 进度条：width 过渡 0.8s ease-out
- 卡片悬停：translateY(-2px) + shadow
- 折叠/展开：height 过渡 0.25s

## 七、实现建议

### 7.1 代码改动范围

| 文件 | 改动 |
|------|------|
| `src/cockpit.html` | 重写 `renderDevTimeline()`，新增布局、交互函数 |
| `src/lib/config.js` | 无需改动 |
| `tests/e2e/roadmap.spec.js` | 更新选择器 |
| `docs/...` | 更新 PRD |

### 7.2 数据结构

复用现有 `roadmap-data.json`，无需改结构。

新增派生数据：
- `latestVersion` = versions 最后一个
- `nextMilestone` = upcoming 第一个
- `delayedCount` = modules.filter(m => m.status === 'todo' && 逾期判断)

### 7.3 工作量估算

| 任务 | 预估 |
|------|------|
| 执行摘要 + 新布局 | 60 分钟 |
| 时间线优化 | 45 分钟 |
| 甘特图优化 | 30 分钟 |
| 看板优化 | 45 分钟 |
| 版本详情折叠 | 30 分钟 |
| 筛选/搜索/视图切换 | 60 分钟 |
| 动画与响应式 | 45 分钟 |
| 测试调整 | 30 分钟 |
| **总计** | **约 5.5 小时** |

### 7.4 风险点

- `renderDevTimeline()` 当前 400+ 行，重写时注意保留数据加载逻辑
- 看板拖拽需要处理状态持久化（先本地，后考虑写文件）
- 移动端测试需要额外时间
