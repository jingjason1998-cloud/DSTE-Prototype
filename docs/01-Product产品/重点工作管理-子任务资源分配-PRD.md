## 一、文档说明

### 1.1 编写目的

本文档定义「重点工作子任务/项目」及其资源分配能力，解决"一个重点工作下包含多个子项目，需要按子项目分配不同资源"的场景。以「垂直细分客户运营」为例，该重点工作可拆解为「行业客户拓展」「华东客户深耕」等子任务，每个子任务由不同人员负责和执行。

### 1.2 变更历史

| 版本 | 日期 | 变更人 | 变更内容 |
|------|------|--------|----------|
| v1.0 | 2026-06-29 | 产品/技术 | 初始版本：定义子任务数据模型、业务规则、交互设计与验收标准 |

### 1.3 术语表

| 术语 | 说明 |
|------|------|
| OMP | Organizational Performance Management，组织绩效管理模块 |
| 重点工作 | 战略级工作项，承接年度经营计划或经营分析会决议 |
| 子任务/子项目 | 隶属于重点工作的独立执行单元，可独立设置负责人、时间、进度、资源 |
| 父任务 | 包含子任务的顶层重点工作 |
| PersonRef | 员工唯一标识，通常为 `ldap` 或员工 ID |

## 二、产品定位与边界

### 2.1 模块定位

「子任务资源分配」是 OMP 重点工作管理的**任务拆解与资源细化能力**，解决"重点工作内部多项目并行、资源需要精细分配"的问题。它在现有「重点工作-人员配置」基础上，增加一层任务维度，使资源分配从"重点工作级"下沉到"子项目级"。

### 2.2 核心价值

1. **细化管理**：将大型重点工作拆分为可独立跟踪的子项目。
2. **精准资源配置**：不同子项目可分配不同执行人员，避免成员列表混在一起。
3. **进度聚合**：父任务进度可根据子任务完成情况自动汇总。

### 2.3 功能边界

| 范围 | 说明 |
|------|------|
| **做** | 为重点工作创建/编辑/删除子任务；按子任务独立分配成员；父任务进度按子任务聚合 |
| **不做** | 子任务的再下级拆解（本期只支持一级子任务）；跨重点工作的资源调度；子项目预算审批流程 |
| **限制** | 子任务不直接来源于年度计划；年度计划派生任务的子任务在 OMP 内创建 |

## 三、用户角色与场景

### 3.1 角色定义

| 角色 | 职责 | 核心场景 |
|------|------|----------|
| 组织绩效管理员 | 统筹 OMP 重点工作的拆解与资源分配 | 将年度重点工作拆分为子项目并分配资源 |
| 部门负责人 | 管理本部门重点工作的执行团队 | 为子项目指定负责人和成员 |
| 任务负责人 | 负责具体子项目的执行 | 查看自己负责的子任务及成员 |

### 3.2 用户故事

#### 故事1：拆解重点工作为子项目
```
作为 组织绩效管理员
我希望 将「垂直细分客户运营」拆解为多个子任务
以便 分别跟踪行业拓展、区域深耕等项目的执行情况
```

#### 故事2：按子项目分配不同资源
```
作为 部门负责人
我希望 为不同子任务分配不同的执行人员
以便 资源精准匹配到具体项目
```

#### 故事3：查看父任务整体进度
```
作为 高管/决策者
我希望 父任务进度根据子任务完成情况自动汇总
以便 快速了解重点工作的整体推进状态
```

## 四、数据模型

### 4.1 OMP_KeyTask 扩展

在现有 `OMP_KeyTask` 基础上，新增 `parentId` 与 `weight` 字段：

```typescript
interface OMP_KeyTask {
  id: string;                          // 工作ID
  cycleId: string;                     // 战略周期ID
  source: 'annual_plan' | 'omp';       // 来源：年度计划 / OMP本地创建
  annualPlanTaskId?: string;           // 来源年度计划重点工作ID（仅年度计划派生任务）
  parentId?: string;                   // 新增：父任务ID；为空表示顶层重点工作
  seq: number;                         // 显示顺序
  name: string;                        // 工作名称
  bscDimension?: string;               // BSC维度
  kmsUrl?: string;                     // 知识库链接
  owner: string;                       // 负责人 PersonRef
  members: string[];                   // 参与人员 PersonRef 数组
  status: string;                      // planning | active | at_risk | delayed | done | closed
  description?: string;                // 详细描述
  type: string;                        // strategic | improvement | capability | risk_response | meeting_resolution
  priority?: string;                   // P0 | P1 | P2
  progress: number;                    // 0-100
  dept: string;                        // 责任部门
  startDate: string;                   // YYYY-MM-DD
  endDate: string;                     // YYYY-MM-DD
  budget?: number;                     // 预算（万）
  actualCost?: number;                 // 实际成本（万）
  relatedKpiIds?: string[];            // 关联KPI
  weight?: number;                     // 新增：子任务权重，用于父任务聚合计算，默认1
}
```

### 4.2 子任务字段约束

| 字段 | 父任务 | 子任务 | 说明 |
|------|--------|--------|------|
| `parentId` | 空 | 父任务ID | 建立层级关系 |
| `source` | `annual_plan` / `omp` | 仅 `omp` | 子任务不在年度计划中直接定义 |
| `annualPlanTaskId` | 可有 | 空 | 年度计划派生任务的子任务在OMP内创建 |
| `owner` | 必填 | 必填 | 子任务必须有负责人 |
| `members` | 独立维护 | 独立维护 | 父子资源独立分配 |
| `progress` | 默认自动聚合 | 手动或独立跟踪 | 父任务进度按子任务加权平均 |
| `weight` | 忽略 | 必填 | 用于父任务聚合计算，默认1 |

### 4.3 本地状态

无需新增全局状态；子任务的展开/折叠可在 `window._ompState.taskViewMode` 相关状态或任务卡片局部状态中维护。

## 五、功能需求

### 5.1 子任务管理

#### FR-1 创建子任务
- 在任务详情弹窗的「子任务」标签页中，点击「+ 添加子任务」创建。
- 新子任务默认继承父任务的 `cycleId`、`dept`、`startDate`、`endDate`、`type`。
- 子任务名称必填，权重默认 1，进度默认 0。

#### FR-2 编辑子任务
- 在子任务列表中点击子任务，进入行内编辑或弹窗编辑。
- 可修改：名称、负责人、起止时间、进度、权重、成员、预算、描述。
- 不可修改：`parentId`、`cycleId`、`source`。

#### FR-3 删除子任务
- 支持删除子任务，删除前二次确认。
- 删除子任务不影响父任务及其他子任务。

#### FR-4 子任务排序
- 支持拖拽或上下箭头调整子任务顺序。
- 顺序通过 `seq` 字段持久化。

### 5.2 按子任务分配资源

#### FR-5 在人员配置视图中展示子任务
- 左侧任务池中的父任务卡片可展开/折叠，显示其下子任务卡片。
- 子任务卡片与父任务卡片样式一致但体积略小，清晰区分层级。
- 子任务卡片同样作为拖拽接收区。

#### FR-6 拖拽分配资源到子任务
- 从右侧通讯录拖拽人员到子任务卡片，该人员加入子任务 `members`。
- 拖拽到父任务卡片，该人员加入父任务 `members`。
- 同一人可参与父任务及多个子任务，系统去重仅在同一任务内生效。

#### FR-7 移除子任务成员
- 在子任务卡片上点击成员 chip 的 ×，从子任务中移除。

### 5.3 父任务进度聚合

#### FR-8 自动聚合进度
- 父任务 `progress` 默认按子任务 `weight` 加权平均：
  - `progress = Σ(sub.progress × sub.weight) / Σ(sub.weight)`
- 仅当父任务存在子任务时启用自动聚合；无子任务时父任务进度独立维护。

#### FR-9 手动覆盖进度
- 用户可手动修改父任务进度；手动修改后暂停自动聚合。
- 提供「恢复自动计算」入口，恢复后按子任务重新聚合。

## 六、页面交互设计

### 6.1 入口

- 重点工作列表页：点击任务卡片进入任务详情/编辑弹窗。
- 人员配置视图：父任务卡片支持展开子任务，直接拖拽分配。

```
┌─────────────────────────────────────────┐
│ 编辑重点工作：垂直细分客户运营           │

│ [基本信息] [里程碑] [子任务] [资源]     │
├─────────────────────────────────────────┤
│                                         │

│ ┌───────────────────────────────────┐   │
│ │ 行业客户拓展    王总监  60%  +3人 │   │
```

- 子任务名称

## 七、业务规则

| 规则编号 | 规则内容 | 优先级 |
|----------|----------|--------|
| BR-1 | 子任务必须属于且仅属于一个父任务 | P0 |
| BR-2 | 子任务与父任务的 `members` 独立维护 | P0 |
| BR-3 | 同一人可同时存在于父任务及多个子任务 | P0 |
| BR-4 | 删除父任务时，级联删除其所有子任务 | P0 |
| BR-5 | 年度计划派生任务可创建子任务；子任务负责人不受年度计划锁定 | P0 |
| BR-6 | 父任务进度默认按子任务权重加权聚合；手动覆盖后保留手动值 | P1 |
| BR-7 | 本期子任务不再支持下级拆解（仅一级子任务） | P0 |
| BR-8 | 子任务不单独出现在年度经营计划中 | P0 |

## 八、非功能需求

### 8.1 性能
- 人员配置视图首屏加载：当前周期任务 + 人员数据 < 1.5s。
- 展开父任务显示子任务：渲染延迟 < 200ms。

### 8.2 可用性
- 创建子任务后即时刷新列表。
- 删除子任务前二次确认。
- 父任务进度自动聚合后给出轻量提示。

### 8.3 兼容性
- 主题切换：使用 CSS 变量。
- 移动端：人员配置视图保持现有响应式规则。

## 九、接口与依赖

### 9.1 新增/调整接口

| 操作 | 方法 | 接口 | 请求体 | 说明 |
|------|------|------|--------|------|
| 查询子任务 | GET | `/api/omp/tasks?parentId={parentId}` | - | 获取某父任务下的子任务 |
| 创建子任务 | POST | `/api/omp/tasks` | `{ ...OMP_KeyTask, parentId }` | 创建时传入 `parentId` |
| 更新子任务 | PATCH | `/api/omp/tasks/{taskId}` | `{ ...fields }` | 更新子任务字段 |
| 删除子任务 | DELETE | `/api/omp/tasks/{taskId}` | - | 删除子任务 |
| 调整顺序 | PATCH | `/api/omp/tasks/reorder` | `[{ taskId, seq }]` | 更新 `seq` |
| 分配成员 | PATCH | `/api/omp/tasks/{taskId}/members` | `{ members: string[] }` | 复用现有接口 |

### 9.2 前端依赖

- `omp_load('tasks')` / `omp_save('tasks', tasks)`：复用现有任务持久化。
- `getEmployees()` / `searchEmployees()`：复用通讯录能力。
- 里程碑 CRUD 模式（`src/pages/business-topics/main.js`）：子任务列表增删改排序参考。

## 十、验收标准

| 编号 | 验收项 | 通过标准 |
|------|--------|----------|
| AC-1 | 创建子任务 | 在任务详情弹窗中成功创建子任务，并出现在父任务下 |
| AC-2 | 子任务独立字段 | 子任务可设置独立的负责人、起止时间、进度、成员 |
| AC-3 | 按子任务分配资源 | 拖拽人员到子任务卡片，该人员仅加入子任务成员 |
| AC-4 | 父子资源独立 | 父任务成员与子任务成员不互相影响 |
| AC-5 | 父任务进度聚合 | 修改子任务进度后，父任务进度按权重自动更新 |
| AC-6 | 删除父任务级联 | 删除父任务后，其所有子任务不再显示 |
| AC-7 | 年度计划派生任务子任务 | 年度计划派生任务可添加子任务，子任务负责人可自由指定 |
| AC-8 | 移动端适配 | 宽度 ≤ 768px 时，子任务卡片正常显示，可操作 |

## 十一、风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 父子任务数据量大导致渲染卡顿 | 中 | 中 | 父任务默认折叠，按需展开；超过20个子任务时分页 |
| 用户混淆父子资源关系 | 中 | 中 | 在人员配置视图用视觉层级区分父/子任务；hover 提示 |
| 父任务进度手动覆盖后遗忘恢复 | 低 | 中 | 手动覆盖时显示「已手动覆盖」标签和「恢复自动计算」入口 |
