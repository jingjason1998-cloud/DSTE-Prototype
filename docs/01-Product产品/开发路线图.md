# DSTE 开发路线图 & 版本时间线

> 文档版本：v1.0
> 创建日期：2026-05-21
> 状态：待确认
> 关联项目：DSTE 战略管理平台

---

## 1. 背景与目标

### 1.1 为什么需要这个页面

DSTE 平台目前处于快速迭代期，已发布 v0.1.0 和 v0.2.0 两个版本。团队需要：
- **对内**：清晰看到产品演进节奏，明确每个模块的开发状态
- **对外**：向领导/客户展示开发进度，建立信任感
- **规划**：将未来开发计划可视化，便于优先级管理

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| 清晰 | 一眼看懂版本历史、当前进度、未来计划 |
| 简洁 | 不引入外部库，纯 CSS/JS 实现，与现有 DSTE 风格一致 |
| 可维护 | 每次发版只需修改一处数据对象，10 秒更新 |

### 3.2 模块说明

#### ① 版本时间线轴（横向）

- 横向排列所有版本节点
- 节点之间用线连接，表示时间演进
- 节点样式区分状态：
  - **已发布**：实心圆，Primary 蓝色，带版本号和日期
  - **开发中**：脉冲动画圆，Warning 黄色
  - **里程碑**：空心圆，边框加粗，表示重要目标

#### ② 模块开发进度（甘特图）

- 纵向列表展示各功能模块
- 每个模块一行：名称 + 进度条 + 百分比
- 进度条分段着色：
  - 已完成段：`var(--success)` 绿色
  - 进行中段：`var(--primary)` 蓝色
  - 未开始段：`var(--border-color)` 灰色
- 悬停显示 Tooltip：目标版本、预计完成时间

#### ③ 版本统计卡片

2×2 网格展示关键指标：
- 已发布版本数
- 累计功能数
- 累计 Bug 修复数
- 计划中功能数

#### ④ 版本详情卡片（纵向时间线）

- 按时间倒序排列版本
- 每个版本一张卡片：
  - 左侧彩色竖线（与状态对应）
  - 版本号 + 发布日期
  - 变更列表（带类型标签：Added/Fixed/Changed）
- "计划中"版本放在最下方，用虚线边框区分

### 4.2 数据更新方式

每次发版时，开发者手动更新 `roadmapData` 对象：
1. 在 `versions` 数组顶部新增版本记录
2. 更新对应 `modules` 的 `progress` 和 `status`
3. 调整 `upcoming` 列表（移除已完成的，新增新计划）

更新耗时约 1-2 分钟，无需后端参与。

## 5. 视觉设计规范

### 5.1 色彩体系（复用 DSTE 现有变量）

| 元素 | 颜色变量 | 说明 |
|------|---------|------|
| 已发布节点 | `var(--success)` #52C41A | 实心圆 |
| 开发中节点 | `var(--warning)` #FAAD14 | 脉冲动画 |
| 里程碑节点 | `var(--primary)` #1677FF | 空心圆+粗边框 |
| 进度条-已完成 | `var(--success)` | 绿色段 |
| 进度条-进行中 | `var(--primary)` | 蓝色段 |
| 进度条-未开始 | `var(--border-color)` | 灰色段 |
| 版本卡片竖线 | 与节点颜色一致 | 左侧 3px 边框 |
| 计划中卡片 | `var(--border-color)` 虚线 | 虚线边框区分 |

### 5.2 布局规范

| 模块 | 布局 | 间距 |
|------|------|------|
| 时间线轴 | 横向 flex，居中对齐 | 节点间距 80px |
| 甘特图+统计 | 左侧 2fr + 右侧 1fr grid | gap: 16px |
| 版本详情 | 纵向 flex，单列 | gap: 12px |
| 卡片内边距 | 16px | 与现有 card 一致 |
| 圆角 | 8px | 与现有设计一致 |

### 5.3 动画效果

| 交互 | 动画 | 时长 |
|------|------|------|
| 页面加载 | fadeIn + translateY(6px) | 0.25s ease |
| 进度条加载 | width 从 0% 到实际值 | 0.8s ease-out |
| 节点脉冲 | box-shadow 缩放循环 | 2s infinite |
| 卡片悬停 | translateX(4px) | 0.2s ease |

## 6. 交互设计

### 6.1 用户操作流程

```
用户进入开发路线图页面
    │
    ▼
┌─────────────────┐
│ ① 看到时间线轴   │ ← 了解已发布/开发中/里程碑版本
│ ② 看到甘特图     │ ← 了解各模块完成度
│ ③ 看到统计卡片   │ ← 了解整体数据
│ ④ 看到版本详情   │ ← 了解每个版本具体改了什么
└─────────────────┘
    │
    ▼
用户点击时间线节点
    │
    ▼
页面自动滚动到对应版本详情卡片
```

### 6.2 筛选功能

顶部提供筛选按钮组：
- **全部**（默认）：显示所有内容
- **已完成**：只显示 status='done' 的模块和已发布版本
- **开发中**：只显示 status='doing' 的模块
- **计划中**：只显示 status='todo' 的模块和 upcoming 列表

筛选时，甘特图和版本详情联动过滤，时间线轴和统计卡片保持不变。

### 6.3 Tooltip 设计

鼠标悬停在甘特图进度条上时，显示：
```
┌────────────────────────┐
│ 驾驶舱首页              │
│ 进度: 80%              │
│ 目标版本: v0.3.0       │
│ 状态: 开发中            │
└────────────────────────┘
```
