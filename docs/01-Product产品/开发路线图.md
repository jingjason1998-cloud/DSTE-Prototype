## 1. 背景与目标

### 1.1 为什么需要这个页面

DSTE 平台目前处于快速迭代期，已发布 v0.1.0 和 v0.2.0 两个版本。团队需要：
- **对内**：清晰看到产品演进节奏，明确每个模块的开发状态
- **对外**：向领导/客户展示开发进度，建立信任感
- **规划**：将未来开发计划可视化，便于优先级管理

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| 清晰 | 一眼看懂版本历史、当前进度、未来计划 |
| 简洁 | 不引入外部库，纯 CSS/JS 实现，与现有 DSTE 风格一致 |
| 可维护 | 每次发版只需修改一处数据对象，10 秒更新 |

### 3.2 模块说明

#### ① 版本时间线轴（横向）

- 横向排列所有版本节点
- 节点之间用线连接，表示时间演进
- 节点样式区分状态：
  - **已发布**：实心圆，Primary 蓝色，带版本号和日期
  - **开发中**：脉冲动画圆，Warning 黄色
  - **里程碑**：空心圆，边框加粗，表示重要目标

#### ② 模块开发进度（甘特图）

- 纵向列表展示各功能模块
- 每个模块一行：名称 + 进度条 + 百分比
- 进度条分段着色：
  - 已完成段：`var(--success)` 绿色
  - 进行中段：`var(--primary)` 蓝色
  - 未开始段：`var(--border-color)` 灰色
- 悬停显示 Tooltip：目标版本、预计完成时间

#### ③ 版本统计卡片

2×2 网格展示关键指标：
- 已发布版本数
- 累计功能数
- 累计 Bug 修复数
- 计划中功能数

#### ④ 版本详情卡片（纵向时间线）

- 按时间倒序排列版本
- 每个版本一张卡片：
  - 左侧彩色竖线（与状态对应）
  - 版本号 + 发布日期（真实数据，来自 Git tag）
  - 变更列表（带类型标签：Added/Fixed/Changed/Security）
- 数据来源：`CHANGELOG.md` 自动解析

#### ⑤ 开发计划看板（Kanban）

- 四列：待评审 / 设计中 / 开发中 / 测试中
- 每列纵向排列计划卡片
- 卡片内容：
  - 计划编号（PLAN-XXX）+ 名称
  - 优先级标签（高/中/低）
  - 负责人头像/名称
  - 目标版本 + ETA
  - 进度条（0-100%）
- 点击卡片展开详情面板：
  - 详细描述
  - 关联 GitHub Issue 链接
  - 历史状态变更记录

### 4.2 数据更新方式（自动 + 手动双轨）

#### 自动更新（版本历史）
每次打 tag 发布时，GitHub Actions 自动运行：

```bash
# .github/workflows/update-roadmap.yml
# trigger: on push tags 'v*'
node scripts/generate-roadmap-data.js
```

脚本逻辑：
1. `git tag -l 'v*'` 获取所有版本 tag
2. 读取 `CHANGELOG.md`，按版本解析 `Added/Fixed/Changed/Security` 分类
3. 生成 `src/data/roadmap-data.json`
4. 自动 commit：`chore(roadmap): auto-update for $(git describe --tags)`

#### 手动更新（开发计划）
开发计划（plans）需人工维护，更新文件 `docs/01-Product产品/开发计划.yml`：

```yaml
plans:
  - id: PLAN-001
    name: 战略地图可视化
    priority: 高
    status: 设计中
    owner: AI-1
    targetVersion: v0.4.0
    eta: "2026-06-15"
    progress: 15
    issues: ["#45", "#46"]
```

计划状态流转：
```
待评审 → 设计中 → 开发中 → 测试中 → 已发布
   ↑___________________________________|
   （测试不通过回退到开发中）
```

## 6. 交互设计

### 6.1 用户操作流程

```
用户进入开发路线图页面
    │
    ▼
┌─────────────────┐
│ ① 看到时间线轴   │ ← 了解已发布/开发中/里程碑版本
│ ② 看到甘特图     │ ← 了解各模块完成度
│ ③ 看到统计卡片   │ ← 了解整体数据
│ ④ 看到版本详情   │ ← 了解每个版本具体改了什么
└─────────────────┘
    │
    ▼
用户点击时间线节点
    │
    ▼
页面自动滚动到对应版本详情卡片
```

### 6.2 筛选功能

顶部提供筛选按钮组：
- **全部**（默认）：显示所有内容
- **已完成**：只显示 status='done' 的模块和已发布版本
- **开发中**：只显示 status='doing' 的模块
- **计划中**：只显示 status='todo' 的模块和 plans 列表
- **只看我的**：只显示当前负责人（sessionStorage 中的用户名）的计划

筛选时，甘特图、版本详情、开发计划看板联动过滤，时间线轴和统计卡片保持不变。

### 6.3 开发计划看板交互

**拖拽流转**：
- 计划卡片可在 Kanban 列之间拖拽
- 拖拽后自动更新 `docs/01-Product产品/开发计划.yml` 中的 status 字段
- 从"测试中"拖到"已发布"时，弹出确认对话框："确认计划已发布？将自动关联到对应版本。"

**卡片点击**：
- 点击展开右侧详情面板（类似经营分析会卡片）
- 面板显示：完整描述、关联 Issues、历史状态变更记录、操作日志

**快捷操作**：
- 卡片右上角「...」菜单：编辑 / 删除 / 标记阻塞 / 关联 Issue
- 阻塞状态：卡片变灰，显示阻塞原因标签

### 7.3 数据生成脚本 `scripts/generate-roadmap-data.js`

```javascript
#!/usr/bin/env node
/**
 * 自动生成 roadmap-data.json
 * 数据来源：Git tag + CHANGELOG.md + docs/01-Product产品/开发计划.yml
 */
const fs = require('fs');
const path = require('path');
const { execSync } = require('child_process');

// 1. 读取 Git tags
const tags = execSync("git tag -l 'v*' --sort=-v:refname")
  .toString().trim().split('\n').filter(Boolean);

// 2. 解析 CHANGELOG.md
const changelog = fs.readFileSync('CHANGELOG.md', 'utf8');
const versions = tags.map(tag => {
  const date = execSync(`git log -1 --format=%ai ${tag}`).toString().trim().split(' ')[0];
  // 从 CHANGELOG 提取该版本的 changes
  const section = extractChangelogSection(changelog, tag);
  return { version: tag, date, status: 'released', changes: section };
});

// 3. 读取开发计划
const plansYaml = fs.readFileSync('docs/01-Product产品/开发计划.yml', 'utf8');
const plans = parseYaml(plansYaml).plans;

// 4. 写入 JSON
const data = { versions, modules: getModulesFromPlans(plans), plans };
fs.writeFileSync('src/data/roadmap-data.json', JSON.stringify(data, null, 2));
console.log(`Generated roadmap-data.json with ${versions.length} versions, ${plans.length} plans`);
```

### 7.4 GitHub Actions 自动更新

```yaml
# .github/workflows/update-roadmap.yml
name: Update Roadmap Data

on:
  push:
    tags:
      - 'v*'

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # 需要完整 Git 历史读取 tag
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - name: Generate roadmap data
        run: node scripts/generate-roadmap-data.js
      - name: Commit and push
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add src/data/roadmap-data.json
          git diff --cached --quiet || git commit -m "chore(roadmap): auto-update for $(git describe --tags)"
          git push
```
