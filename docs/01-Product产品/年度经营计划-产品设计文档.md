## 一、场景还原：2026 年营销线组织绩效分解

### 1.1 业务背景

营销线作为公司核心增长引擎，2026 年组织绩效分解涉及 **11 个大区**、**3 个职能小组**（销售/售前/项目）、**多个本部团队**，覆盖财务/客户/内部流程三个 BSC 维度。

### 1.2 当前业务表单结构（KMS 页面还原）

```
2026年组织绩效_营销线 (KMS页面)
├── 一、营销线组织绩效（总）目标
│   └── 表格：牵引点 × KPI指标 × 全年目标 × H1占比 × 权重 × 三档目标(底线/达标/挑战) × 备注
├── 二、战区组织绩效主要指标及目标值
│   └── 表格：考核指标 × 合计 × 11个大区列
├── 三、本部团队组织绩效目标指标与目标值以及重点工作清单
│   └── 表格：职能线 × KPI × 权重 × 备注
├── 四、各子部门组织绩效 & 负责人PBC/OKR链接
│   └── 表格：部门 × 战区组织绩效链接 × PBC链接
└── 五、专项激励方案（拟）
```

### 1.3 从业务表单中提取的产品能力需求

| 业务现象 | 产品能力需求 |
|---------|------------|
| 一张大表同时包含"总目标 + 战区分解 + 职能线权重" | 需要**多视角切换**：总览视图 / 战区分解视图 / 职能线视图 |
| 同一 KPI（如销售额-D）在战区分解和职能线评价中都有出现，但权重不同 | 需要**同一指标的多维度挂载**：战区维度 × 职能维度 |
| 三档目标（底线70%/达标100%/挑战120%） | 需要**目标档位管理**，支持自定义档位和计算公式 |
| 复杂的计分规则（如"同比增长率±5个百分点为上下限"） | 需要**计分规则引擎**，支持公式配置 |
| 重点工作清单与绩效目标并列展示 | 需要**KPI ↔ 重点工作双向关联** |
| 各战区/团队有自己的 KMS 子页面 | 需要**子计划引用/链接**，支持穿透到下级分解 |
| PBC/OKR 链接独立维护 | 需要**个人绩效计划关联入口** |
| H1/H2 半年度拆分（45%/55% 默认） | 需要**周期拆分规则**，支持自动按比例拆分 |

## 二、模块定位与边界

### 2.1 在 DSTE 中的位置

```
战略制定 (SP)
    └── 战略地图（编辑态）
            ↓ 只读引用
战略解码 (BP)
    ├── 战略指标库（字典）←──── 指标定义、口径、类型
    ├── BEM 战略解码 ←──────── 战略级 KPI 锚定（5~8 个公司级）
    └── 【年度经营计划】←────── 本模块：组织级分解、KTI 挂载、任务关联
            ↓ 发布
战略执行 (Execute)
    └── 组织绩效管理 ←──────── 执行追踪、actual 录入、达成看板
```

### 2.2 职责边界

| 模块 | 做什么 | 不做什么 |
|------|-------|---------|
| **年度经营计划** | 承接 BEM 战略级 KPI → 按组织/职能分解 → 挂载 KTI → 关联任务 → 配置资源 | 不定义指标（指标库负责）、不做战略推导（BEM 负责）、不做任务执行管理（重点工作负责） |
| **战略指标库** | 指标字典：名称、口径、公式、BSC 分类、类型 | 不存年度目标值、不存组织分解关系 |
| **BEM 战略解码** | 从战略地图推导 CSF → 战略举措 → 锚定公司级 KPI | 不做组织分解 |
| **组织绩效管理** | 执行期追踪：actual 录入、达成率计算、趋势看板 | 不做树结构编辑（经营计划发布后冻结） |

## 三、数据模型设计

### 3.1 核心实体

基于真实业务场景，设计以下数据模型：

```yaml
# 年度周期
AnnualCycle:
  id: "cycle_2026"
  year: 2026
  name: "2026年度"
  phase: planning | executing | reviewing | archived
  organization: "营销线"           # 本周期所属组织
  parentCycleId: null             # 上级周期（如公司级周期）

# KPI 实例（结果指标）
KpiInstance:
  id: "kpi_2026_sales_d"
  cycleId: "cycle_2026"
  indicatorId: "ind_sales_d"      # 引用战略指标库
  parentId: null                  # null = 公司/营销线级根节点
  name: "销售额-D"                 # 实例别名，可覆盖指标库默认名
  
  # 目标体系（三档目标）
  targetValue: 178623             # 达标目标
  challengeValue: 214347.6        # 挑战目标 = 达标 × 1.2
  baselineValue: 125036.1         # 底线目标 = 达标 × 0.7
  unit: "万元"
  
  # 权重与评分
  weight: 10                      # 在父级中的权重（分值）
  scoringRule: "同比增长率±5个百分点为上下限..."   # 计分规则文本/公式
  
  # 周期拆分
  periodSplit:
    h1Ratio: 0.45                 # H1 占比
    h2Ratio: 0.55                 # H2 占比
    q1Ratio: 0.20
    q2Ratio: 0.25
    q3Ratio: 0.30
    q4Ratio: 0.25
  
  # 组织归属
  owner: "营销线负责人"
  dept: "营销线"
  level: 0                        # 0=公司/线级, 1=战区, 2=职能组, 3=个人
  
  # 分组（画布视觉）
  groupId: null                   # 可挂到分组框下
  
  # 关联
  ktiIds: []                      # 挂载的过程指标
  linkedTaskIds: []               # 关联的重点工作
  linkedPbcIds: []                # 关联的 PBC/OKR
  
  # 状态
  status: active
  x, y: null                      # 画布位置

# 分组节点（如"战区""本部团队"）
GroupNode:
  id: "grp_warzones"
  cycleId: "cycle_2026"
  name: "战区"
  type: "org"                     # org / product / region / custom
  parentGroupId: null

# 过程指标挂载（KTI）
KtiMapping:
  resultKpiId: "kpi_2026_sales_d"
  processKpiId: "kpi_2026_opp_count"
  sortOrder: 1

# 职能线权重配置（解决同一KPI在不同职能线权重不同的问题）
FunctionWeight:
  kpiInstanceId: "kpi_2026_sales_d"
  functionLine: "sales"           # sales / presale / project / overall
  weight: 10                      # 在该职能线中的权重
  scoringRule: null               # 可覆盖默认计分规则
```

## 五、核心功能流程

### 5.1 创建年度经营计划（首次）

```
[年度经营计划列表]

```
年度经营计划

## 九、实现优先级

| 优先级 | 功能 | 说明 | 依赖 |
|-------|------|------|------|
| **P0** | 年度周期模型 | `cycle` 表 + 状态机 | 无 |
| **P0** | KPI 实例表增强 | `parentId` + `cycleId` + `name` + `weight` | 战略指标库 |
| **P0** | 总览 Tab | 复刻 KMS 总目标表格 | KPI 实例表 |
| **P0** | 战区分解 Tab | 矩阵分解表 + 自动校验 | 总览 Tab |
| **P1** | 三档目标 + 周期拆分 | 底线/达标/挑战 + H1/H2/Q1-Q4 | 总览 Tab |
| **P1** | 职能线评价 Tab | 同一KPI多职能线权重 | 战区分解 Tab |
| **P1** | 发布到执行 | 状态切换 + 同步到组织绩效管理 | 全部 Tab |
| **P2** | 画布视图 | 树形可视化 + 分组框 | 分解数据 |
| **P2** | KTI 挂载 | 过程指标关联 | 画布视图 |
| **P2** | 重点工作关联 | 双向链接 | 重点工作模块 |
| **P3** | 计分规则引擎 | 复杂公式配置 | P1 完成后 |
| **P3** | 子计划穿透 | 链接到下级组织绩效页面 | KMS 集成 |
