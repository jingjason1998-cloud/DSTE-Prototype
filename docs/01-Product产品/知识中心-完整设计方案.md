## 一、产品定位

### 1.1 模块定位

**知识中心（Knowledge Center）** 是 DSTE 的独立知识资产管理模块，原内嵌在「战略专题管理」页面的「知识沉淀 Tab」将迁移至此，成为跨阶段、跨模块的统一知识入口。

知识中心不负责业务执行，只负责：
- **汇聚** 各模块产生的可复用知识资产；
- **组织** 按主题、年份、来源、类型等多维度结构化呈现；
- **检索** 支持快速定位历史决策、方法论、专题总结；
- **复用** 为新的战略洞察、专题、会议议题提供参考输入。

### 1.2 与相关模块的关系

```
┌─────────────────────────────────────────────────────────────────────┐
│                           知识中心                                   │
│  (knowledge-center)                                                  │
│   ├─ 战略专题沉淀  ←──── 战略专题管理 (sp/strategy-topics)            │
│   ├─ 业务专题沉淀  ←──── 业务专题管理 (business-topics)               │
│   ├─ 洞察沉淀      ←──── 战略洞察 (sp/insights)                       │
│   ├─ 会议决议沉淀  ←──── 经营分析会 / ST / AT                         │
│   └─ 方法论/模板库  ←──── 手动维护                                     │
└─────────────────────────────────────────────────────────────────────┘
           ↓
    为新周期 SP / BP / 执行提供参考输入
```

| 来源模块 | 沉淀内容 | 进入知识中心的方式 | 说明 |
|---------|---------|-------------------|------|
| **战略专题管理** | 已闭环战略专题的 `summary`、里程碑、关键决策 | 自动同步 | 含战区专题，仅作为知识资产 |
| **业务专题管理** | 已闭环/已归档业务专题的 `summary`、执行复盘 | 自动同步 | 聚焦当年执行，闭环后沉淀 |
| **战略洞察** | 高影响洞察（`impact === 'high'`） | 自动/手动精选 | 洞察本身不是执行结果，但可作为背景知识 |
| **经营分析会** | ST/AT 会议决议、行动项复盘 | 自动同步（预留） | 待会议模块输出接口 |
| **知识中心自身** | 方法论、模板、最佳实践 | 手动维护 | 由战略管理员或 KM 负责人录入 |

### 1.3 关键设计原则

- **独立模块**：知识中心是独立页面/模块，有自己的 URL、导航入口和数据存储；
- **只读聚合为主**：知识中心以读取各模块已闭环数据为主，不直接修改来源模块数据；
- **来源可追溯**：每条知识资产必须标注来源模块、来源 ID、来源名称，支持一键跳转回来源详情；
- **多维度组织**：支持按年份、模块来源、知识类型、标签、负责人等维度筛选和分组；
- **精选与归档**：自动同步的内容默认进入「待整理」状态，管理员可精选、打标签、归档或隐藏；
- **当年执行不混淆**：业务专题沉淀进入知识中心后，不再与当年执行态业务专题产生联动，避免状态混淆。

## 三、数据模型

### 3.1 知识资产（KnowledgeAsset）

```typescript
interface KnowledgeAsset {
  // === 主键 ===
  id: string;                    // 格式: kc_{timestamp}_{random}，safeKnowledgeAssetId 校验

  // === 来源信息（必填，确保可追溯）===
  sourceModule: 'strategy-topic' | 'business-topic' | 'insight' | 'meeting' | 'manual';
  sourceId: string;              // 来源模块的原生 ID，如 st_xxx / bt_xxx / ins_xxx
  sourceName: string;            // 来源名称，如专题名、洞察标题、会议名称
  sourceUrl?: string;            // 可选，跳转到来源详情页的 URL/路由
  externalUrl?: string;          // 可选，来源模块的外部链接（如战略专题的 KMS 链接）

  // === 知识内容 ===
  title: string;                 // 知识标题
  summary: string;               // 知识摘要/总结（富文本，XSS 防护必需）
  content?: string;              // 详细内容，支持方法论正文、模板等

  // === 分类与标签 ===
  type: 'topic-summary' | 'insight' | 'meeting-decision' | 'methodology' | 'template' | 'best-practice';
  tags: string[];                // 自定义标签，如 "大客户" "战区" "财务域"
  year: number;                  // 归属年份，用于分组和筛选
  dimension?: string;            // 战略维度（来自来源），如 BSC 四维度

  // === 负责人 ===
  owner: string | PersonObject;  // 知识负责人/来源专题负责人

  // === 状态 ===
  status: 'pending' | 'featured' | 'archived' | 'hidden';
                                // pending: 自动同步待整理
                                // featured: 精选展示
                                // archived: 已归档
                                // hidden: 隐藏不展示

  // === 元数据 ===
  createdAt: string;             // 知识资产创建时间 YYYY-MM-DD
  updatedAt: string;             // 最后更新时间 YYYY-MM-DD
  closedAt?: string;             // 来源闭环时间（如有）
}
```

## 四、功能模块设计

### 4.1 知识中心首页

页面顶部为**搜索 + 筛选栏**，下方为**统计卡片** + **知识资产列表**。

#### 4.1.1 搜索与筛选

| 筛选维度 | 控件 | 选项 |
|---------|------|------|
| 关键词 | 搜索框 | 匹配 title / summary / tags |
| 来源模块 | 多选下拉 | 战略专题 / 业务专题 / 洞察 / 会议 / 手动 |
| 知识类型 | 多选下拉 | 专题总结 / 洞察 / 会议决议 / 方法论 / 模板 / 最佳实践 |
| 年份 | 单选下拉 | 全部年份 / 具体年份 |
| 状态 | 多选下拉 | 待整理 / 精选 / 已归档 / 隐藏（管理员可见隐藏） |
| 标签 | 标签云/输入 | 热门标签快捷选择 |

#### 4.1.2 统计卡片

| 指标 | 计算方式 |
|------|---------|
| 知识资产总数 | 当前筛选范围内的 `KnowledgeAsset` 数量 |
| 战略专题沉淀 | `sourceModule === 'strategy-topic'` 数量 |
| 业务专题沉淀 | `sourceModule === 'business-topic'` 数量 |
| 本年新增 | `createdAt` 年份等于当前年的数量 |
| 精选数量 | `status === 'featured'` 数量 |

#### 4.1.3 知识资产列表

默认按 `year` 降序、`updatedAt` 降序排列。支持列表/卡片两种视图。

卡片展示字段：
- 标题
- 来源标签（战略专题 / 业务专题 / 洞察 / 会议 / 手动）
- 知识类型标签
- 年份
- 负责人
- 摘要前 120 字
- 标签
- 操作：查看详情、编辑（管理员）、精选/归档/隐藏

### 4.2 知识资产详情弹窗/页面

展示字段：
- 标题、来源模块、来源名称（可跳转）、类型、年份、状态
- 负责人
- 摘要/内容
- 外部链接（如 KMS 链接），可点击跳转
- 标签
- 年度深化链（仅战略专题来源）：展示上一阶段/下一阶段关联专题，可跳转
- 创建时间、更新时间、闭环时间
- 操作：关闭、编辑（管理员）、查看来源

### 4.3 知识资产管理（管理员）

#### 4.3.1 手动创建知识资产

用于录入方法论、模板、最佳实践等非自动同步内容。

字段：
- 标题（必填）
- 知识类型（必填）
- 摘要/内容（必填，富文本）
- 年份（默认当前年）
- 负责人
- 标签
- 状态（默认 featured）

#### 4.3.2 编辑知识资产

可编辑：标题、类型、摘要、内容、年份、负责人、标签、状态。
不可编辑：sourceModule、sourceId、sourceName（来源信息只读）。

#### 4.3.3 删除知识资产

- 自动同步来源的知识资产：提供「隐藏」和「从知识中心移除」两种操作；
  - 「隐藏」：仍保留在存储中，列表不展示；
  - 「从知识中心移除」：删除 `KnowledgeAsset` 记录，但不影响来源模块数据。
- 手动创建的知识资产：物理删除。

### 4.4 自动同步机制

#### 4.4.1 同步触发时机

知识中心页面加载时执行一次同步扫描：

```javascript
function syncKnowledgeAssets() {
  syncFromStrategyTopics();
  syncFromBusinessTopics();
  syncFromInsights();
  // syncFromMeetings(); // 预留
}
```

#### 4.4.2 同步规则

| 来源模块 | 同步条件 | 生成资产类型 | 初始状态 |
|---------|---------|-------------|---------|
| 战略专题管理 | `status === 'closed' && !deleted` | `topic-summary` | `pending` |
| 业务专题管理 | `status === 'done' \|\| status === 'archived'` | `topic-summary` | `pending` |
| 战略洞察 | `impact === 'high'` | `insight` | `pending` |
| 经营分析会 | 决议已关闭且有总结 | `meeting-decision` | `pending`（预留） |

同步时以 `sourceModule + sourceId` 为主键去重，已存在的资产只更新字段，不重复创建。

#### 4.4.3 同步字段映射

```javascript
function mapStrategyTopicToAsset(topic) {
  return {
    id: generateKnowledgeAssetId(),
    sourceModule: 'strategy-topic',
    sourceId: topic.id,
    sourceName: topic.name,
    title: topic.name,
    summary: topic.summary,
    content: topic.researchObjectives || '',
    type: 'topic-summary',
    tags: [], // 首次同步为空，由管理员后续打标签
    year: topic.year,
    dimension: topic.dimension,
    owner: topic.owner,
    externalUrl: topic.kmsUrl || '',
    status: 'pending',
    createdAt: today(),
    updatedAt: today(),
    closedAt: topic.endDate
  };
}
```

业务专题映射类似，使用 `business-topic`。

## 六、数据流设计

```
┌─────────────────────┐    ┌─────────────────────┐    ┌─────────────────────┐
│  战略专题管理        │    │  业务专题管理        │    │  战略洞察            │
│  dste_strategy_      │    │  dste_business_      │    │  dste_insights_v1   │
│  topics_v2           │    │  topics_v2          │    │                     │
└──────────┬──────────┘    └──────────┬──────────┘    └──────────┬──────────┘
           │                          │                          │
           └──────────────────────────┼──────────────────────────┘
                                      ▼
                         ┌─────────────────────────┐
                         │   知识中心同步引擎        │
                         │   syncKnowledgeAssets() │
                         │   - 按规则过滤来源数据    │
                         │   - 生成/更新资产        │
                         └───────────┬─────────────┘
                                     ▼
                         ┌─────────────────────────┐
                         │  dste_knowledge_assets_v1 │
                         │     KnowledgeAsset[]     │
                         └───────────┬─────────────┘
                                     ▼
                         ┌─────────────────────────┐
                         │      知识中心页面         │
                         │   搜索/筛选/展示/管理     │
                         └─────────────────────────┘
```

### 6.1 关键数据流说明

1. **单向同步**：知识中心只读取来源模块数据，不反向修改来源模块；
2. **去重机制**：以 `sourceModule + sourceId` 为主键，避免重复生成资产；
3. **状态独立**：知识资产的状态（pending/featured/archived/hidden）独立于来源模块状态；
4. **删除隔离**：删除或隐藏知识资产不影响来源模块数据；来源模块数据删除后，知识中心中的对应资产可选择保留（显示为来源已失效）或自动隐藏。

## 八、安全设计

| 防线 | 措施 | 说明 | 优先级 |
|------|------|------|--------|
| **XSS 防护** | 所有用户输入字段使用 `escapeHtml()` 后再插入 DOM | title / summary / content / tags / sourceName | P0 |
| **ID 校验** | `safeKnowledgeAssetId()` 白名单校验 | 防止非法 ID 注入 | P0 |
| **来源校验** | 同步时校验 `sourceId` 格式 | 如 `safeTopicId()` / safeBusinessTopicId() | P0 |
| **数据拷贝** | `loadKnowledgeAssets()` 返回浅拷贝 | 防止外部直接修改缓存 | P1 |
| **存储保护** | localStorage 读写加 try-catch | 损坏时返回默认值 | P1 |
| **状态权限** | 隐藏/删除/归档操作仅管理员可用 | 普通用户仅可查看和搜索 | P1 |
