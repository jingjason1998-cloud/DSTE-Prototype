## 一、产品概述

### 1.1 模块定位

**战略地图模块**是 DSTE 战略管理平台在 **战略制定 (SP) 阶段** 的核心工具。它帮助企业以可视化方式构建、编辑和展示战略目标体系，基于平衡计分卡 (Balanced Scorecard, BSC) 的四维度框架，通过因果关系链将战略目标从上至下（财务→客户→内部流程→学习成长）和从下至上（驱动因素→结果）完整呈现。

**核心价值主张**：
- **让战略"看得见"** —— 将抽象的战略目标转化为直观的可视化地图
- **揭示因果逻辑** —— 明确展示"为什么做"和"怎么做"之间的因果链条
- **对齐组织共识** —— 战略地图是高管团队对齐战略认知的统一语言
- **为战略解码提供输入** —— 战略地图是 BEM 战略解码和 KPI 体系设计的前置输入

## 三、数据模型

### 3.1 核心实体

```typescript
// ========== 战略地图 ==========
interface StrategyMap {
  id: string;                    // 如 "sm_2025_2027"
  name: string;                  // 如 "2025-2027战略规划地图"
  description: string;           // 地图描述
  // 规划周期（SP 为 3 年，如 2025-2027）
  planPeriod: {
    startYear: number;           // 规划起始年，如 2025
    endYear: number;             // 规划结束年，如 2027
    label: string;               // 显示文本，如 "2025-2027"
  };
  status: 'draft' | 'review' | 'approved' | 'archived';
  version: number;               // 版本号，如 1
  versionLabel: string;          // 版本标签，如 "v1.0 初始版"
  createdBy: string;
  createdAt: string;             // ISO 时间戳
  updatedBy: string;
  updatedAt: string;
  approvedBy?: string;
  approvedAt?: string;
  // 画布布局配置
  canvas: {
    width: number;               // 画布宽度，默认 1200
    height: number;              // 画布高度，默认 800
    showGrid: boolean;           // 是否显示网格
    gridSize: number;            // 网格大小，默认 20
  };
  // 年度视角配置
  annualView: {
    activeYear: number;          // 当前高亮年份，如 2025（null 则显示全周期）
    showMilestones: boolean;     // 是否在卡片上显示年度里程碑
  };
}

// ========== 战略目标 ==========
interface StrategicObjective {
  id: string;                    // 如 "so_fin_001"
  mapId: string;                 // 所属战略地图 ID
  name: string;                  // 目标名称，如 "提升营收增长率"
  description: string;           // 目标详细描述
  // BSC 四维度
  dimension: 'financial' | 'customer' | 'internal' | 'learning';
  // 在画布上的位置（百分比或像素坐标）
  position: {
    x: number;                   // 相对于画布的 X 坐标
    y: number;                   // 相对于画布的 Y 坐标
  };
  // 卡片尺寸
  size: {
    width: number;               // 默认 200
    height: number;              // 默认 80
  };
  // 样式
  style: {
    color: string;               // 卡片主色调（继承维度色或自定义）
  };
  // 关联指标（引用 KPI 指标库，非必须）
  linkedKpiIds: string[];        // 关联的 KPI 指标 ID 列表
  // 关联行动
  linkedActionIds: string[];     // 关联的重点工作/行动 ID 列表
  // ===== 年度里程碑（3年规划分解）=====
  milestones: {
    // 每个年份的指标值和完成状态
    [year: number]: {
      target: number | string;   // 当年目标值（如 "12%" 或 10000）
      actual?: number | string;  // 当年实际值（BP阶段回填）
      status: 'not_started' | 'in_progress' | 'on_track' | 'at_risk' | 'achieved';
      focusLevel: 'primary' | 'secondary' | 'none';  // 当年关注级别：重点/次要/不关注
      linkedBpKpiId?: string;    // 关联的年度BP KPI实例ID
    }
  };
  // ===== 元数据 =====
  owner: string;                 // 目标负责人/责任部门
  priority: 'high' | 'medium' | 'low';
  status: 'active' | 'at_risk' | 'achieved' | 'deprecated';
  createdAt: string;
  updatedAt: string;
}

// ========== 因果关系（目标间链接）==========
interface StrategyLink {
  id: string;                    // 如 "sl_001"
  mapId: string;                 // 所属战略地图 ID
  fromId: string;                // 源目标 ID（驱动因素）
  toId: string;                  // 目标 ID（结果）
  // 链接类型
  linkType: 'drives' | 'influences' | 'supports';
  // 链接强度
  strength: 'strong' | 'moderate' | 'weak';
  // 路径控制点（贝塞尔曲线控制点）
  controlPoints?: {
    x1: number;                  // 控制点1 X
    y1: number;                  // 控制点1 Y
    x2: number;                  // 控制点2 X
    y2: number;                  // 控制点2 Y
  };
  // 路径样式
  style?: {
    color: string;               // 线条颜色
    dashed: boolean;             // 是否虚线
    width: number;               // 线条宽度
  };
  description?: string;          // 关系说明
  createdAt: string;
}

// ========== 四维度配置 ==========
interface DimensionConfig {
  id: string;
  mapId: string;
  dimension: 'financial' | 'customer' | 'internal' | 'learning';
  label: string;                 // 显示名称，如 "财务维度"
  description: string;           // 维度描述
  // 维度在画布上的区域（可选，用于分区域布局）
  region?: {
    top: number;
    left: number;
    width: number;
    height: number;
  };
  // 维度颜色主题
  color: string;                 // 如 "#1677FF"（蓝色-财务）
  bgColor: string;               // 维度背景色
  icon: string;                  // 维度图标
  order: number;                 // 排序（从上到下：财务=1, 客户=2, 内部=3, 学习=4）
}
```

### 3.2 数据存储

```javascript
// localStorage 键名规范
const STORAGE_KEYS = {
  strategyMaps: 'dste_strategy_maps_v1',      // StrategyMap[] — 规划周期为 3 年（如 2025-2027）
  objectives: 'dste_strategy_objectives_v1',   // StrategicObjective[] — 含年度里程碑 milestones
  links: 'dste_strategy_links_v1',             // StrategyLink[] — 3 年内保持不变
  dimensions: 'dste_dimension_configs_v1',     // DimensionConfig[]
};
```

### 3.3 数据关系图

```
StrategyMap (1)
    ├── StrategicObjective (N) — 通过 mapId 关联
    │       ├── linkedKpiIds → KPI 指标库（引用）
    │       └── linkedActionIds → 重点工作（引用）
    ├── StrategyLink (N) — 通过 mapId 关联
    │       ├── fromId → StrategicObjective.id
    │       └── toId → StrategicObjective.id
    └── DimensionConfig (4) — 通过 mapId 关联
```

  - 点击已有链接选中，按 Delete 删除或右键编辑

### 4.8 战略地图与 BP（年度经营计划）的衔接

**4.8.1 数据流关系**

```
战略地图 (SP) ──────────────────────────► 年度经营计划 (BP)
     │                                          │
     │ 1. 战略目标（3年）                        │ 1. 选取当年重点目标
     │ 2. 因果逻辑关系                           │ 2. 分解为年度 KPI
     │ 3. 年度里程碑（目标值）                    │ 3. 配置当年重点工作
     │                                          │ 4. 执行并回写实际值
     │◄─────────────────────────────────────────│
              5. BP 回填实际值到战略地图
```

**4.8.2 衔接规则**

| 环节 | 战略地图 (SP) | 年度经营计划 (BP) | 说明 |
|------|--------------|------------------|------|
| **目标来源** | 战略地图定义 3 年目标 | BP 从地图中选取当年目标 | 不能脱离地图自行定义目标 |
| **指标分解** | 定义 3 年总指标和各年里程碑 | 将当年里程碑细化为季度/月度 KPI | 年度指标值必须与里程碑一致 |
| **重点工作** | 不直接关联 | 基于当年目标配置重点工作 | 重点工作的战略来源必须是地图 |
| **数据回写** | 接收 BP 回填的年度实际值 | 执行期录入实际数据 | 自动同步到战略地图的里程碑 |
| **视图查看** | 全周期视角 + 年度视角 | 只看当年数据 | BP 页面可穿透到战略地图 |

**4.8.3 使用场景**

```
场景 1：SP 阶段（4-9 月）制定 2025-2027 战略地图
        └── 确定 3 年目标、因果关系、年度里程碑

场景 2：BP 阶段（10-12 月）制定 2025 年度经营计划
        └── 从战略地图切换「2025 年度视角」
        └── 查看 2025 年重点目标（🔥 标注）
        └── 将这些目标分解为年度 KPI 和重点工作
        └── 年度 KPI 的目标值自动继承里程碑中的 2025 年值

场景 3：执行阶段（2025 全年）跟踪战略达成
        └── 在战略地图「2025 年度视角」下查看各目标完成进度
        └── BP 系统每月回写实际值到战略地图里程碑
        └── 战略地图自动计算 3 年总目标达成率

场景 4：年度复盘（2026 年初）回顾 2025
        └── 在战略地图「2025 年度视角」下查看全年完成情况
        └── 评估是否需要调整 2026-2027 的里程碑
```

### 6.4 核心算法

**6.4.1 反环检测（拓扑排序）**

```javascript
function hasCycle(mapId, newFromId, newToId) {
  // 获取现有链接 + 新链接
  const links = getLinksByMapId(mapId);
  links.push({ fromId: newFromId, toId: newToId });

  // 构建邻接表
  const adj = new Map();
  links.forEach(l => {
    if (!adj.has(l.fromId)) adj.set(l.fromId, []);
    adj.get(l.fromId).push(l.toId);
  });

  // DFS 检测环
  const visited = new Set();
  const recStack = new Set();

  function dfs(node) {
    visited.add(node);
    recStack.add(node);
    const neighbors = adj.get(node) || [];
    for (const neighbor of neighbors) {
      if (!visited.has(neighbor) && dfs(neighbor)) return true;
      if (recStack.has(neighbor)) return true;
    }
    recStack.delete(node);
    return false;
  }

  for (const node of adj.keys()) {
    if (!visited.has(node) && dfs(node)) return true;
  }
  return false;
}
```

**6.4.2 贝塞尔曲线路径计算**

```javascript
function calculateBezierPath(fromEl, toEl) {
  const fromRect = fromEl.getBoundingClientRect();
  const toRect = toEl.getBoundingClientRect();

  // 计算连接点（右出左入）
  const fromX = fromRect.right;
  const fromY = fromRect.top + fromRect.height / 2;
  const toX = toRect.left;
  const toY = toRect.top + toRect.height / 2;

  // 控制点：水平方向的平滑曲线
  const cp1x = fromX + (toX - fromX) * 0.5;
  const cp1y = fromY;
  const cp2x = toX - (toX - fromX) * 0.5;
  const cp2y = toY;

  return `M ${fromX} ${fromY} C ${cp1x} ${cp1y}, ${cp2x} ${cp2y}, ${toX} ${toY}`;
}
```

### 8.2 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| SVG 链接在缩放/平移时位置计算复杂 | 中 | 使用百分比坐标 + 视图变换矩阵统一处理 |
| 大量目标（>50）时渲染性能下降 | 中 | 虚拟渲染：仅渲染视口内元素；后续优化 |
| 多标签页同时编辑导致数据冲突 | 低 | 增加版本时间戳，保存前检测冲突 |
| 与其他模块数据引用（KPI/行动）未实现 | 中 | 先使用 Mock 数据占位，待模块实现后替换 |
