# 落后战区业绩承诺会审核原则

> **适用场景**：战区级别业绩落后人员的业绩承诺汇报材料
> **材料形式**：KMS链接（飞书文档/企业Wiki/PPT等）
> **审核目标**：确保业绩承诺「可衡量、可追踪、可问责」，不是「喊口号」而是「立军令状」
> **方法论参考**：华为"一报一会"经营分析会 —— 差距→根因→改善→下一场怎么打

---

## 一、审核框架总览

| 审核维度 | 权重 | 核心拷问 | 一票否决 |
|----------|------|---------|---------|
| **目标承诺** | 20% | 业绩承诺是否明确、完整、可衡量？经营质量指标是否齐全？ | **目标承诺 = 0 分 → R5 退回重写** |
| **行动具体化** | 30% | 行动方案是否符合 SMART 原则？是否对应真因？是否有记分牌？ | **行动具体化 < 10 分 → R5 退回重写** |
| **材料规范度** | 35% | 是否遵循华为"一报一会"结构？必需章节是否齐全？ | 缺少必需章节 → 严重扣分 |
| **数据质量** | 15% | 数据是否准确、逻辑自洽？经营指标间关系是否合理？ | **数据质量 < 5 分 → R5 退回重写** |

**R 级制**：
- **R1 (90-100)**：材料达标，可直接上会
- **R2 (80-89)**：基本达标，微调后上会
- **R3 (70-79)**：明显不足，修改后需重新评审
- **R4 (60-69)**：较多问题，大幅修改后重新评审
- **R5 (0-59)**：严重不达标，退回重写

**一票否决红线**：
- 零业绩承诺
- 行动方案完全无责任人
- 行动方案完全无时间节点
- 数据存在重大错误导致决策误导
- 行动与根因完全不对应

## 二、华为"一报一会"标准结构

材料必须遵循华为经营分析会标准结构：

### 2.1 五大必需章节

| # | 章节 | 核心要求 | 华为原则 |
|---|------|---------|---------|
| 1 | **上月决议执行复盘** | 上月承诺→实际→差距→根因→纠正 | 铁律：先复盘上月，再谈本月 |
| 2 | **经营仪表盘** | 一页纸关键经营指标全貌 | 一页纸看清经营全貌 |
| 3 | **差距分析** | 目标vs实际vs基准，按维度拆解 | 差距导向，不绕弯子 |
| 4 | **根因分析** | 用5Why或鱼骨图，找到真因 | 归因于内，主观原因≥40% |
| 5 | **改善行动（任务令）** | 差距→根因→行动→责任人→时间→记分牌 | 一一对应，闭环管理 |

### 2.2 华为"差距→根因→改善→下一场怎么打"逻辑流

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   差距分析   │ → │   根因分析   │ → │   改善行动   │ → │   下一场怎么打 │
│  (目标vs实际)│    │  (找到真因)  │    │  (任务令)    │    │  (业绩承诺)   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

## 三、4大审核维度详解

### 3.1 目标承诺（20%）

**目标承诺必须包含"三层目标"**：

| 层级 | 说明 | 示例 |
|------|------|------|
| **保底** | 最低可接受目标 | 签约400万 |
| **目标** | 正常努力可达成的目标 | 签约500万 |
| **挑战** | 需要突破性努力的目标 | 签约600万 |

**经营质量指标**（华为"大质量观"）：

| 指标类型 | 要求 | 说明 |
|----------|------|------|
| 收入/签约 | **必须有** | 最基本的业绩指标 |
| 利润/毛利率 | **建议有** | 避免"增收不增利" |
| 回款/现金流 | **必须有** | 华为最看重的指标之一 |
| 客户满意度 | **建议有** | NPS或等效指标 |

### 3.2 行动具体化（30%）

**华为根因三标准**：

| 标准 | 检验问题 | 不合格示例 | 合格示例 |
|------|---------|-----------|---------|
| **标准1** | 做了这个动作，问题还会发生吗？ | "加大市场投入"（市场还是会激烈） | "建立竞品价格监控周报机制"（问题不再发生） |
| **标准2** | 做了这个动作，目标就一定能实现吗？ | "加强客户沟通"（不一定能签约） | "提前2周与客户CFO对齐审批节点"（确保签约） |
| **标准3** | 是我能做的、尽我能力能做的吗？ | "等市场好转"（不可控） | "增加2名销售，拓展新渠道"（可控） |

**任务令格式**：

```
任务令：[序号]
对应根因：[根因描述]
行动内容：[具体动作]
责任人：[姓名+职位]
完成时间：[具体日期]
验收标准：[完成标志]
记分牌：[跟踪频率+输出物]
```

### 3.3 材料规范度（35%）

**上月决议执行复盘**（如适用）：
- 上月承诺 → 实际结果 → 差距 → 根因 → 纠正措施
- 每个任务令必须有完成状态：✅完成 / ⚠️进行中 / ❌未完成
- 未完成的必须有根因分析和纠正措施

**经营仪表盘**要求：
- 一页纸
- 至少4个关键经营指标
- 每个指标显示：目标 / 实际 / 差距 / 同比趋势
- 可视化呈现（表格/图表）

**差距分析**要求：
- 按客户/区域/产品/时间维度拆解
- 量化每个子差距
- 识别最大缺口来源（Pareto分析）

**根因分析**要求：
- 使用结构化工具（5Why / 鱼骨图 / 漏斗分析）
- 连续问三个"为什么"找到真因
- 主观/内部原因≥40%
- 区分症状和根因

**改善行动**要求：
- 一一对应：每个根因→至少1个行动
- 任务令格式：行动→责任人→时间→验收标准→记分牌
- 记分牌机制：每周/每月review

### 3.4 数据质量（15%）

**6项校验**：

| # | 检查项 | 合格标准 |
|---|--------|---------|
| 1 | 同比/环比计算 | 公式正确，基数明确 |
| 2 | 百分比总和 | 占比合计=100% |
| 3 | 关键指标计算 | 如利润率=利润/收入 |
| 4 | 正文与表格一致 | 数字前后一致 |
| 5 | 数量级合理 | 无小数点/单位错误 |
| 6 | 经营指标逻辑 | 指标间关系合理（如收入增长利润下降需解释） |

## 四、红线清单

| 红线 | 触发条件 | 后果 |
|------|---------|------|
| 零业绩承诺 | 材料没有明确的业绩承诺目标 | R5 退回重写 |
| 零责任人 | 所有行动项都没有指定负责人 | R5 退回重写 |
| 零时间节点 | 所有行动项都没有具体日期 | R5 退回重写 |
| 数据重大错误 | 计算错误导致决策误导 | R5 退回重写 |
| 行动与根因不对应 | 所有行动都没有对应到材料中的根因 | R5 退回重写 |

---

## 五、问题分级

| 级别 | 问题类型 | 处理方式 |
|------|---------|---------|
| **R5** | 触碰红线、零承诺、零责任人、零时间、数据重大错误、行动与根因不对应 | 退回重写 |
| **R4** | 缺少上月复盘、缺少经营仪表盘、差距无拆解、根因全外部、建议雷同、缺少记分牌 | 大幅修改后重审 |
| **R3** | 同比基数未说明、承诺缺数据支撑、经营指标逻辑矛盾、缺验收标准、数量级错误 | 修改后重审 |
| **R2** | 个别建议不够具体、个别数据小错误、格式问题 | 微调后上会 |
| **R1** | 全部达标 | 直接上会 |
