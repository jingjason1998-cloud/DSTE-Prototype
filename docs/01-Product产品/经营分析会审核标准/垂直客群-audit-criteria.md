# 垂直客群客户 · 业绩落后述职材料审核原则

> **适用场景**：垂直客群客户业务线中，业绩目标未达成人员需向上级/HRBP进行述职说明  
> **材料形式**：KMS链接（飞书文档/企业Wiki等）  
> **审核目标**：确保述职材料不回避问题、分析到位、下一步计划体现前瞻性工作部署，避免流于形式

---

## 一、审核框架总览

| 审核维度 | 权重 | 说明 |
|----------|------|------|
| **完整性** | 30% | 必备要素是否齐全 |
| **数据真实性** | 15% | 业绩数据是否准确、口径是否一致 |
| **根因分析深度** | 20% | 是否穿透表象，找到真正原因 |
| **下一步计划/改进措施** | 20% | 是否明确下一阶段工作部署，不仅响应问题，更要有前瞻性规划 |
| **SP战略关联度** | 10% | 下一步计划是否对齐公司/部门SP战略规划，避免局部优化 |
| **态度与反思** | 5% | 是否真诚面对问题，而非推卸责任 |

**通过标准**：总分 ≥ 75分，且「根因分析深度」≥ 12分（单维度底线，防止套话敷衍）

---

## 二、完整性审核（30%）

### 2.1 必备要素清单

述职材料必须包含以下 **6大模块**，缺一项扣15分（封顶25分）：

| 序号 | 必备模块 | 审核要点 | 常见问题 |
|------|----------|----------|----------|
| 1 | **业绩回顾** | 明确列出考核周期、目标值、实际值、达成率 | 只写"未完成"，没有具体数字 |
| 2 | **差距拆解** | 将总差距拆解到客户/产品/区域/时间等维度 | 笼统说"市场不好"，没有结构化解构 |
| 3 | **根因分析** | 用数据或事实支撑，区分主观原因 vs 客观原因 | 只列客观原因（市场环境、竞品降价），回避自身问题 |
| 4 | **下一步计划/改进措施** | 明确下一阶段工作部署：既有对根因的响应，也要有前瞻性规划 | 仅罗列问题对应的补救措施，缺乏下一阶段系统部署 |
| 5 | **资源需求** | 明确需要什么支持（人力、预算、政策、培训） | 不写资源需求，或需求不合理（要10个人做1个人的事） |
| 6 | **时间节点** | 下一步计划有明确的里程碑和验收标准 | "尽快完成""努力提升"等模糊表述 |

### 2.2 辅助要素（加分项）

- 对标优秀案例：是否分析了同期优秀同事的做法差异
- 过程数据呈现：是否有过程指标（拜访量、转化率、客单价等）佐证
- 风险预判：是否预判了改进过程中可能遇到的新风险

### 2.3 补充材料：体系客户SP战略规划对照表

述职人需额外提交一份 **体系客户SP战略规划对照表**（表格形式），作为审核「SP战略关联度」的基础依据。

**表格模板**：

| 字段 | 填写说明 | 必填 |
|------|----------|------|
| **客户体系名称** | 述职人所属/负责的客户体系全称（如：垂直客群-金融行业子体系） | ✅ |
| **体系SP战略规划KMS链接** | 该客户体系当前生效的SP战略规划文档KMS链接（提交后由后台自动抓取解析） | ✅ |
| **述职人负责客户范围** | 在该体系中具体负责的客户名单/区域/行业细分 | ✅ |
| **SP重要机会点（系统自动提取）** | 系统从SP文档中自动提取的关键机会点/重点方向，供比对用 | 🤖 自动 |
| **述职人下一步计划与关键战役的对应关系** | 简述述职人的下一步计划如何支撑该体系的关键战役 | ❌ |

**提交方式**：
- 在KMS述职材料中直接嵌入该表格
- 或在提交审核时，在「补充材料」字段中粘贴该表格的KMS链接

**后台自动处理流程**：
1. 系统收到KMS链接后，自动抓取SP文档内容
2. 自动提取SP中的重要机会点、关键战役、战略方向
3. 与述职材料进行文本比对，检查述职人是否提及这些机会点
4. 比对结果直接写入审核报告「SP战略关联度」维度

**审核规则**：
- 未提交该对照表，完整性扣 **10分**
- 提交了但「体系SP战略规划KMS链接」为空或无效，扣 **5分**
- 链接内容与述职人声称的客户体系不符，扣 **10分**

## 三、数据真实性审核（15%）

### 3.1 数据口径一致性

| 检查项 | 审核规则 | 问题等级 |
|--------|----------|----------|
| 目标值口径 | 述职中的目标值与年初签订的业绩合同/OKR是否一致 | 🔴 致命 |
| 实际值口径 | 实际值与财务/CRM系统导出数据是否一致 | 🔴 致命 |
| 时间口径 | 考核周期是否明确（如"2025年Q1"而非"今年上半年"） | 🟡 警告 |
| 计算逻辑 | 达成率 = 实际值/目标值，计算是否正确 | 🔴 致命 |
| 同比环比 | 如涉及历史对比，数据是否准确 | 🟡 警告 |

### 3.2 数据呈现规范

- 必须有表格或图表呈现核心数据，不能只有文字描述
- 数据单位统一（万元/元、客户数/订单数）
- 数据来源标注（CRM系统/财务报表/手工统计）

---

## 四、根因分析深度审核（20%）

### 4.1 分析框架检查

**必须使用的分析工具**（至少用一种）：

| 分析工具 | 适用情况 | 检查要点 |
|----------|----------|----------|
| **5Why分析法** | 单一关键问题反复出现 | 是否连续追问5层，找到根因 |
| **鱼骨图/石川图** | 多因素交织导致业绩落后 | 是否覆盖人、机、料、法、环、测六维度 |
| **对比分析法** | 自己与优秀同事的差距 | 是否控制变量对比（同区域/同客群/同时段） |
| **漏斗分析法** | 转化环节出问题 | 是否拆解获客→触达→转化→复购各环节转化率 |

### 4.2 根因质量评估

| 根因类型 | 示例 | 审核判定 |
|----------|------|----------|
| ✅ 高质量根因 | "我自己的客户分层能力不足，导致80%时间花在低价值客户上" | 具体、可验证、指向自身能力 |
| ⚠️ 中等根因 | "市场竞争加剧，竞品降价20%抢了3个大客户" | 有数据支撑，但主要是外部因素 |
| ❌ 低质量根因 | "市场环境不好""客户预算缩减""公司产品竞争力不足" | 空洞、无法验证、推卸责任 |

**审核规则**：
- 根因中「主观/自身原因」占比 ≥ 40%，否则扣10分
- 每条根因必须有数据或事实案例支撑，纯主观感受扣5分/条

---

## 五、下一步计划/改进措施审核（20%）

### 5.1 SMART 原则检查

每条下一步计划/改进措施必须符合 **SMART** 原则：

| 原则 | 检查项 | 反面示例 | 正面示例 |
|------|--------|----------|----------|
| **S具体** | 计划是否明确具体 | "加强客户沟通" | "每周对TOP20客户进行1次电话回访" |
| **M可衡量** | 是否有量化指标 | "提升客户满意度" | "NPS从30提升至45" |
| **A可达成** | 目标是否现实 | "下个月业绩翻3倍" | "下个月业绩恢复至目标的80%" |
| **R相关** | 是否既响应根因、又有前瞻部署 | 根因是"获客不足"，计划却是"提高客单价"（未响应根因） | 根因是"获客不足"，计划是"开拓2个新获客渠道+每周新增50条线索"（响应根因且有新部署） |
| **T有时限** | 是否有明确 deadline | "尽快完成" | "6月30日前完成客户分层盘点" |

### 5.2 下一步计划与根因对应矩阵

要求述职材料中呈现以下对应关系（可用表格）。**注意**：计划列不仅要写「补救措施」，更要写「下一阶段怎么打」。

```
┌─────────────┬──────────────────────────┬─────────────────┬──────────┐
│ 根因        │ 下一步计划/改进措施       │ 验收标准        │ 时间节点 │
├─────────────┼──────────────────────────┼─────────────────┼──────────┤
│ 根因1: ...  │ 计划1: ...（响应+前瞻）   │ 标准1: ...      │ 6月15日  │
│ 根因2: ...  │ 计划2: ...（响应+前瞻）   │ 标准2: ...      │ 6月30日  │
└─────────────┴──────────────────────────┴─────────────────┴──────────┘
```

**审核规则**：
- 每条根因必须有 ≥1 条对应计划，否则扣5分/条
- 计划中只有「补救」没有「前瞻部署」的，扣3分/条
- 计划中没有明确时间节点的，扣3分/条
- 计划中没有明确责任人的（默认述职人本人也需标注），扣2分/条

## 六、SP战略关联度审核（10%）

### 6.1 关联度检查框架

业绩落后的下一步计划不能只看短期补救，必须与公司/部门的 **SP（Strategic Planning，战略规划）** 对齐，避免「局部赢了、全局输了」的情况。

| 检查层级 | 审核要点 | 反面示例 | 正面示例 |
|----------|----------|----------|----------|
| **公司SP对齐** | 改进方向是否与公司整体战略一致 | "为了冲业绩，降低客户准入门槛大量签单"（与公司质量优先战略冲突） | "聚焦高价值客群，宁可减少客户数也要提升单客产出"（与公司提质增效战略一致） |
| **部门SP对齐** | 计划是否支撑部门年度关键战役/重点项目 | "我自己多跑跑客户"（与部门「数字化获客」战略无关） | "积极试用公司新推出的智能营销工具，提升获客效率"（直接支撑部门数字化战略） |
| **客户结构对齐** | 客户分层/取舍是否符合SP中的客群定位 | "继续维护所有老客户，不分优先级"（与SP中「聚焦战略客户」矛盾） | "按SP重新盘点客户，把80%精力投入A类战略客户" |
| **能力建设对齐** | 个人能力短板是否通过培训/轮岗补齐，而非单纯加班 | "下个月每天加班到10点"（不可持续，未解决能力问题） | "申请参加公司组织的解决方案销售培训，补足方案能力不足" |
| **SP重要机会点提及** | SP中明确列举的机会点/新方向，述职材料中是否有响应 | SP提到"下沉市场是新增长点"，但述职只字未提（错失机会） | SP提到"下沉市场是新增长点"，述职计划明确写"Q3启动下沉市场试点，覆盖3个新城市"（主动把握机会） |

### 6.2 关联度判定标准

| 等级 | 判定标准 | 得分 |
|------|----------|------|
| **强关联** | 下一步计划明确引用SP关键方向/重点项目，**且SP重要机会点有提及和响应**，逻辑闭环 | 8-10分 |
| **中关联** | 计划方向与SP不冲突，提及了SP方向，但**SP重要机会点未充分响应** | 4-7分 |
| **弱关联** | 计划与SP方向存在潜在冲突，或完全未考虑SP | 0-3分 |

### 6.3 审核规则（含后台自动比对）

**系统自动比对逻辑**：
1. 系统从提交的「体系SP战略规划KMS链接」中自动抓取文档内容
2. 通过NLP提取SP中的「重要机会点」「新增长方向」「战略重点」等关键词/短语
3. 与述职材料全文进行语义比对，识别述职人是否提及这些机会点
4. 比对结果自动生成「SP机会点覆盖度」指标（提及数/总机会点数）

**人工/系统审核规则**：
- **SP机会点覆盖度 < 50%**（即超过一半的机会点未在述职中提及），扣 **4分**
- **SP机会点覆盖度 = 0%**（完全未提及任何机会点），扣 **7分**
- 下一步计划与SP方向存在明显冲突的（如SP要求聚焦高端客户，计划却是低价抢量），本维度直接扣至 **0分**
- 主动引用SP原文或部门年度关键战役，并说明自身计划如何支撑，加 **2分**

## 七、态度与反思审核（5%）

### 7.1 态度识别关键词

| 正向信号（加分） | 负向信号（扣分） |
|-----------------|-----------------|
| "我复盘后发现..." | "主要是市场/公司/政策的问题..." |
| "我在XX环节确实做得不够..." | "我已经尽力了..." |
| "我计划从XX方面改进，下一阶段重点部署..." | "如果公司能给我XX资源..." |
| "我对比了优秀同事的做法，差距在..." | "同事运气好/客户质量好..." |

### 7.2 审核规则

- 材料中「推卸责任类表述」≥3处，本维度直接扣至0分
- 有具体自我反思案例（而非空泛表态），加3分
- 有向优秀同事学习的具体对标分析，加3分

## 八、审核报告改进建议生成规则

> **原则**：审核报告不仅是「挑错」，更要「教怎么改」。每条问题诊断必须附带一条可操作的改进建议，让述职人拿到报告后知道下一步该做什么。

### 改进建议生成规范

| 规范 | 说明 |
|------|------|
| **具体性** | 建议不能只说"请完善"，要给出具体动作（如"使用5Why工具，连续追问5层直到找到根因"） |
| **可执行性** | 建议要在述职人能力范围内可执行，避免建议本身成为新难题 |
| **范例参考** | 建议中尽可能附带一个正面示例，降低理解成本 |
| **分级对应** | 致命问题 → 必须修改的明确指令；警告问题 → 具体修改方向；建议问题 → 优化提升思路 |

## 九、常见问题诊断与改进建议对照表（供智能审核直接引用）

> **使用方式**：智能审核系统识别到以下问题时，直接调用对应的「改进建议」文案，写入审核报告。

### 🔴 致命问题（一票否决，直接打回）

| 序号 | 问题诊断 | 系统生成的改进建议 |
|------|----------|-------------------|
| 1 | **业绩数据造假或与系统不符** | 请核实业绩数据来源，对照CRM系统/财务报表修正目标值和实际值，确保口径一致。修正后重新提交。 |
| 2 | **未做根因分析，或根因全是外部因素** | 请使用5Why分析法对核心问题进行追问（至少到第3层），确保主观/自身原因占比≥40%。避免只写"市场不好""竞品降价"等外部因素。 |
| 3 | **下一步计划与根因完全不对应** | 请建立「根因-计划对应矩阵」，确保每条根因至少对应一条具体的下一步计划。计划要直接回应根因，而非另起炉灶。 |
| 4 | **缺少下一步计划/改进措施或资源需求模块** | 请补充完整的「下一步计划/改进措施」和「资源需求」章节。下一步计划需符合SMART原则，资源需求需明确具体（人力/预算/培训/政策支持）。 |

### 🟡 警告问题（必须修改）

| 序号 | 问题诊断 | 系统生成的改进建议 |
|------|----------|-------------------|
| 5 | **目标值/实际值口径不一致** | 请核对年初签订的业绩合同或OKR，确保述职中的目标值与正式文件一致。如有口径调整，请在材料中注明调整原因和依据。 |
| 6 | **根因分析缺少数据支撑，纯主观描述** | 请为每条根因补充具体数据或事实案例支撑。例如：不要写"客户流失严重"，要写"Q2流失客户12家，其中8家因交付延迟超过7天流失"。 |
| 7 | **下一步计划缺少时间节点** | 请为每条下一步计划设定明确的deadline和验收标准。避免使用"尽快""努力"等模糊词，改用"6月30日前完成""每周新增50条线索"等可量化表述。 |
| 8 | **数据呈现只有文字没有表格/图表** | 请将核心数据（目标/实际/达成率、差距拆解、过程指标等）用表格或图表呈现，提升可读性和可信度。 |
| 9 | **下一步计划未考虑与SP战略的对齐，存在局部优化风险** | 请查阅本体系SP战略规划（对照表已提交KMS链接），检查计划方向是否与SP一致。如SP要求"聚焦高价值客群"，计划不应是"低价抢量覆盖更多客户"。 |
