# 通用议题材料审核原则

> **适用场景**：经营分析会、月度例会、季度复盘会等会议的上会材料（以本部会为例）
> **材料形式**：KMS链接（飞书文档/企业Wiki/PPT等）
> **审核目标**：确保上会材料「经得起问、对得上数、说得清逻辑」，不是「通报情况」而是「请求决策」

---

## 一、审核框架总览

| 审核维度 | 权重 | 核心拷问 |
|----------|------|---------|
| **目标-解决方案对齐度** | 30% | 为什么要上会？方案对不对症？ |
| **决策支撑度** | 30% | 数据够不够硬？选项够不够全？ |
| **行动具体化（SMART）** | 25% | 谁来干？怎么衡量？什么时候干完？ |
| **材料规范度** | 15% | 结构顺不顺？篇幅是否可控？ |

**通过标准**：总分 ≥ 80分，且未触碰任何红线

**红线（触碰即总分封顶60分）**：
- 零决策请求 → 材料只是「通报情况」，没有明确的上会目的
- 零数据支撑 → 关键论点没有任何数据/案例/对比
- 零责任主体 → 行动计划中没有明确的责任人
- 零时间计划 → 行动计划中没有明确的deadline

**问题分级**：
- 🔴 致命（P0）：触碰红线、数据造假、方案与问题完全不对症
- 🟡 警告（P1）：单方案无备选、缺风险评估、数据计算错误、材料过长
- 🟢 建议（P2）：可增加历史对比、可增加外部对标、可增加可视化

---

## 二、4大审核维度速查

### 2.1 目标-解决方案对齐度（30%）

| 检查项 | 合格标准 | 常见问题 |
|--------|---------|---------|
| 问题背景 | 清晰的背景和触发因素 | 只有现象没有根因 |
| 问题范围 | 边界明确 | 什么都想解决 = 什么都解决不了 |
| 影响量化 | 有具体的业务影响数据 | "影响较大"没有数字 |
| 方案对症 | 方案直接回应根因 | 根因是获客不足，方案是提高客单价 |
| 决策请求 | 明确写出需要决策什么 | 材料只是汇报，没有决策需求 |

### 2.2 决策支撑度（30%）

| 检查项 | 合格标准 | 常见问题 |
|--------|---------|---------|
| 数据来源 | 可信、可追溯 | 没有数据来源标注 |
| 数据相关性 | 直接支撑决策 | 堆砌无关数据 |
| 数据准确性 | 计算正确、逻辑自洽 | 数学错误、前后矛盾 |
| 备选方案 | ≥2个可行方案 | 只有一个方案 |
| 权衡分析 | 每个方案的优缺点 | 没有对比矩阵 |
| 风险评估 | 识别风险及应对 | "没有风险" |

### 2.3 行动具体化（25%）

| SMART | 合格标准 | 反面示例 | 正面示例 |
|-------|---------|---------|---------|
| S具体 | 行动明确 | "加强客户沟通" | "每周对TOP20客户进行1次电话回访" |
| M可衡量 | 有量化指标 | "提升客户满意度" | "NPS从30提升至45" |
| A可达成 | 目标现实 | "下个月业绩翻3倍" | "下个月业绩恢复至目标的80%" |
| R相关 | 回应决策 | 计划与决策无关 | 计划直接解决决策涉及的问题 |
| T有时限 | 有deadline | "尽快完成" | "6月30日前完成客户分层盘点" |

### 2.4 材料规范度（15%）

| 检查项 | 合格标准 | 常见问题 |
|--------|---------|---------|
| 逻辑流 | 背景→问题→选项→建议→行动 | 顺序混乱 |
| 层级结构 | 清晰的标题层级 | 大段文字无标题 |
| 执行摘要 | 开头有1页摘要 | 读者必须翻完才能找到重点 |
| 可视化 | 表格/图表呈现复杂数据 | 10页文字0张表 |
| 篇幅控制 | ≤15页/30张PPT | 50页史诗没人看 |
| 空话比例 | 空话套话≤20% | 满篇"高度重视""战略意义" |

## 三、红线清单

| 红线 | 触发条件 | 后果 |
|------|---------|------|
| 零决策请求 | 材料没有明确的上会目的和决策需求 | 总分封顶60分，必须重写 |
| 零数据支撑 | 关键论点没有任何数据/案例/证据 | 总分封顶60分，必须补充 |
| 零责任主体 | 行动计划中没有明确的责任人 | 总分封顶60分，必须分配 |
| 零时间计划 | 行动计划中没有明确的deadline | 总分封顶60分，必须添加 |
