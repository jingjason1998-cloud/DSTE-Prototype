## 一、产品定位

### 1.1 模块定位

**战略洞察（Strategic Insights）** 是 DSTE **战略制定 (SP)** 阶段的核心入口之一，与**战略地图**、**战略专题管理**共同构成 SP 阶段的核心视图。

自 v1.2 起，本模块从产品层面独立为单独页面，职责边界如下：

- **战略洞察（Strategic Insights）**
  - 负责基于华为 DSTE "五看"模型（看趋势/看客户/看竞争/看自己/看机会）录入、管理市场洞察；
  - 核心动作：洞察新增/编辑/删除、按五看类型过滤、将高影响洞察一键生成战略专题；
  - 不直接管理专题生命周期；当前版本不再通过 `linkedTopicId` 与战略专题建立引用，关联关系规划至下一阶段由独立模块承接。

- **战略专题管理（Strategic Topic Management）**（参见《战略专题管理-完整设计方案.md》）
  - 负责将洞察或手动输入转化为中长期战略专题，并跟踪其里程碑、进度、资源与闭环；
  - 核心动作：专题新建/查看/编辑/软删除、按年份分组筛选、甘特图视图、年度深化链；
  - 页面内部仅保留**专题列表**一个 Tab，已闭环专题的沉淀统一由独立「知识中心」承接。

v1.1 及之前，两者以 Tab 形式共处在同一个页面（`sp/insights-topics`）。v1.2 产品方案要求：**战略洞察**单独成页，**战略专题管理**单独成页；当前代码实现仍共用 `renderInsightsTopics()`，需后续按本方案进行页面拆分重构。

### 1.2 与相关模块的关系

```
战略制定 (SP)
├── 战略地图 (sp/strategy-map)          ← 战略全景与目标因果链
├── 战略洞察 (sp/insights)              ← 五看洞察录入、洞察 → 生成专题
└── 战略专题管理 (sp/strategy-topics)    ← 战略专题全生命周期管理
           ↓
战略解码 (BP) —— BEM / 年度经营计划    ← 战略专题输出可作为解码输入
           ↓
战略执行 (Execution) —— 业务专题管理    ← 非战区战略专题可分解为业务专题执行；战区专题独立闭环，不进入业务专题
```

| 模块         | 时间跨度   | 核心对象            | 与本模块关系         |
| ---------- | ------ | --------------- | -------------- |
| **战略地图**   | 3-5 年  | 战略目标、BSC 四维度    | 专题可关联战略目标（预留）  |
| **战略洞察**   | 持续     | 洞察（五看）          | 本方案核心模块，专题的输入源 |
| **战略专题管理** | 1-3 年  | 战略专题            | 洞察的下游载体        |
| **业务专题管理** | 季度-1 年 | 业务议题、ST/AT 会议议题 | 非战区战略专题可分解为业务专题；战区专题不关联业务专题 |
| **年度经营计划** | 1 年    | KPI 实例、组织绩效     | 专题目标可输入到年度经营计划 |

### 1.3 关键设计原则

- **洞察驱动专题**：每条高影响洞察可一键生成专题，避免洞察与执行脱节；
- **页面职责分离**：战略洞察聚焦"发现与输入"，战略专题管理聚焦"规划与闭环"，两个页面独立导航、独立 URL；
- **五看模型结构化**：所有洞察必须归属五看类型之一，便于分类浏览与盲区识别；
- **共享数据模型**：洞察与专题继续共用 `dste_insights_v1` / `dste_strategy_topics_v2`，但分别由两个页面读写各自关心的字段；
- **纯前端实现**：与当前系统一致，使用 localStorage 持久化，单页 HTML + 原生 JS。

## 三、数据模型

### 3.1 洞察（Insight）

```typescript
interface Insight {
  // === 主键 ===
  id: string;                    // 格式: ins_{timestamp}_{random}，safeInsightId 校验

  // === 五看分类 ===
  type: 'trend' | 'customer' | 'competitor' | 'self' | 'opportunity';

  // === 内容字段（XSS 防护必需）===
  title: string;                 // 洞察标题
  content: string;               // 洞察详细内容
  source: string;                // 来源，如 "Gartner 2026"

  // === 质量评估 ===
  confidence: number;            // 置信度 0-100
  impact: 'high' | 'medium' | 'low';  // 影响程度

  // === 关联（下一阶段）===
  // linkedTopicId: string | null;  // 已生成专题时回填；当前版本暂不实现，规划至下一阶段

  // === 元数据 ===
  createdAt: string;             // YYYY-MM-DD
}
```

### 3.2 关键枚举映射

| 字段 | 存储值 | 展示值 | 样式 |
|------|--------|--------|------|
| `type` | `breakthrough` | 突破型 | 主色标签 |
| `type` | `incremental` | 增量型 | 信息色标签 |
| `type` | `maintenance` | 维持型 | 默认标签 |
| `status` | `insight` | 洞察阶段 | `status-default` |
| `status` | `planning` | 规划阶段 | `status-primary` |
| `status` | `execution` | 执行阶段 | `status-warning` |
| `status` | `closed` | 已闭环 | `status-success` |
| `impact` | `high` | 高影响 | `--danger` |
| `impact` | `medium` | 中影响 | `--warning` |
| `impact` | `low` | 低影响 | `--success` |

### 3.3 数据存储键名

```javascript
const STORAGE_KEY_INSIGHTS = 'dste_insights_v1';              // Insight[]
const STORAGE_KEY_TOPICS = 'dste_strategy_topics_v2';         // StrategyTopic[]（由战略专题管理页面维护，本模块只读/关联）
```

## 四、功能模块设计

### 4.1 页面职责

- 维护基于"五看"模型的洞察库；
- 支持洞察的新增、编辑、删除、按类型过滤；
- 支持将高影响洞察一键生成战略专题；
- 不直接管理专题的里程碑、进度与闭环状态，这些信息在**战略专题管理页面**维护。

### 4.2 五看模型卡片

页面顶部展示 5 张卡片，每张对应一个洞察类型：

| 类型 | 标签 | 图标 | 颜色变量 | 描述 |
|------|------|------|---------|------|
| `trend` | 看趋势 | 📈 | `--primary` | 宏观环境、行业趋势 |
| `customer` | 看客户 | 👥 | `--success` | 客户需求、痛点、价值 |
| `competitor` | 看竞争 | ⚔️ | `--warning` | 竞争对手战略、能力 |
| `self` | 看自己 | 🔍 | `--dim-learning` | 内部能力、资源、差距 |
| `opportunity` | 看机会 | 💡 | `--info` | 市场机会、战略机会点 |

**交互**：
- 点击卡片：洞察列表仅显示该类型的洞察；
- 再次点击或点击"全部洞察"标题：取消过滤；
- 卡片右上角显示当前类型洞察数量。

### 4.3 洞察列表

每条洞察展示：影响标签、置信度、创建日期、标题、内容、来源。

操作按钮：
- **生成专题**（未关联专题时显示）→ 打开预填充的战略专题创建弹窗；
- **编辑** → 打开洞察编辑弹窗；
- **删除** → 确认弹窗 → 物理删除。

### 4.4 洞察编辑弹窗

字段与校验：

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|---------|
| 五看类型 | select | 是 | 枚举 5 项 |
| 标题 | text | 是 | 长度 1-100 |
| 内容 | textarea | 是 | 长度 1-2000 |
| 来源 | text | 否 | 长度 0-100 |
| 置信度 | number range | 是 | 0-100 |
| 影响程度 | radio | 是 | high/medium/low |

保存时：
- 新增：`id = generateInsightId()`，`createdAt = today`；
- 编辑：保留 `id`、`createdAt`，更新其余字段。

### 4.5 删除洞察

洞察为轻量记录，采用**物理删除**。当前版本洞察与专题不再建立关联，删除洞察无需清理专题数据。

### 4.6 生成专题功能

从洞察生成专题的预填充映射：

| 专题字段 | 来源 | 默认值 |
|---------|------|--------|
| `name` | 洞察 `title` | — |
| `summary` | 洞察 `content` | — |
| `researchObjectives` | 洞察 `content` | — |
| `status` | — | `'insight'` |
| `year` | — | 当前年份 |
| `type` | — | `'breakthrough'` |
| `deliverables` | — | `[]` |
| `kmsUrl` | — | `''`（用户可后续补充）|
| `code` | — | 按专题年份自动生成 |
| `sortOrder` | — | 当前年份最大排序号 + 1 |

保存流程：
1. 用户在弹窗中可修改预填充内容；
2. 保存新专题到 `dste_strategy_topics_v2`；
3. 关闭弹窗后，用户可切换到**战略专题管理页面**继续维护该专题。

> **注**：当前版本洞察与专题不再建立双向关联；专题不再预填充 `insights` 数组，洞察也不再回写 `linkedTopicId`。关联关系规划至下一阶段由独立模块承接。

### 4.7 页面清单

|----------|------|------|
| `sp/insights` | 主页面 | 五看洞察库、生成专题入口 |

### 5.1 关键数据流说明

1. **洞察 → 专题（单向）**：在战略洞察页面生成专题时，新专题 `name` / `summary` / `researchObjectives` 由洞察预填充；当前版本不再回写 `linkedTopicId`，专题也不再包含 `insights` 数组；

## 六、接口设计

### 6.1 数据存储键名

### 6.2 核心函数接口

```javascript
// === 共享数据层 ===
function loadInsights(): Insight[]
function saveInsights(insights: Insight[]): void
function generateInsightId(): string       // ins_时间戳_随机数

// === 战略洞察页面 ===

function executeDeleteInsight(insightId: string): void
function filterInsightsByType(type: string): void
function generateTopicFromInsight(insightId: string): void   // 打开专题创建弹窗（当前版本不再预填充 insights 数组、不回写 linkedTopicId）

// === 工具函数 ===
```

### 7.1 防御矩阵

| 防线 | 措施 | 说明 | 优先级 |
|------|------|------|--------|
| **XSS 防护** | 所有用户输入字段使用 `escapeHtml()` 后再插入 DOM | 包括洞察标题/内容/来源 | P0 |
| **ID 校验** | `safeInsightId()` 白名单校验 | 防止非法 ID 注入 | P0 |
| **数据拷贝** | `loadInsights()` 返回浅拷贝 | 防止外部直接修改缓存 | P1 |
| **存储保护** | localStorage 读写加 try-catch | 损坏时返回默认值 | P1 |

### 7.2 安全函数实现

```javascript
/**
 * HTML 转义
 */
function escapeHtml(text) {
  if (text == null) return '';
  const div = document.createElement('div');
  div.textContent = String(text);
  return div.innerHTML;
}

/**
 * 洞察 ID 白名单校验
 */
function safeInsightId(id) {
  return typeof id === 'string' && /^ins_\d+_\d+$/.test(id);
}

/**
 * 安全加载洞察数据
 */
function safeLoadInsights() {
  try {
    const raw = DSTE.Storage.getString(STORAGE_KEY_INSIGHTS);
    if (!raw) return [];
    const insights = JSON.parse(raw);
    if (!Array.isArray(insights)) return [];
    return insights.filter(i => safeInsightId(i.id));
  } catch (e) {
    console.error('[SafeLoad] 洞察数据损坏:', e);
    return [];
  }
}
```
