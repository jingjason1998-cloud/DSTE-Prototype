## 二、产品定位与边界

### 2.1 模块定位

「指标关联」是 OMP 重点工作管理的**目标对齐能力**，回答“这项工作支撑哪些 KPI”的问题。它连接：

- **上游**：KPI 管理（指标库 + KPI 实例）
- **下游**：重点工作台账、里程碑、进度汇报
- **横向**：经营分析会（会议决议可生成新的改进工作并关联 KPI）

关联关系以**重点工作为入口维护**，在重点工作列表/详情展示；本次不做 KPI 侧的反向展示。

### 2.2 核心价值

1. **目标可见**：每项重点工作都能明确看到支撑的核心 KPI，避免工作与目标脱节。
2. **来源一致**：年度计划中的关联关系发布到 OMP 后保持同步，减少重复维护。

### 2.3 功能边界

| 范围 | 说明 |
|------|------|
| **做** | 为 OMP 重点工作（本地 + 年度计划派生）关联本周期内的 KPI 实例；区分主指标与相关指标；在重点工作列表/详情展示 |
| **不做** | KPI 侧反向查看关联重点工作；替代 KPI 目标值维护；替代指标库管理；不支持跨周期关联；不支持直接关联指标库（必须先创建 KPI 实例） |
| **限制** | 年度计划派生任务的关联关系由年度计划侧生成，OMP 侧可查看、可补充相关指标，但不可移除源头已关联的主指标 |

## 三、现状与问题

### 3.1 当前实现

- 重点工作通过 `task.relatedKpiIds`（字符串数组）存储关联关系。
- 数组中保存的是 **KPI 实例 ID**（`kpiInstances.id`）。
- 编辑表单已正确按 KPI 实例维护关联。

### 3.2 当前问题

| 问题 | 影响 | 示例 |
|------|------|------|
| **展示逻辑错误** | 列表页、详情页把 `relatedKpiIds` 误当成 `indicatorId` 查指标库，导致显示「未知指标」 | `垂直细分客户运营 关联: 未知指标` |
| **关联语义单一** | 所有关联 KPI 平铺展示，无法区分“主指标”与“相关指标” | 用户无法判断哪项 KPI 是核心目标 |
| **缺少业务约束** | 可跨周期、跨部门任意关联，容易产生脏数据 | 2026 任务关联到 2025 KPI 实例 |
| **年度计划侧未暴露编辑** | 年度计划重点工作也支持 `relatedKpiIds`，但编辑弹窗无此字段 | 源头数据无法维护 |

## 四、关联模型设计

### 4.1 关联对象

**重点工作关联的是 KPI 实例，而不是指标库指标。**

原因：
- 同一指标在不同周期、不同部门会有不同的目标值与达成情况，只有 KPI 实例才能反映“这项工作对哪一期目标有贡献”。
- 指标库指标只定义口径，不具备周期和部门上下文。

### 4.2 关联角色

引入两级角色，解决语义单一问题：

| 角色 | 英文名 | 数量限制 | 含义 |
|------|--------|----------|------|
| 主指标 | `primary` | 0 或 1 | 重点工作的核心目标 KPI，优先在列表展示 |
| 相关指标 | `supporting` | 0..N | 与重点工作相关、需关注但不作为核心目标的 KPI |

规则：
- 一个任务最多只能有一个主指标。
- 若用户只关联了一个 KPI 且未指定角色，系统自动将其设为主指标。
- 若用户关联了多个 KPI 但未指定主指标，保存时提示“请选择一个主指标”。

### 4.3 关联约束

1. **同周期**：只能关联当前 `cycleId` 下的 KPI 实例。
2. **同部门（建议）**：优先推荐与任务 `dept` 相同的 KPI 实例；跨部门关联需给出警告提示，不强制禁止。
3. **去重**：同一个 KPI 实例不能同时以主指标和相关指标两种角色存在。
4. **派生任务保护**：年度计划派生任务的主指标由年度计划侧生成，OMP 侧不可删除或修改其角色；OMP 侧可额外补充相关指标。

### 4.4 关联变更规则

| 操作 | 年度计划源头任务 | 年度计划派生任务 | OMP 本地任务 |
|------|------------------|------------------|--------------|
| 添加主指标 | 年度计划页面支持 | 不可添加/修改主指标 | 支持 |
| 添加相关指标 | 年度计划页面支持 | 支持 | 支持 |
| 移除主指标 | 支持 | 不支持（源头控制） | 支持 |
| 移除相关指标 | 支持 | 仅可移除 OMP 侧补充的 | 支持 |
| 修改主指标为其他 KPI | 支持 | 不支持 | 支持 |

## 五、数据模型

### 5.1 任务实体扩展

将当前的 `relatedKpiIds: string[]` 升级为结构化的 `kpiAssociations` 数组：

```typescript
interface KpiAssociation {
  kpiInstanceId: string;   // KPI 实例 ID
  relationType: 'primary' | 'supporting';
  // 未来可扩展：weight（影响权重）、contributionType（直接/间接）等
}

interface Task {
  // ... 原有字段
  kpiAssociations: KpiAssociation[];
}
```

### 5.2 旧数据迁移

- 启动时执行一次性迁移：将 `relatedKpiIds` 中的每个字符串转为 `{ kpiInstanceId, relationType: 'supporting' }`。
- 若迁移后数组长度为 1，自动将其 `relationType` 设为 `'primary'`。
- 迁移完成后删除 `relatedKpiIds` 字段（兼容读取时可继续兜底）。

### 5.3 指标库删除校验修正

删除指标库指标前，需先找到所有引用该指标的 KPI 实例，再检查这些 KPI 实例是否被任何 `task.kpiAssociations` 引用。

```typescript
const indicatorId = 'ind_sales_d';
const kpiIds = kpiInstances
  .filter(k => k.indicatorId === indicatorId)
  .map(k => k.id);
const usedByTasks = tasks.filter(t =>
  (t.kpiAssociations || []).some(a => kpiIds.includes(a.kpiInstanceId))
);
```

## 六、业务规则

### 6.1 关联推荐

新建/编辑任务时，系统按以下优先级推荐 KPI 实例：

1. 同部门且同周期。
2. 同 BSC 维度（如任务类型为 `customer`，优先推荐 `bscDimension: 'customer'` 的 KPI）。
3. 负责人与 KPI 负责人一致。
4. 已在年度计划源头任务中关联过的 KPI。

### 6.2 列表展示规则

- 只显示**主指标**名称；若未设置主指标，显示第一个相关指标名称并标注“相关”。
- 鼠标悬停主指标名称时，Tooltip 展示全部关联 KPI 列表（主指标 + 相关指标）。
- 已删除或找不到的 KPI 实例显示「指标已失效」而不是「未知指标」。

### 6.3 年度计划发布同步

- 年度计划发布时，`ap_publishToExecution` 将源头任务的 `kpiAssociations` 原样复制到派生任务。
- 复制后，派生任务主指标的角色固定为 `primary`，OMP 侧不可改。

## 七、用户场景与流程

### 7.1 场景 1：在 OMP 新建重点工作并关联 KPI

```
用户点击「新建工作」
  → 填写名称、负责人、部门等必填项
  → 在「关联 KPI」区域搜索并选择主指标（单选）和相关指标（多选）
  → 已选指标以 Tag 形式展示在搜索框下方
  → 保存
  → 列表页显示主指标名称
```

### 7.2 场景 2：年度计划侧维护关联并发布

```
用户在年度经营计划页面编辑重点工作
  → 在编辑弹窗中新增「关联 KPI」字段
  → 选择主指标与相关指标
  → 发布到执行阶段
  → OMP 派生任务自动携带该关联关系
```

## 八、页面交互

### 8.1 重点工作列表页

- 移除当前错误的「关联: 未知指标」行内展示。
- 在工作名称右侧展示主指标 Tag：`📈 大客户销售额`。
- 悬停 Tag 显示完整关联列表。

### 8.2 重点工作详情页

在基本信息区域新增「关联 KPI」区块：

```
关联 KPI
├── 主指标：大客户销售额（销售部）
└── 相关指标：客户满意度（客服部）、线索转化率（市场部）
```

### 8.3 新建/编辑弹窗

将原来的「关联 KPI (多选)」改造为模糊搜索选择器：

```
关联 KPI
├── 主指标 [搜索输入框，占位文案：输入指标名称 / KPI 负责人 / 部门进行搜索]
│   └── 已选：大客户销售额（销售部）
└── 相关指标 [搜索输入框，占位文案同上]
    ├── 已选：客户满意度（客服部）
    └── 已选：线索转化率（市场部）
```

**搜索交互：**
- 用户点击搜索框后，不直接展示全部 KPI 列表，仅当输入关键词时触发搜索。
- 支持按指标名称、KPI 负责人、归属部门进行模糊匹配。
- 下拉结果最多展示 10 条，按推荐优先级排序（同部门 → 同 BSC 维度 → 其他）。
- 无匹配时显示「未找到相关 KPI，请尝试其他关键词」。
- 搜索结果中，已选为主指标的 KPI 不再出现在相关指标候选中。

**已选项展示：**
- 选择后，该 KPI 以 Tag/Chip 形式展示在对应搜索框下方。
- Tag 右侧显示「×」移除按钮；主指标移除后相关指标搜索框恢复该 KPI 的可选状态。
- 相关指标支持多个 Tag 横向排列，超出宽度换行。
- 已选项区域不保留搜索下拉列表，页面仅展示用户已选择的指标。

**辅助提示：**
- 跨部门选择时，Tag 下方出现黄色提示条：「该 KPI 归属部门与任务部门不同，请确认关联合理性。」
- 搜索输入框保留最近使用的 5 个 KPI 快捷入口（可选），减少重复输入。

### 8.4 年度经营计划重点工作编辑弹窗

- 在现有字段基础上新增「关联 KPI」区域，与 OMP 侧保持一致，采用相同的模糊搜索选择器。
- 选择后仅展示已选指标 Tag，不再展示全部候选 KPI。
- 保存时写入源头任务的 `kpiAssociations`。

## 九、API 与持久化

### 9.1 接口变化

当前 `tasks` 已走批量同步端点 `POST /api/omp/tasks`，本次改动无需新增接口。

请求体中的任务对象由：

```json
{
  "relatedKpiIds": ["kpi_sales_d"]
}
```

变为：

```json
{
  "kpiAssociations": [
    { "kpiInstanceId": "kpi_sales_d", "relationType": "primary" }
  ]
}
```

### 9.2 后端兼容性

- 后端若仍按 `relatedKpiIds` 消费，需同步升级；或在同步层做字段映射。
- 本地存储以新结构为准，旧结构仅用于一次性迁移。

### 9.3 同步策略

- 任务保存后仍由现有 OMP 批量同步队列处理。
- 在本地存储迁移完成后，立即触发一次任务批量同步，确保服务端数据结构一致。

## 十、验收标准

### 10.1 功能验收

- [ ] 新建/编辑任务时可分别通过模糊搜索选择主指标和相关指标。
- [ ] 搜索未输入关键词时不展示全部 KPI 列表。
- [ ] 选择后仅展示已选指标 Tag，可通过「×」移除已选项。
- [ ] 列表页正确显示主指标名称，不再出现「未知指标」。
- [ ] 详情页正确展示主指标和相关指标。
- [ ] 年度计划派生任务的主指标不可在 OMP 侧修改/删除。
- [ ] 年度经营计划编辑弹窗支持维护关联 KPI，交互与 OMP 侧一致。
- [ ] 删除指标库指标时，引用校验正确拦截被重点工作引用的指标。

### 10.2 数据验收

- [ ] 旧 `relatedKpiIds` 数据正确迁移为 `kpiAssociations`。
- [ ] 本地存储、服务端数据、页面展示三方一致。
- [ ] 跨周期 KPI 实例不可被选入关联。

### 10.3 非功能验收

- [ ] 关联 KPI 搜索响应时间在 300ms 以内（1000 个 KPI 实例内）。
- [ ] 搜索框输入防抖 150ms~300ms，避免频繁触发搜索。
- [ ] 无 CAS/登录流程外的新增依赖。
