# 经营分析会议 · 会议材料智能审核助手

> **版本**：产品设计草案 v0.1  
> **范围**：只做设计，不涉及技术实现  
> **目标**：为经营分析会议提供材料预审的完整产品体验设计

---

## 一、产品定位与价值主张

| 维度 | 定义 |
|------|------|
| **一句话定位** | 企业经营分析会议的「材料预审官」，在会议召开前自动完成材料质量把关 |
| **核心用户** | 汇报人（业务负责人）、会议组织者（财务BP/战略部） |
| **解决痛点** | 会议材料质量参差 → 会议效率低 → 决策质量差 |
| **价值主张** | 让每一份上会的材料，都经得起问、对得上数、说得清逻辑 |

---

## 二、用户角色与核心场景

### 2.1 角色地图

```
┌─────────────────────────┐     ┌─────────────────────────┐
│        汇报人           │     │      会议组织者         │
│    （业务负责人）        │     │   （财务BP / 战略部）   │
└───────────┬─────────────┘     └───────────┬─────────────┘
            │                               │
            ▼                               ▼
  · 写材料前看标准                  · 创建会议，配置审核规则
  · 写完一键自检                    · 批量审核各部门材料
  · 按反馈修改完善                  · 质量把关，不通过则打回
  · 提交最终版                      · 汇总状态，确认准入
```

### 2.2 核心场景故事

| 场景 | 用户 | 任务 | 期望结果 |
|------|------|------|----------|
| **自检** | 汇报人 | 材料写完后，想先自己过一遍 | 5分钟知道哪里要补、哪里要改 |
| **预审把关** | 会议组织者 | 收到多个部门材料，要统一审核质量 | 批量处理，自动标红问题，优先处理致命错误 |
| **会前确认** | 会议组织者 | 确认所有材料是否达到上会标准 | 一眼看清：几份已过、几份待改、几份未交 |

## 四、核心功能模块设计（用户视角）

### 模块A：会议规则配置

**设计目标**：让审核标准可配置，不同会议、不同部门可以有不同的严格程度。

| 功能点 | 交互设计 |
|--------|----------|
| 规则模板库 | 按会议场景绑定审核模板：选择场景后自动加载对应审核标准 |
| 自定义检查项 | 开关式配置：完整性/数据一致性/格式规范/分析深度，每项可设「必过」或「提醒」 |
| 指标口径绑定 | 关联企业指标字典，自动识别材料中出现的指标，提示口径定义 |

### 模块B：KMS链接提交与解析

**设计目标**：KMS链接提交过程无负担，解析结果可视化。

| 功能点 | 交互设计 |
|--------|----------|
| 智能识别区 | 粘贴KMS链接为主 + 支持从飞书/钉钉/企业微信一键导入（展示链接卡片） |
| 解析进度 | 链接卡片展示解析阶段：获取中 → 识别中 → 审核中 → 完成 |
| 内容预览 | 侧边栏展示材料目录树，主区域预览内容，类似PDF阅读器体验 |

### 模块C：智能审核报告

**设计目标**：问题分级清晰，修改路径最短。

| 功能点 | 交互设计 |
|--------|----------|
| 总览评分卡 | 顶部固定栏：总分（百分制）+ 四个维度雷达图 + 通过/不通过状态 |
| 问题导航 | 左侧问题面板，按「致命/警告/建议」三色标签分类，点击定位到具体页面/段落 |
| 内联批注 | 材料预览区直接叠加批注气泡，类似Google Docs批注，不遮挡原文 |
| 修改建议 | 每个问题附带「建议修改文案」，可一键复制或采纳替换 |
| 批量确认 | 对同一类问题（如多处字体不统一）可批量确认/忽略 |

### 模块D：批量审核视图

**设计目标**：会议组织者视角，横向对比所有材料。

| 功能点 | 交互设计 |
|--------|----------|
| 材料矩阵 | 表格视图：行=材料，列=审核维度，单元格=红绿灯状态 |
| 汇总结论 | 自动汇总所有材料的共性问题（如"3个部门都缺少环比分析"） |
| 穿透查看 | 点击任意单元格，弹窗查看该材料在该维度的详细问题 |

### 模块E：会议场景管理

**设计目标**：后台灵活定义会议场景，前端一键选择，自动匹配对应审核标准。

| 功能点 | 交互设计 |
|--------|----------|
| 场景配置（后台） | 新增/编辑/删除会议场景，绑定审核规则模板、必检指标、严格程度 |
| 场景列表（前端） | 创建会议或提交材料时，下拉选择场景，自动加载对应审核标准 |
| 场景预览 | 选择场景后展示该场景的审核要点摘要，让汇报人提前知晓重点 |

```
开始

  ▼

└──────┬──────┘
       │
       ▼
┌─────────────┐
│  加载审核标准 │ ← 自动匹配该场景对应的审核规则模板
└──────┬──────┘

└──────┬──────┘
       │
       ▼
┌─────────────┐
│  查看报告    │ ← 顶部总分 + 雷达图
```

## 七、视觉设计方向

### 7.1 设计关键词

**专业 · 清晰 · 可信 · 高效**

### 7.2 色彩系统建议

| 角色 | 色值建议 | 用途 |
|------|----------|------|
| 主色 | 深海蓝 `#1a3a5c` | 导航、主按钮、品牌认知 |
| 辅色 | 雾灰蓝 `#f0f4f8` | 背景、卡片底色、分隔 |
| 成功 | 松石绿 `#0d9f6f` | 已通过、正确、完成 |
| 警告 | 琥珀黄 `#f59e0b` | 警告、需注意 |
| 致命 | 砖红 `#dc2626` | 错误、不通过、阻断 |
| 文字主色 | 墨黑 `#1f2937` | 标题、正文 |
| 文字次色 | 灰 `#6b7280` | 辅助说明、时间戳 |

### 7.3 排版建议

- **数字字体**：等宽字体显示分数、数据（如 Roboto Mono），增强可信感
- **中文字体**：干净无衬线（如思源黑体/苹方），标题加粗、正文常规
- **信息密度**：中高密度，适合办公场景的高效扫视

### 7.4 视觉风格参考

- **Notion** 的卡片式信息组织
- **Figma** 的侧边栏导航结构
- **Google Docs** 的内联批注交互

## 八、设计原则

| 原则 | 说明 |
|------|------|
| **问题分级显化** | 用颜色+数字+标签，让用户0.5秒判断严重程度 |
| **定位零成本** | 任何批注都能一键定位到原文具体位置 |
| **修改路径最短** | 看到问题 → 查看建议 → 确认修改，三步内完成 |
| **批量不丢细节** | 矩阵视图看全局，单点穿透看细节 |
| **反馈即时可见** | 修改后重新审核，分数变化即时呈现，形成正向激励 |

## 十、用户旅程地图 (User Journey Map)

> **说明**：以下覆盖两个核心角色在「经营分析会议材料准备 → 审核 → 提交」全周期中的真实体验。

### 旅程一：汇报人 — 小陈（销售部负责人）

**背景**：月度经营分析会，需提交本月销售复盘及下月计划 PPT。小陈不是专业数据分析出身，材料主要依靠财务给的数据 + 自己写分析。

#### 旅程概览

| 阶段 | 行为 (Doing) | 想法 (Thinking) | 感受 (Feeling) | 触点 | 痛点 | 机会点 |
|------|-------------|-----------------|---------------|------|------|--------|
| **1. 接收通知** | 收到财务BP的会议通知邮件，附件有材料模板 | "又要开经营会了，这次得把华东区下滑讲清楚" | 😐 平静，轻微焦虑 | 邮件 / 飞书 | 要求模糊，不知道重点在哪 | 智能助手推送「本次会议重点关注清单」 |
| **2. 撰写材料** | 打开模板，粘贴数据，写分析页，做图表 | "这个数据口径对不对？环比是怎么算的？" | 😫 焦虑、疲惫 | PPT / Excel | 频繁切换工具核对数据，容易出错 | 助手提供「指标口径速查」浮窗 |

| **7. 重新审核** | 改完再跑一遍审核，分数从78 → 91 | "91分应该稳了！致命问题都清掉了" | 😊 自信、轻松 | 审核报告页 | 想确认修改是否真的生效 | 对比视图：修改前 vs 修改后，问题逐一打勾 |

### 旅程二：会议组织者 — 王姐（财务BP）

**背景**：月度经营分析会材料收口人，需统筹会议全流程：从创建会议、配置审核规则，到批量审核各部门材料、确认上会准入，确保材料质量过关后才能上会。

|------|-------------|-----------------|---------------|------|------|--------|

| **7. 修改确认** | 运营部重新提交，系统自动对比修改前后，致命问题已清零 | "修改到位了，这次应该没问题了" | ✅ 放心 | 修改对比视图 | 需要人工逐条确认是否真的改了 | 「修改对比」自动打勾已修复项，只留未修复 |
| **8. 确认收齐** | 所有部门状态变绿，导出质量汇总报告 | "这次材料质量比上个月好，上会效率应该高很多" | 😌 满意、成就感 | 导出按钮 / PDF报告 | — | 汇总报告自动生成，可直接附在会议通知中 |

### 旅程地图总结：跨角色关键洞察

| 洞察编号 | 发现 | 涉及角色 | 设计机会 |
|----------|------|----------|----------|
| **J1** | 汇报人写材料时最焦虑的是「不知道标准在哪」，写完后「不知道缺什么」 | 汇报人 | 在撰写阶段就提供「实时合规检查」侧边栏，边写边提示 |
| **J2** | 会议组织者80%的时间花在「等提交」和「发现低级错误」，真正有价值的深度分析反而没时间做 | 会议组织者 | 自动预审拦截低级错误，让组织者专注业务逻辑判断和会前统筹 |
| **J3** | 会议组织者真正的焦虑不是「材料没交」，而是「交了但质量不行，领导现场发飙」 | 组织者 | 建立「上会准入门槛」机制：未达分数自动拦截，不允许提交 |
| **J4** | 两个角色都需要「确定性」——汇报人要确定自己不会出丑，会议组织者要确定材料没问题、会议不会翻车 | 全员 | 审核报告设计为「确定性凭证」：评分 + 通过章 + 问题清零清单 |
| **J5** | 材料修改后的「重新审核」是汇报人建立信任的关键时刻，分数提升带来强烈正向反馈 | 汇报人 | 强化修改前后的对比可视化，鼓励迭代优化 |

### 11.2 审核助手在平台中的角色定位

| 维度 | 定位说明 |
|------|----------|
| **模块名称** | 材料提交与智能审核（或简称「材料预审」） |
| **在平台中的层级** | 核心支撑模块，是「会前准备」阶段的关键质量控制节点 |

### 11.5 整合后的用户旅程调整

将第十章的两条旅程置于平台整合语境下，关键变化点：

#### 汇报人旅程（平台整合版）

| 阶段 | 平台内动作 | 变化点 |
|------|-----------|--------|
| 接收通知 | 在平台「会议日历」中看到本月经营会，点击后自动弹出「审核规则+重点议题」 | 不再只是邮件通知，而是与会议系统直接联动 |
| 撰写材料 | 平台提供「指标口径速查」浮窗，直接对接营销线数据字典 | 不需要再切到财务系统查口径 |
| 一键自检 | 在「议题与材料中心」粘贴KMS链接提交，系统自动触发审核 | 不是独立工具，而是平台原生流程 |
| 提交材料 | 审核通过后，材料自动进入「会议执行中心」的会中材料库 | 不需要再通过邮件/飞书反复发送 |

#### 会议组织者旅程（平台整合版）

| 阶段 | 平台内动作 | 变化点 |
|------|-----------|--------|
| 选择场景 | 在「会议管理中心」创建会议时，下拉选择场景，自动加载对应审核标准 | 场景化配置，减少人工选择模板出错 |
| 发布规则 | 审核模板和必检议题随场景自动同步给所有汇报人 | 不需要单独发邮件强调要求 |
| 跟进进度 | 「议题与材料中心」看板直接展示：议题收集进度 + 材料提交进度 + 审核通过进度 | 一个看板看清三件事 |
| 批量审核 | 在「议题与材料中心」一键批量审核，结果直接反映在「材料提交看板」 | 与会议看板一体化，自动预审 |
| 反馈批注 | 批注自动同步到汇报人的「我的待办」，修改后状态自动更新 | 不需要人工追踪修改进度 |
| 确认收齐 | 所有材料变绿后，「会议执行中心」自动解锁，可生成会议材料包 | 审核与会议执行无缝衔接 |
| 会前打包 | 审核通过的材料自动打包，生成「会议材料包 + 质量摘要 + 议题时间分配表」 | 一键生成，不需要手动合并 |
| 会后复盘 | 「数据复盘中心」自动生成：材料质量 vs 会议效率关联分析 | 可以量化证明审核带来的效率提升 |

```
│   │   纪要和任务追踪决定「闭环效果」                       │ │
│   │                                                     │ │
│   │   最终沉淀到数据复盘中心：                            │ │
```
