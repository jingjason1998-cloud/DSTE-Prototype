## 一、产品定位

### 1.1 模块定位

**战略洞察**与**战略专题管理**是 DSTE **战略制定 (SP)** 阶段的两个并列入口，与**战略地图**共同构成 SP 阶段的核心视图。

自 v1.2 起，两个功能从产品层面拆分为**两个独立页面**，职责边界如下：

- **战略洞察（Strategic Insights）**
  - 负责基于华为 DSTE "五看"模型（看趋势/看客户/看竞争/看自己/看机会）录入、管理市场洞察；
  - 核心动作：洞察新增/编辑/删除、按五看类型过滤、将高影响洞察一键生成战略专题；
  - 不直接管理专题生命周期，只通过 `linkedTopicId` 与战略专题建立单向引用。

- **战略专题管理（Strategic Topic Management）**
  - 负责将洞察或手动输入转化为中长期战略专题，并跟踪其里程碑、进度、资源与闭环；
  - 核心动作：专题新建/查看/编辑/软删除、按年份分组筛选；
  - 页面内部仅保留**专题列表**一个 Tab，已闭环专题的沉淀统一由独立「知识中心」承接。

v1.1 及之前，两者以 Tab 形式共处在同一个页面（`sp/insights-topics`）。v1.2 产品方案要求：**战略洞察**单独成页，**战略专题管理**单独成页；当前代码实现仍共用 `renderInsightsTopics()`，需后续按本方案进行页面拆分重构。

### 1.2 与相关模块的关系

```
战略制定 (SP)
├── 战略地图 (sp/strategy-map)          ← 战略全景与目标因果链
├── 战略洞察 (sp/insights)              ← 五看洞察录入、洞察 → 生成专题
└── 战略专题管理 (sp/strategy-topics)    ← 战略专题全生命周期管理、知识沉淀
           ↓
战略解码 (BP) —— BEM / 年度经营计划    ← 战略专题输出可作为解码输入
           ↓
战略执行 (Execution) —— 业务专题管理    ← 非战区战略专题可分解为业务专题执行；战区专题独立闭环，不进入业务专题
```

| 模块         | 时间跨度   | 核心对象            | 与本模块关系         |
| ---------- | ------ | --------------- | -------------- |
| **战略地图**   | 3-5 年  | 战略目标、BSC 四维度    | 专题可关联战略目标（预留）  |
| **战略洞察**   | 持续     | 洞察（五看）          | 生成战略专题的输入源     |
| **战略专题管理** | 1-3 年  | 战略专题            | 本方案核心模块之一      |
| **业务专题管理** | 季度-1 年 | 业务议题、ST/AT 会议议题 | 非战区战略专题可分解为业务专题；战区专题不关联业务专题 |
| **年度经营计划** | 1 年    | KPI 实例、组织绩效     | 专题目标可输入到年度经营计划 |

### 1.3 关键设计原则

- **洞察驱动专题**：每条高影响洞察可一键生成专题，避免洞察与执行脱节；
- **页面职责分离**：战略洞察聚焦"发现与输入"，战略专题管理聚焦"规划与闭环"，两个页面独立导航、独立 URL；
- **战区专题独立闭环**：战区专题作为战略专题的特殊类型，用于沉淀战区三年规划等知识资产，不向下分解为业务专题，避免跨年度规划与当年执行混淆；
- **年份分组管理**：专题按自然年分组，支持跨年度战略对比；
- **闭环沉淀**：专题闭环后，其总结摘要同步到独立知识中心展示，成为可复用资产；
- **共享数据模型**：洞察与专题继续共用 `dste_insights_v1` / `dste_strategy_topics_v2`，但分别由两个页面读写各自关心的字段；
- **纯前端实现**：与当前系统一致，使用 localStorage 持久化，单页 HTML + 原生 JS。

## 三、数据模型

### 3.1 洞察（Insight）

```typescript
interface Insight {
  // === 主键 ===
  id: string;                    // 格式: ins_{timestamp}_{random}，safeInsightId 校验

  // === 五看分类 ===
  type: 'trend' | 'customer' | 'competitor' | 'self' | 'opportunity';

  // === 内容字段（XSS 防护必需）===
  title: string;                 // 洞察标题
  content: string;               // 洞察详细内容
  source: string;                // 来源，如 "Gartner 2026"

  // === 质量评估 ===
  confidence: number;            // 置信度 0-100
  impact: 'high' | 'medium' | 'low';  // 影响程度

  // === 关联 ===
  linkedTopicId: string | null;  // 已生成专题时回填

  // === 元数据 ===
  createdAt: string;             // YYYY-MM-DD
}
```

### 3.2 战略专题（StrategyTopic）

```typescript
interface StrategyTopic {
  // === 主键 ===
  id: string;                    // 格式: st_{timestamp}_{random}，safeTopicId 校验

  // === 基本信息（XSS 防护必需）===
  name: string;                  // 专题名称
  summary: string;               // 结项总结/背景说明
  researchObjectives: string;    // 专题研究目标

  // === 外部链接 ===
  kmsUrl: string;                // KMS 页面链接，如 https://kms.fineres.com/pages/...

  // === 分类 ===
  type: 'breakthrough' | 'incremental' | 'maintenance';  // 突破型 / 增量型 / 维持型
  dimension: '财务' | '客户' | '流程' | '学习成长';      // BSC 维度

  // === 交付件 ===
  deliverables: string[];         // 交付件清单，如 ['市场调研报告', '三年业务规划']

  // === 生命周期 ===
  status: 'insight' | 'planning' | 'execution' | 'closed';

  // === 时间与年份 ===
  year: number;                  // 专题归属年份，用于分组与筛选
  startDate: string;             // YYYY-MM-DD
  endDate: string;               // YYYY-MM-DD

  // === 负责人与成员 ===
  owner: string | PersonObject;  // 支持人员选择器
  members: (string | PersonObject)[]; // 专题成员，支持多人

  // === 进度 ===
  progress: number;              // 0-100，根据里程碑完成率自动计算

  // === 关联 ===
  insights: string[];            // 关联洞察 ID 列表

  // === 年度深化链 ===
  previousTopicId: string | null; // 上一年/上一阶段关联专题 ID
  nextTopicId: string | null;     // 下一年深化专题 ID

  // === 里程碑 ===
  milestones: Milestone[];

  // === 软删除 ===
  deleted: boolean;              // true 表示已删除，列表中过滤

  // === 元数据 ===
  createdAt: string;             // YYYY-MM-DD
  updatedAt: string;             // YYYY-MM-DD
}

interface Milestone {
  title: string;                 // 里程碑标题（XSS 防护必需）
  date: string;                  // YYYY-MM-DD
  completed: boolean;            // 是否已完成
}
```

### 3.3 知识沉淀（KnowledgeItem）— v1.1 实现说明

v1.1 不引入独立的 `KnowledgeItem` 编辑器，而是直接读取已闭环专题（`status === 'closed'`）的 `summary` 字段作为沉淀内容。未来如需独立方法论模板库，再启用本结构。

```typescript
interface KnowledgeItem {
  id: string;
  topicId: string;               // 关联专题
  type: 'report' | 'template' | 'methodology';
  title: string;
  content: string;
  createdAt: string;
}
```

**v1.1 实际实现**：
- 知识沉淀 Tab 从 `dste_strategy_topics_v2` 中过滤 `status === 'closed'` 的专题
- 按 `year` 分组，以卡片形式展示专题名称、战略维度、负责人、闭环日期、总结摘要
- 提供"查看详情"按钮跳转到专题详情弹窗
- 顶部统计卡片展示：累计闭环专题数、有总结报告数、覆盖年份数

### 3.4 关键枚举映射

| 字段 | 存储值 | 展示值 | 样式 |
|------|--------|--------|------|
| `type` | `breakthrough` | 突破型 | 主色标签 |
| `type` | `incremental` | 增量型 | 信息色标签 |
| `type` | `maintenance` | 维持型 | 默认标签 |
| `status` | `insight` | 洞察阶段 | `status-default` |
| `status` | `planning` | 规划阶段 | `status-primary` |
| `status` | `execution` | 执行阶段 | `status-warning` |

| `impact` | `high` | 高影响 | `--danger` |
| `impact` | `medium` | 中影响 | `--warning` |
| `impact` | `low` | 低影响 | `--success` |

### 3.5 数据存储键名

```javascript
const STORAGE_KEY_INSIGHTS = 'dste_insights_v1';              // Insight[]
const STORAGE_KEY_TOPICS = 'dste_strategy_topics_v2';         // StrategyTopic[]
const STORAGE_KEY_TOPICS_LEGACY = 'dste_strategy_topics_v1';  // 旧版数据，迁移后删除
const STORAGE_KEY_KNOWLEDGE = 'dste_knowledge_v1';            // KnowledgeItem[]（v1.1 未启用，直接读取 closed Topic.summary）
```

### 3.6 数据迁移

从旧版 `dste_strategy_topics_v1` 迁移到 v2 时，为没有 `year` 字段的专题补充年份：

```javascript
function migrateTopicsFromV1() {
  const legacy = DSTE.Storage.getString(STORAGE_KEY_TOPICS_LEGACY);
  if (!legacy) return false;
  const topics = JSON.parse(legacy);
  const migrated = topics.map(t => {
    const year = t.year || (t.startDate ? parseInt(t.startDate.split('-')[0], 10) : new Date().getFullYear());
    const item = {
      ...t,
      year,
      members: t.members || [],
      kmsUrl: t.kmsUrl || '',
      researchObjectives: t.researchObjectives || '',
      deliverables: t.deliverables || [],
      previousTopicId: t.previousTopicId || null,
      nextTopicId: t.nextTopicId || null
    };
    // v2 起取消 budget、actualCost 字段
    delete item.budget;
    delete item.actualCost;
    delete item.priority;
    return item;
  });
  DSTE.Storage.set(STORAGE_KEY_TOPICS, migrated);
  DSTE.Storage.remove(STORAGE_KEY_TOPICS_LEGACY);
  return true;
}
```

## 四、功能模块设计

自 v1.2 起，原 `sp/insights-topics` 混合页面拆分为两个独立页面：

- **战略洞察页面**：聚焦洞察输入与"洞察 → 专题"的转化；
- **战略专题管理页面**：聚焦战略专题全生命周期管理与知识沉淀。

以下按页面分别描述。

---

### 4.1 战略洞察页面

#### 4.1.1 页面职责

- 维护基于"五看"模型的洞察库；
- 支持洞察的新增、编辑、删除、按类型过滤；
- 支持将高影响洞察一键生成战略专题；
- 不直接管理专题的里程碑、进度与闭环状态，这些信息在**战略专题管理页面**维护。

#### 4.1.2 五看模型卡片

页面顶部展示 5 张卡片，每张对应一个洞察类型：

| 类型 | 标签 | 图标 | 颜色变量 | 描述 |
|------|------|------|---------|------|
| `trend` | 看趋势 | 📈 | `--primary` | 宏观环境、行业趋势 |
| `customer` | 看客户 | 👥 | `--success` | 客户需求、痛点、价值 |
| `competitor` | 看竞争 | ⚔️ | `--warning` | 竞争对手战略、能力 |
| `self` | 看自己 | 🔍 | `--dim-learning` | 内部能力、资源、差距 |
| `opportunity` | 看机会 | 💡 | `--info` | 市场机会、战略机会点 |

**交互**：
- 点击卡片：洞察列表仅显示该类型的洞察；
- 再次点击或点击"全部洞察"标题：取消过滤；
- 卡片右上角显示当前类型洞察数量。

#### 4.1.3 洞察列表

每条洞察展示：影响标签、置信度、创建日期、标题、内容、来源。已生成专题的洞察显示"已生成专题: {name}"标签。

操作按钮：
- **生成专题**（未关联专题时显示）→ 打开预填充的战略专题创建弹窗；
- **编辑** → 打开洞察编辑弹窗；
- **删除** → 确认弹窗 → 物理删除。

#### 4.1.4 洞察编辑弹窗

字段与校验：

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|---------|
| 五看类型 | select | 是 | 枚举 5 项 |
| 标题 | text | 是 | 长度 1-100 |
| 内容 | textarea | 是 | 长度 1-2000 |
| 来源 | text | 否 | 长度 0-100 |
| 置信度 | number range | 是 | 0-100 |
| 影响程度 | radio | 是 | high/medium/low |

保存时：
- 新增：`id = generateInsightId()`，`createdAt = today`，`linkedTopicId = null`；
- 编辑：保留 `id`、`createdAt`、`linkedTopicId`，更新其余字段。

#### 4.1.5 删除洞察

洞察为轻量记录，采用**物理删除**。若该洞察已关联专题，删除前提示：
> "该洞察已生成专题「{topic.name}」，删除后将解除关联，是否继续？"

删除后需清理对应专题 `insights` 数组中的该 ID。

#### 4.1.6 生成专题功能

从洞察生成专题的预填充映射：

| 专题字段 | 来源 | 默认值 |
|---------|------|--------|
| `name` | 洞察 `title` | — |
| `summary` | 洞察 `content` | — |
| `researchObjectives` | 洞察 `content` | — |
| `insights` | `[insight.id]` | — |
| `status` | — | `'insight'` |
| `year` | — | 当前年份 |
| `type` | — | `'breakthrough'` |
| `dimension` | — | `'流程'` |
| `deliverables` | — | `[]` |
| `kmsUrl` | — | `''`（用户可后续补充）|

保存流程：
1. 用户在弹窗中可修改预填充内容；
2. 保存新专题到 `dste_strategy_topics_v2`；
3. 回写 `insight.linkedTopicId = newTopic.id`；
4. 关闭弹窗后，洞察列表显示"已生成专题"标签；用户可切换到**战略专题管理页面**继续维护该专题。

### 4.2 战略专题管理页面

#### 4.2.1 页面职责

#### 4.2.2 专题列表 Tab

##### 年份选择器

- 下拉选项包含"全部年份"、所有专题已有年份、当前年、下一年；
- 默认选中"全部年份"，确保历史专题（如 2025 年已闭环专题）不会被默认隐藏；

##### 统计卡片

| 指标 | 计算方式 | 说明 |
|------|---------|------|
| 进行中 | `status !== 'closed' && status !== 'insight'` 的专题数 | 已进入规划/执行阶段 |

| 高风险 | `progress < 50 && status === 'execution'` 的专题数 | 需重点关注 |

- 提供「逐条添加」和「批量粘贴」两种模式；

关联洞察：

- 保存时生成 `id = generateTopicId()`，`createdAt = today`，`deleted = false`。

##### 删除专题

采用**软删除**：
1. 确认弹窗展示专题名称；
2. 执行删除时设置 `deleted = true`、`updatedAt = today`；
3. 保存后刷新页面，已删除专题不再显示；
4. 已删除专题关联的洞察，其 `linkedTopicId` 应同步清空。

#### 4.2.5 年度深化链

针对跨年持续深化的战略专题（如 2025 年专题在 2026 年继续深化），提供**年度深化链**功能，通过 `previousTopicId` / `nextTopicId` 建立链式关联，既保留按年分组的独立性，又支持追溯历年深化脉络。

##### 触发条件

专题详情页显示「年度深化」入口：
- 当专题 `status === 'execution' || status === 'closed'` 时可见；
- 若该专题已存在 `nextTopicId`，则入口改为「查看下一年深化专题」。

##### 创建下一年专题

点击「2026 年继续深化」（按钮文案动态取当前专题年份 + 1）后，打开预填充的专题创建弹窗：

| 字段 | 预填充来源 | 默认值/说明 |
|------|-----------|------------|
| `name` | 原专题 `name` | 原名称 + "（2026）"，用户可改 |
| `researchObjectives` | 原专题 `researchObjectives` | 继承原目标 |
| `members` | 原专题 `members` | 继承成员 |
| `kmsUrl` | 原专题 `kmsUrl` | 继承 KMS 链接 |
| `dimension` | 原专题 `dimension` | 继承 |
| `type` | 原专题 `type` | 继承 |
| `year` | — | 原专题年份 + 1 |
| `status` | — | `'planning'` |
| `previousTopicId` | 原专题 `id` | 建立反向关联 |

保存流程：
1. 用户确认/修改预填充内容；
2. 保存新专题到 `dste_strategy_topics_v2`；
3. 回写原专题 `nextTopicId = newTopic.id`；
4. 若原专题未闭环，提示用户是否同步将原专题标记为 `closed` 并填写阶段总结。

##### 列表页链式标识

- 存在 `nextTopicId` 的专题：行尾显示「↳ 2026 年已深化」标签，点击跳转；
- 存在 `previousTopicId` 的专题：行首显示「↰ 承继 2025」标签，点击跳转；
- 存在双向关联的专题：同时显示两个标签，并用同色系底色标识为链上专题。

##### 详情页关联展示

专题详情弹窗底部增加「年度深化链」区域：
- 上一阶段：显示 `previousTopicId` 对应专题名称，可点击跳转；
- 当前阶段：高亮当前专题；
- 下一阶段：显示 `nextTopicId` 对应专题名称，可点击跳转；
- 若链上有多个阶段，以横向时间轴形式展示。

##### 甘特图跨年链

- 同一年度深化链的专题在甘特图中使用同色系；
- 用虚线或箭头连接相邻年份专题的条形；
- hover 整条链时展示各年份专题名称与阶段。

##### 知识中心关联

- 链上每年的专题各自作为独立知识资产沉淀；
- 在知识资产详情中展示「关联年度专题」时间轴，可跳转到链上其他年份专题详情；
- 支持按链聚合搜索（未来 v1.1 可扩展）。

##### 数据安全

- 创建下一年专题前校验 `safeTopicId(previousTopicId)`；
- 删除专题时，同步清空关联专题的 `previousTopicId` 或 `nextTopicId`，防止断链指向已删除专题；
- 加载专题列表时，对孤儿 `previousTopicId` / `nextTopicId` 自动置空。

### 6.1 关键数据流说明

1. **洞察 → 专题**：在战略洞察页面生成专题时，新专题 `insights` 包含洞察 ID，洞察 `linkedTopicId` 回写为专题 ID；
2. **专题编辑 → 洞察**：在战略专题管理页面取消关联某洞察时，清空该洞察的 `linkedTopicId`；
3. **专题删除**：在战略专题管理页面软删除后，清空其关联洞察的 `linkedTopicId`；
4. **洞察删除**：在战略洞察页面物理删除洞察后，从所有专题 `insights` 数组中移除该 ID；
5. **专题闭环 → 知识中心**：已闭环战略专题的 `summary`、`year`、`dimension`、`owner` 等字段同步到独立知识中心，由知识中心统一沉淀与检索；
6. **年度深化链**：专题通过 `previousTopicId` / `nextTopicId` 建立链式关联，创建下一年深化专题时双向回写关联 ID；删除专题时同步清理链上关联；
7. **页面间跳转**：战略洞察页面可通过「生成专题」弹窗创建专题；创建完成后，用户可导航至战略专题管理页面继续维护；两个页面读取同一套 localStorage 数据，状态实时同步。

### 8.2 安全函数实现

```javascript
/**
 * HTML 转义
 */
function escapeHtml(text) {
  if (text == null) return '';
  const div = document.createElement('div');
  div.textContent = String(text);
  return div.innerHTML;
}

/**
 * 洞察 ID 白名单校验
 */
function safeInsightId(id) {
  return typeof id === 'string' && /^ins_\d+_\d+$/.test(id);
}

/**
 * 专题 ID 白名单校验
 */
function safeTopicId(id) {
  return typeof id === 'string' && /^st_\d+_\d+$/.test(id);
}

/**
 * 安全加载洞察数据
 */
function safeLoadInsights() {
  try {
    const raw = DSTE.Storage.getString(STORAGE_KEY_INSIGHTS);
    if (!raw) return [];
    const insights = JSON.parse(raw);
    if (!Array.isArray(insights)) return [];
    return insights.filter(i => safeInsightId(i.id));
  } catch (e) {
    console.error('[SafeLoad] 洞察数据损坏:', e);
    return [];
  }
}

/**
 * 安全加载专题数据
 */
function safeLoadTopics() {
  try {
    const raw = DSTE.Storage.getString(STORAGE_KEY_TOPICS);
    if (!raw) return [];
    const topics = JSON.parse(raw);
    if (!Array.isArray(topics)) return [];
    return topics.filter(t => safeTopicId(t.id));
  } catch (e) {
    console.error('[SafeLoad] 专题数据损坏:', e);
    return [];
  }
}
```

5. 在战略专题管理模块中补充「战区专题」定义：属于战略专题的特殊类型，用于沉淀战区三年规划等知识资产，不与业务专题建立关联；
6. **新增独立模块「知识中心」**：负责跨模块知识资产沉淀与检索；原战略专题管理页面内的「知识沉淀 Tab」移除，已闭环专题同步至知识中心；
7. **战略专题字段调整**：移除 `priority`（优先级）、`budget`（预算）、`actualCost`（实际成本）字段；新增 `members`（成员）、`kmsUrl`（KMS 链接）、`researchObjectives`（研究目标）、`deliverables`（交付件，支持批量修改）、`previousTopicId` / `nextTopicId`（年度深化链）；
8. **甘特图视图**：战略专题管理页面支持列表视图与甘特图视图切换，展示专题时间轴与里程碑；
9. **年度深化链**：支持跨年专题链式关联，详情页、列表页、甘特图、知识中心均展示链上关系。
