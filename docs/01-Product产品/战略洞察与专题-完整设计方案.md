## 一、产品定位

### 1.1 模块定位

**战略洞察与专题**是 DSTE **战略制定 (SP)** 阶段的核心入口之一，与**战略地图**并列。它将原"市场洞察"（占位页面）和"战略专题"（静态只读表格）合并为一个可运营的混合页面：

- **洞察视图**：基于华为 DSTE "五看"模型（看趋势/看客户/看竞争/看自己/看机会）记录和管理市场洞察；
- **专题列表**：将关键洞察转化为中长期战略专题，跟踪里程碑、进度与资源；
- **知识沉淀**：专题闭环后沉淀复盘报告与方法论，形成企业战略知识资产。

### 1.2 与相关模块的关系

```
战略制定 (SP)
├── 战略地图 (sp/strategy-map)          ← 战略全景与目标因果链
└── 战略洞察与专题 (sp/insights-topics)  ← 本模块：洞察输入 + 专题深化
           ↓
战略解码 (BP) —— BEM / 年度经营计划    ← 专题输出可作为解码输入
           ↓
战略执行 (Execution) —— 业务专题管理    ← 战略专题分解为业务专题执行
```

| 模块 | 时间跨度 | 核心对象 | 与本模块关系 |
|------|---------|---------|-------------|
| **战略地图** | 3-5 年 | 战略目标、BSC 四维度 | 专题可关联战略目标（预留） |
| **战略洞察与专题** | 1-3 年 | 洞察、战略专题 | 本模块 |
| **业务专题管理** | 季度-1 年 | 业务议题、ST/AT 会议议题 | 战略专题可向下分解为业务专题 |
| **年度经营计划** | 1 年 | KPI 实例、组织绩效 | 专题目标可输入到年度经营计划 |

### 1.3 关键设计原则

- **洞察驱动专题**：每条高影响洞察可一键生成专题，避免洞察与执行脱节；
- **年份分组管理**：专题按自然年分组，支持跨年度战略对比；
- **最小闭环**：v1.0 先完成洞察→专题→进度跟踪闭环，知识沉淀 v1.1 再扩展；
- **纯前端实现**：与当前系统一致，使用 localStorage 持久化，单页 HTML + 原生 JS。

## 三、数据模型

### 3.1 洞察（Insight）

```typescript
interface Insight {
  // === 主键 ===
  id: string;                    // 格式: ins_{timestamp}_{random}，safeInsightId 校验

  // === 五看分类 ===
  type: 'trend' | 'customer' | 'competitor' | 'self' | 'opportunity';

  // === 内容字段（XSS 防护必需）===
  title: string;                 // 洞察标题
  content: string;               // 洞察详细内容
  source: string;                // 来源，如 "Gartner 2026"

  // === 质量评估 ===
  confidence: number;            // 置信度 0-100
  impact: 'high' | 'medium' | 'low';  // 影响程度

  // === 关联 ===
  linkedTopicId: string | null;  // 已生成专题时回填

  // === 元数据 ===
  createdAt: string;             // YYYY-MM-DD
}
```

### 3.2 战略专题（StrategyTopic）

```typescript
interface StrategyTopic {
  // === 主键 ===
  id: string;                    // 格式: st_{timestamp}_{random}，safeTopicId 校验

  // === 基本信息（XSS 防护必需）===
  name: string;                  // 专题名称
  summary: string;               // 结项总结/背景说明

  // === 分类 ===
  type: 'breakthrough' | 'incremental' | 'maintenance';  // 突破型 / 增量型 / 维持型
  dimension: '财务' | '客户' | '流程' | '学习成长';      // BSC 维度
  priority: 'P0' | 'P1' | 'P2';

  // === 生命周期 ===
  status: 'insight' | 'planning' | 'execution' | 'closed';

  // === 时间与年份 ===
  year: number;                  // 专题归属年份，用于分组与筛选
  startDate: string;             // YYYY-MM-DD
  endDate: string;               // YYYY-MM-DD

  // === 负责人 ===
  owner: string | PersonObject;  // 支持人员选择器

  // === 进度与资源 ===
  progress: number;              // 0-100，根据里程碑完成率自动计算
  budget: number | string;       // 预算，支持数字或文本（如 "50万"）
  actualCost: number | string;   // 实际成本

  // === 关联 ===
  insights: string[];            // 关联洞察 ID 列表

  // === 里程碑 ===
  milestones: Milestone[];

  // === 软删除 ===
  deleted: boolean;              // true 表示已删除，列表中过滤

  // === 元数据 ===
  createdAt: string;             // YYYY-MM-DD
  updatedAt: string;             // YYYY-MM-DD
}

interface Milestone {
  title: string;                 // 里程碑标题（XSS 防护必需）
  date: string;                  // YYYY-MM-DD
  completed: boolean;            // 是否已完成
}
```

### 3.3 知识沉淀（KnowledgeItem）— v1.1 预留

```typescript
interface KnowledgeItem {
  id: string;
  topicId: string;               // 关联专题
  type: 'report' | 'template' | 'methodology';
  title: string;
  content: string;
  createdAt: string;
}
```

### 3.4 关键枚举映射

| 字段 | 存储值 | 展示值 | 样式 |
|------|--------|--------|------|
| `type` | `breakthrough` | 突破型 | 主色标签 |
| `type` | `incremental` | 增量型 | 信息色标签 |
| `type` | `maintenance` | 维持型 | 默认标签 |
| `status` | `insight` | 洞察阶段 | `status-default` |
| `status` | `planning` | 规划阶段 | `status-primary` |
| `status` | `execution` | 执行阶段 | `status-warning` |
| `status` | `closed` | 已闭环 | `status-success` |
| `priority` | `P0` | 🔴 P0 | 红色 |
| `priority` | `P1` | 🟡 P1 | 黄色 |
| `priority` | `P2` | 🟢 P2 | 绿色 |
| `impact` | `high` | 高影响 | `--danger` |
| `impact` | `medium` | 中影响 | `--warning` |
| `impact` | `low` | 低影响 | `--success` |

### 3.5 数据存储键名

```javascript
const STORAGE_KEY_INSIGHTS = 'dste_insights_v1';              // Insight[]
const STORAGE_KEY_TOPICS = 'dste_strategy_topics_v2';         // StrategyTopic[]
const STORAGE_KEY_TOPICS_LEGACY = 'dste_strategy_topics_v1';  // 旧版数据，迁移后删除
const STORAGE_KEY_KNOWLEDGE = 'dste_knowledge_v1';            // KnowledgeItem[]（v1.1启用）
```

### 3.6 数据迁移

从旧版 `dste_strategy_topics_v1` 迁移到 v2 时，为没有 `year` 字段的专题补充年份：

```javascript
function migrateTopicsFromV1() {
  const legacy = DSTE.Storage.getString(STORAGE_KEY_TOPICS_LEGACY);
  if (!legacy) return false;
  const topics = JSON.parse(legacy);
  const migrated = topics.map(t => {
    if (t.year) return t;
    const year = t.startDate ? parseInt(t.startDate.split('-')[0], 10) : new Date().getFullYear();
    return { ...t, year };
  });
  DSTE.Storage.set(STORAGE_KEY_TOPICS, migrated);
  DSTE.Storage.remove(STORAGE_KEY_TOPICS_LEGACY);
  return true;
}
```

## 四、功能模块设计

### 4.1 洞察视图 Tab

#### 4.1.1 五看模型卡片

页面顶部展示 5 张卡片，每张对应一个洞察类型：

| 类型 | 标签 | 图标 | 颜色变量 | 描述 |
|------|------|------|---------|------|
| `trend` | 看趋势 | 📈 | `--primary` | 宏观环境、行业趋势 |
| `customer` | 看客户 | 👥 | `--success` | 客户需求、痛点、价值 |
| `competitor` | 看竞争 | ⚔️ | `--warning` | 竞争对手战略、能力 |
| `self` | 看自己 | 🔍 | `--dim-learning` | 内部能力、资源、差距 |
| `opportunity` | 看机会 | 💡 | `--info` | 市场机会、战略机会点 |

**交互**：
- 点击卡片：洞察列表仅显示该类型的洞察；
- 再次点击或点击"全部洞察"标题：取消过滤；
- 卡片右上角显示当前类型洞察数量。

#### 4.1.2 洞察列表

每条洞察展示：影响标签、置信度、创建日期、标题、内容、来源。已生成专题的洞察显示"已生成专题: {name}"标签。

操作按钮：
- **生成专题**（未关联专题时显示）→ 打开预填充的专题创建弹窗；
- **编辑** → 打开洞察编辑弹窗；
- **删除** → 确认弹窗 → 物理删除。

#### 4.1.3 洞察编辑弹窗

字段与校验：

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|---------|
| 五看类型 | select | 是 | 枚举 5 项 |
| 标题 | text | 是 | 长度 1-100 |
| 内容 | textarea | 是 | 长度 1-2000 |
| 来源 | text | 否 | 长度 0-100 |
| 置信度 | number range | 是 | 0-100 |
| 影响程度 | radio | 是 | high/medium/low |

保存时：
- 新增：`id = generateInsightId()`，`createdAt = today`，`linkedTopicId = null`；
- 编辑：保留 `id`、`createdAt`、`linkedTopicId`，更新其余字段。

#### 4.1.4 删除洞察

洞察为轻量记录，采用**物理删除**。若该洞察已关联专题，删除前提示：
> "该洞察已生成专题「{topic.name}」，删除后将解除关联，是否继续？"

删除后需清理对应专题 `insights` 数组中的该 ID。

### 4.2 专题列表 Tab

#### 4.2.1 年份选择器

- 下拉选项从所有专题年份 + 当前年 + 下一年生成；
- 默认选中当前年份；
- 切换年份后重新渲染专题列表。

#### 4.2.2 统计卡片

基于当前选中年份计算：

| 指标 | 计算方式 | 说明 |
|------|---------|------|
| 进行中 | `status !== 'closed' && status !== 'insight'` 的专题数 | 已进入规划/执行阶段 |
| 已完成 | `status === 'closed'` 的专题数 | 已闭环 |
| 平均进度 | 所有专题 `progress` 的平均值 | 无专题时为 0 |
| 高风险 | `progress < 50 && status === 'execution'` 的专题数 | 需重点关注 |

#### 4.2.3 专题表格

按年份分组展示，每组一个卡片，包含：
- 年份标题 + 专题数量标签
- 进行中/已完成计数
- 数据表格：名称、战略维度、类型、优先级、负责人、阶段、进度、更新日期、操作

操作按钮：查看、编辑、删除。

#### 4.2.4 专题详情弹窗

展示字段：
- 标题区：专题名称、年份、类型、优先级
- 信息网格：负责人、战略维度、阶段、进度、预算、实际成本、开始日期、结束日期
- 里程碑列表：标题、日期、完成状态
- 关联洞察：来源 + 标题
- 操作按钮：关闭、编辑

#### 4.2.5 专题编辑弹窗

可编辑字段：

| 字段 | 控件 | 说明 |
|------|------|------|
| 专题名称 | text | 必填 |
| 负责人 | person-input | 支持人员选择器 |
| 战略维度 | text | 必填 |
| 类型 | select | breakthrough/incremental/maintenance |
| 优先级 | select | P0/P1/P2 |
| 阶段 | select | insight/planning/execution/closed |
| 开始日期 | date | — |
| 结束日期 | date | — |
| 预算 | text | 支持文本如 "50万" |
| 进度 | number | 0-100，默认由里程碑自动计算 |
| 年份 | text(disabled) | 编辑时不可修改 |

里程碑管理：
- 动态添加/删除里程碑行；
- 每行包含：完成 checkbox、标题输入、日期输入、删除按钮；
- 保存时过滤标题为空的行。

关联洞察：
- 多选下拉框，选项为当前所有未删除洞察；
- 保存时同步更新被移除洞察的 `linkedTopicId`（清空）。

#### 4.2.6 新建专题

复用编辑弹窗结构，差异点：
- 标题为"新建专题"；
- 所有字段为空/默认值；
- **年份可编辑**（下拉选择当前年/下一年/历史年份）；
- 保存时生成 `id = generateTopicId()`，`createdAt = today`，`deleted = false`。

#### 4.2.7 删除专题

采用**软删除**：
1. 确认弹窗展示专题名称；
2. 执行删除时设置 `deleted = true`、`updatedAt = today`；
3. 保存后刷新页面，已删除专题不再显示；
4. 已删除专题关联的洞察，其 `linkedTopicId` 应同步清空。

### 4.3 生成专题功能

从洞察生成专题的预填充映射：

| 专题字段 | 来源 | 默认值 |
|---------|------|--------|
| `name` | 洞察 `title` | — |
| `summary` | 洞察 `content` | — |
| `insights` | `[insight.id]` | — |
| `status` | — | `'insight'` |
| `year` | — | 当前年份 |
| `priority` | — | `'P1'` |
| `type` | — | `'breakthrough'` |
| `dimension` | — | `'流程'` |

保存流程：
1. 用户可修改预填充内容；
2. 保存新专题；
3. 回写 `insight.linkedTopicId = newTopic.id`；
4. 刷新洞察视图，该洞察显示"已生成专题"标签。

### 4.4 知识沉淀 Tab（v1.1 范围）

v1.0 仅展示占位页面，v1.1 实现最小可用：

```
知识沉淀区
├── 已完成专题报告
│   └── 读取所有 status === 'closed' 的专题
│   └── 展示：专题名称、闭环日期、摘要 summary
├── 方法论模板（占位）
└── 复盘报告（占位）
```

数据从 `StrategyTopic.summary` 直接读取，不新增独立编辑器。

### 6.1 关键数据流说明

1. **洞察 → 专题**：生成专题时，新专题 `insights` 包含洞察 ID，洞察 `linkedTopicId` 回写为专题 ID；
2. **专题编辑 → 洞察**：取消关联某洞察时，清空该洞察的 `linkedTopicId`；
3. **专题删除**：软删除后，清空其关联洞察的 `linkedTopicId`；
4. **洞察删除**：物理删除后，从所有专题 `insights` 数组中移除该 ID。

## 八、安全设计

### 8.1 防御矩阵

| 防线 | 措施 | 说明 | 优先级 |
|------|------|------|--------|
| **XSS 防护** | 所有用户输入字段使用 `escapeHtml()` 后再插入 DOM | 包括洞察标题/内容/来源、专题名称/摘要/里程碑标题 | P0 |
| **ID 校验** | `safeInsightId()` / `safeTopicId()` 白名单校验 | 防止非法 ID 注入 | P0 |
| **软删除** | 专题标记 `deleted: true`，列表过滤 | 防止误删导致数据丢失 | P0 |
| **人员输入安全** | 使用 `ensurePersonInput()` + `getPersonValue()` | 统一人员选择器取值，避免内联脚本 | P1 |
| **数据拷贝** | `loadInsights()` / `loadTopics()` 返回浅拷贝 | 防止外部直接修改缓存 | P1 |
| **存储保护** | localStorage 读写加 try-catch | 损坏时返回默认值 | P1 |

### 8.2 安全函数实现

```javascript
/**
 * HTML 转义
 */
function escapeHtml(text) {
  if (text == null) return '';
  const div = document.createElement('div');
  div.textContent = String(text);
  return div.innerHTML;
}

/**
 * 洞察 ID 白名单校验
 */
function safeInsightId(id) {
  return typeof id === 'string' && /^ins_\d+_\d+$/.test(id);
}

/**
 * 专题 ID 白名单校验
 */
function safeTopicId(id) {
  return typeof id === 'string' && /^st_\d+_\d+$/.test(id);
}

/**
 * 安全加载洞察数据
 */
function safeLoadInsights() {
  try {
    const raw = DSTE.Storage.getString(STORAGE_KEY_INSIGHTS);
    if (!raw) return [];
    const insights = JSON.parse(raw);
    if (!Array.isArray(insights)) return [];
    return insights.filter(i => safeInsightId(i.id));
  } catch (e) {
    console.error('[SafeLoad] 洞察数据损坏:', e);
    return [];
  }
}

/**
 * 安全加载专题数据
 */
function safeLoadTopics() {
  try {
    const raw = DSTE.Storage.getString(STORAGE_KEY_TOPICS);
    if (!raw) return [];
    const topics = JSON.parse(raw);
    if (!Array.isArray(topics)) return [];
    return topics.filter(t => safeTopicId(t.id));
  } catch (e) {
    console.error('[SafeLoad] 专题数据损坏:', e);
    return [];
  }
}
```

### 10.3 孤儿 linkedTopicId 处理

```javascript
function cleanupOrphanLinkedTopicIds(insights, topics) {
  const activeTopicIds = new Set(topics.filter(t => !t.deleted).map(t => t.id));
  return insights.map(i => ({
    ...i,
    linkedTopicId: activeTopicIds.has(i.linkedTopicId) ? i.linkedTopicId : null
  }));
}
```
