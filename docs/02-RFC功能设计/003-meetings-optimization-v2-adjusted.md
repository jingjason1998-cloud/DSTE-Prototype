

## 附录C：经营分析会页面价值主张设计方案

> 状态：`design` | 作者：AI Assistant | 日期：2026-05-21

### C.1 设计目标

1. **心智锚定**：用户进入页面第一眼理解经营分析会的核心价值——不是"会议管理"，而是"战略执行核心引擎"。
2. **三段式闭环**：建立"会前→会中→会后"的视觉锚点，与 PRD 6.2.2 "一报一会"机制形成呼应。
3. **效率导向**：价值主张区域作为页面"门面"，向下自然过渡到 KPI 效率四卡片，形成"理念→指标"的叙事流。

### C.2 页面位置

```
面包屑（驾驶舱 / 战略执行 / 经营分析会）
  ↓
页面标题 + [📅 日历视图] [+ 新建会议]
  ↓
【价值主张横幅】← 新增
  ↓
KPI 效率四卡片（材料及时率 / 决议及时率 / 闭环率 / 满意度）
  ↓
筛选栏
  ↓
主体区域（会议列表 + 标签页 | 统计面板）
```

### C.3 内容文案

#### C.3.1 主标题

> 🎯 战略执行核心引擎

引用 PRD 模块6定位："经营分析会不是简单的会议管理，而是战略执行的中央枢纽"。

#### C.3.2 三段式价值卡片

| 卡片 | 标题 | 文案 | 左侧竖线色 |
|------|------|------|-----------|
| **会前** | 📊 会前 · 数据驱动准备 | 自动汇聚战略执行数据，生成9章节结构化经营报告，AI识别异常与风险点 | `var(--primary)` 蓝 |
| **会中** | 💬 会中 · 结构化讨论 | 六步固定议程聚焦数据与事实，决策即时录入，计时提醒保障效率 | `var(--success)` 绿 |
| **会后** | ✅ 会后 · 行动项闭环 | 纪要24h内发布，行动项自动派发并跟踪，下次会议验证闭环 | `var(--warning)` 橙 |

> **文案来源**：PRD 6.2.2 "一报一会"机制 + 6.4 会前功能 + 6.6 会后闭环 + 6.9 关键规则。

### C.5 交互设计

- **纯展示型组件**，无点击/悬停交互。
- 不占用额外数据请求，所有文案为静态文本。
- 不影响现有筛选联动、标签页切换等功能。

### C.6 与现有页面的关系

| 现有元素 | 关系 |
|---------|------|
| 页面标题 | 价值主张在其下方，形成"标题→定位→指标"的层级 |
| KPI 四卡片 | 价值主张中的"会前/会中/会后"理念，自然对应到 KPI 的"材料及时率/决议及时率/闭环率/满意度"指标 |
| 筛选栏 | 无影响，筛选栏在 KPI 卡片下方，位置不变 |
| 会议列表 | 无影响 |

### C.7 实现要点

1. 插入位置：`renderMeetings()` 返回的 HTML 中，`page-header` 闭合标签之后，`kpi-grid` 之前。
2. 技术实现：纯 HTML + inline style，不引入新 CSS 类，保持与现有代码风格一致。
3. 文案国际化：当前为中文静态文本，后续如需多语言，可提取为变量。

### C.8 验收标准

- [ ] 页面加载后，价值主张横幅位于页面标题与 KPI 卡片之间
- [ ] 三列卡片在桌面端水平排列，移动端自动纵向堆叠
- [ ] 会前/会中/会后三张卡片左侧竖线颜色分别为蓝/绿/橙
- [ ] 不影响现有筛选联动、标签页切换、Playwright/pytest 测试通过
