

## 附录C：经营分析会页面价值主张设计方案

> 状态：`design` | 作者：AI Assistant | 日期：2026-05-21

### C.1 设计目标

1. **心智锚定**：用户进入页面第一眼理解经营分析会的核心价值——不是"会议管理"，而是"战略执行核心引擎"。
2. **三段式闭环**：建立"会前→会中→会后"的视觉锚点，与 PRD 6.2.2 "一报一会"机制形成呼应。
3. **效率导向**：价值主张区域作为页面"门面"，向下自然过渡到 KPI 效率四卡片，形成"理念→指标"的叙事流。

### C.2 页面位置

```
面包屑（驾驶舱 / 战略执行 / 经营分析会）
  ↓
页面标题 + [📅 日历视图] [+ 新建会议]
  ↓
【价值主张横幅】← 新增
  ↓
KPI 效率四卡片（材料及时率 / 决议及时率 / 闭环率 / 满意度）
  ↓
筛选栏
  ↓
主体区域（会议列表 + 标签页 | 统计面板）
  ↓
【会议详情弹窗】← 点击卡片触发
  ↓
【会议编辑弹窗】← 详情内点击编辑按钮触发
```

### C.3 内容文案

#### C.3.1 主标题

> 🎯 战略执行核心引擎

引用 PRD 模块6定位："经营分析会不是简单的会议管理，而是战略执行的中央枢纽"。

#### C.3.2 三段式价值卡片

| 卡片 | 标题 | 文案 | 左侧竖线色 |
|------|------|------|-----------|
| **会前** | 📊 会前 · 数据驱动准备 | 自动汇聚战略执行数据，生成9章节结构化经营报告，AI识别异常与风险点 | `var(--primary)` 蓝 |
| **会中** | 💬 会中 · 结构化讨论 | 六步固定议程聚焦数据与事实，决策即时录入，计时提醒保障效率 | `var(--success)` 绿 |
| **会后** | ✅ 会后 · 行动项闭环 | 纪要24h内发布，行动项自动派发并跟踪，下次会议验证闭环 | `var(--warning)` 橙 |

## 附录D：会议详情与编辑弹窗设计方案

> 状态：`implemented` | 作者：AI Assistant | 日期：2026-05-23

### D.1 设计目标

1. **信息分层**：点击卡片先进入只读详情，避免误触编辑；明确区分"查看"与"修改"两个意图。
2. **全局信息一览**：详情弹窗集中展示会议的全部关键信息，无需在列表中展开大量标签页。
3. **编辑入口明确**：详情弹窗底部放置"✏️ 编辑"按钮，用户主动选择进入编辑模式。

### D.2 交互流程

```
用户点击会议卡片
  ↓
打开【会议详情弹窗】（只读）
  ├─ 基本信息：名称、场景、层级、日期、地点、主持人、记录人、状态
  ├─ 议程列表：序号、议题类型、标题、时长、负责人
  ├─ 行动项：状态图标、内容、负责人、截止日期
  ├─ 决议：状态图标、内容、负责人、截止日期
```

### D.5 数据持久化机制

#### D.5.1 问题背景

原始实现中 `meetings` 为 `renderMeetings()` 内部局部变量，`saveMeeting()` 修改后调用 `navigate('exe/meetings')` 重新渲染，导致修改被重新初始化覆盖。

#### D.5.2 解决方案

```javascript
// renderMeetings() 内部
if (!window._meetingsData) {
  window._meetingsData = [ /* 初始化数据 */ ];
}
const meetings = window._meetingsData;
```

- 首次渲染：初始化 `window._meetingsData`
- 后续渲染：复用已有数据
- `saveMeeting()` 修改 `meetings`（即 `window._meetingsData` 的引用），修改自然持久化
- `apiSave('/api/meetings', meetings)` 异步同步到后端（如有 API）

### D.6 实现要点

1. **函数位置**：`openMeetingDetail` / `closeMeetingDetail` / `openEditorFromDetail` / `renderMeetingDetail` 均定义在 `renderMeetings()` 内部、`return` 之前，与编辑函数同级。
2. **弹窗 HTML**：嵌入 `renderMeetings()` 的 `return` 模板字符串中，位于编辑弹窗之前。
3. **z-index**：详情与编辑弹窗均为 2000，通过 `display` 切换互斥显示，不会同时出现。
4. **数据同步**：编辑弹窗的 `saveMeeting` 和 `deleteMeeting` 均调用 `apiSave` 后 `navigate('exe/meetings')` 刷新。
