# RFC-002: 经营分析会模块深化（v0.4.0）

> 状态：`implemented` | 作者：AI Assistant | 日期：2026-05-21 | 对应版本：v0.4.0

---

## 摘要

将经营分析会从基础会议列表升级为 DSTE 闭环的"战略执行中央枢纽"，新增会议纪要生成、会议效果评估、决议分类跟踪、待办提醒四大核心能力。

---

## 背景

v0.2.0 已交付经营分析会基础页面（会议列表 + 决议跟踪 + 简单统计），但存在以下缺口：

1. **会后无沉淀**：会议结束后无结构化纪要，决议散落在各处
2. **效果不可衡量**：开会质量无法评估，是否"白开"无从知晓
3. **决议管理粗放**：无分类、无逾期提醒、无闭环统计
4. **待办分散**：材料审核、纪要生成、决议到期等提醒散落在不同页面

---

## 目标

1. 会议纪要可生成、可查看、可追溯（AI 辅助生成初稿）
2. 会议效果可量化评估（四维度评分体系）
3. 决议跟踪精细化（分类标签 + 逾期标记 + 闭环率统计）
4. 待办事项集中提醒（材料审核 + 纪要生成 + 决议到期）

### 样式规范

- 保持现有 DSTE 设计系统：CSS 变量 + 内联样式混合
- 新增状态标签映射：`ready→warning`, `done→success`, `plan→default`
- 会议类型标签配色：月度(primary)、季度(success)、年度(warning)、专项(danger)
- 决议分类配色：财务(success)、客户(primary)、流程(warning)、学习成长(danger)
