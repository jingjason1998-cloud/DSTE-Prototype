## 三、框架搭建方案

### 3.1 占位页面标准化

所有占位页统一为以下结构：

```html
<div class="placeholder-page">
  <div class="placeholder-header">
    <div class="placeholder-icon">🔧</div>
    <div class="placeholder-title">模块名称</div>
    <div class="placeholder-desc">模块一句话说明。</div>
  </div>
  <div class="placeholder-roadmap">
    <div class="roadmap-step done">1. 已完成步骤</div>
    <div class="roadmap-step todo">2. 待实现步骤</div>
    ...
  </div>
</div>
```

> **关于全面预算管理**：按 PRD 定位，不拆独立一级导航，而是作为业财融合能力嵌入：
> - SP：战略地图/专题中标注战略投资预算；
> - BP：年度经营计划中增加「资源配置/预算分配」Tab；
> - Execute：OMP 与重点工作中复用 `budget` / `actualCost` 字段，经营分析会中增加预算执行预警；
> - Review：经营分析会/差距分析中增加预算偏差分析；
> - 驾驶舱：预算执行看板汇总关键指标。

## 四、建议实施顺序

### 第一阶段：占位页 + 导航（1-2 天）

1. 升级 `renderPlaceholder()` 为带进度骨架的占位页。
2. 为 `#bp/bem`、`#rev/*`、`#admin/rule-engine`、`#admin/alert-hub`、`#admin/requirement-pool` 等占位页定制描述和路线图。
3. 新增「重点工作管理」独立页面骨架。
4. 新增「用户中心」「通知中心」「规则引擎中心」「预警中心」骨架入口。
5. 统一更新 `src/lib/config.js`、`cockpit.html` 导航、`vite.config.js` 构建入口。

### 第二阶段：主流程骨架（2-3 天）

1. BEM 战略解码：六步页面骨架（导入地图 → CSF → CTQ → KPI → 重点工作 → 评审）。
2. 年度经营计划：补齐 KTI 挂载、任务关联、分组节点、**资源配置/预算分配** Tab。
3. AI 战略助手：增加「主动洞察」「报告生成」「会议助手」三个 Tab 骨架。
4. 经营分析会：增加「会前准备」「会中驾驶舱」「会后闭环」三个视图骨架，**含预算执行预警**。
5. 驾驶舱：增加「预算执行看板」汇总卡片。

### 第三阶段：数据打通标记（1 天）

1. 在战略专题、重点工作、业务专题、会议等模块中，用 TODO 标记数据打通点。
2. 统一 `localStorage` key 命名规范，为后续数据流整合做准备。
3. 明确预算字段（`budget` / `actualCost` / `budget_completion_rate`）在各模块的写入/读取点。

## 五、关键约束

- 继续沿用现有架构：`cockpit.html` 作为 SPA 主页面，独立页面在 `vite.config.js` 注册入口。
- `cockpit.html` 的 JS 在 IIFE 中，局部函数不能被 `onclick` 直接访问；占位页内交互优先使用事件委托或 `window` 暴露全局函数。
- CSS 继续使用变量（`var(--primary)` 等），禁止硬编码颜色。
- 每阶段完成后应跑 `npm run build` 和核心 Playwright 用例，确保无新增回归。
