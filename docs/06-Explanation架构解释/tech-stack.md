# 技术栈说明

## 前端

| 技术 | 版本 | 用途 |
|------|------|------|
| HTML5 | - | 页面结构 |
| CSS3 | - | 样式（CSS 变量实现主题系统） |
| Vanilla JS | ES6+ | 交互逻辑（无框架，保持简单） |
| Vite | 5.x | 构建工具（多页面、HMR、代码压缩） |

### 为什么不用 React/Vue？

1. **项目规模可控**：当前 4 个页面，不需要组件框架的复杂度
2. **AI 协作友好**：纯 HTML/JS 更容易让 AI 理解和生成
3. **零运行时成本**：没有框架 bundle，加载更快
4. **渐进式**：未来如果需要，随时可以迁移

## 测试

| 技术 | 版本 | 用途 |
|------|------|------|
| pytest | 7.x | 结构/内容回归测试 |
| Playwright | 1.40+ | E2E 端到端测试 |
| Vitest | 1.x | 单元测试（预留） |

### 为什么选 Playwright？

- **跨浏览器**：Chromium/Firefox/WebKit
- **自动等待**：不需要手动 sleep
- **Trace 调试**：失败时自动截图+录像
- **CI 友好**：GitHub Actions 原生支持
