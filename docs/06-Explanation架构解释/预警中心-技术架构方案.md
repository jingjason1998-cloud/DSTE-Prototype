## 一、技术架构总览

### 1.1 架构原则

1. **事件驱动**: 基于消息队列接收预警事件，解耦与规则引擎的依赖
2. **高可用**: 预警接收和通知服务必须高可用，确保预警不丢失
3. **可扩展**: 支持新增通知渠道、预警类型无需修改代码
4. **实时性**: 预警触发到通知送达延迟 < 30秒
5. **可追溯**: 完整的预警事件链路追踪

## 五、核心流程设计

### 5.1 预警事件处理流程

```java
@Service
public class AlertIngestionService {
    
    @Autowired
    private AlertEventRepository eventRepository;
    
    @Autowired
    private AlertRecordRepository recordRepository;
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    @Autowired
    private AlertEventValidator validator;
    
    public IngestionResult ingest(AlertEvent event) {
        // 1. 校验事件格式
        ValidationResult validation = validator.validate(event);
        if (!validation.isValid()) {
            return IngestionResult.fail(validation.getErrors());
        }
        
        // 2. 去重检查（基于规则+部门+时间窗口）
        String dedupKey = buildDedupKey(event);
        if (redisTemplate.hasKey(dedupKey)) {
            return IngestionResult.duplicate();
        }
        
        // 3. 存储事件
        eventRepository.save(event);
        
        // 4. 创建预警记录
        AlertRecord record = AlertRecord.builder()
            .eventId(event.getEventId())
            .status(AlertStatus.NEW)
            .currentLevel(event.getLevel())
            .originalLevel(event.getLevel())
            .deptId(event.getContext().getDeptId())
            .build();
        recordRepository.save(record);
        
        // 5. 设置去重缓存（5分钟窗口）
        redisTemplate.opsForValue().set(dedupKey, "1", 5, TimeUnit.MINUTES);
        
        // 6. 发送事件到分级服务
        rabbitTemplate.convertAndSend(
            "alert.event.received", 
            new ClassifyEvent(record.getAlertId(), event)
        );
        
        return IngestionResult.success(record.getAlertId());
    }
    
    private String buildDedupKey(AlertEvent event) {
        return String.format("alert:dedup:%s:%s:%s",
            event.getSourceRule(),
            event.getContext().getDeptId(),
            LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyyMMddHH"))
        );
    }
}
```

### 5.2 通知发送流程

```java
@Service
public class AlertNotifierService {
    
    @Autowired
    private NotificationStrategyRepository strategyRepository;
    
    @Autowired
    private NotificationTemplateRepository templateRepository;
    
    @Autowired
    private List<NotificationChannel> channels;
    
    @Autowired
    private NotificationRecordRepository notificationRepository;
    
    public void sendNotification(AlertLevelChangedEvent event) {
        AlertRecord alert = event.getAlert();
        
        // 1. 获取通知策略
        NotificationStrategy strategy = strategyRepository
            .findByLevelAndType(alert.getCurrentLevel(), alert.getAlertType());
        
        // 2. 获取通知模板
        NotificationTemplate template = templateRepository
            .findByCode(strategy.getTemplateCode());
        
        // 3. 渲染通知内容
        String content = renderTemplate(template, alert);
        
        // 4. 获取接收人列表
        List<Recipient> recipients = resolveRecipients(strategy, alert);
        
        // 5. 发送通知
        for (Recipient recipient : recipients) {
            for (String channelType : strategy.getChannels()) {
                NotificationChannel channel = getChannel(channelType);
                
                NotificationRecord record = NotificationRecord.builder()
                    .alertId(alert.getAlertId())
                    .channel(channelType)
                    .recipient(recipient.getId())
                    .templateCode(template.getCode())
                    .content(content)
                    .status(NotificationStatus.PENDING)
                    .build();
                
                notificationRepository.save(record);
                
                // 异步发送
                channel.sendAsync(record)
                    .thenAccept(result -> updateStatus(record, result))
                    .exceptionally(ex -> {
                        handleSendFailure(record, ex);
                        return null;
                    });
            }
        }
    }
    
    private NotificationChannel getChannel(String type) {
        return channels.stream()
            .filter(c -> c.getType().equals(type))
            .findFirst()
            .orElseThrow(() -> new IllegalArgumentException("Unknown channel: " + type));
    }
}
```

### 5.3 逾期升级定时任务

```java
@Component
public class EscalationJob {
    
    @Autowired
    private AlertRecordRepository alertRepository;
    
    @Autowired
    private EscalationRuleRepository ruleRepository;
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    @XxlJob("escalationCheckJob")
    public ReturnT<String> execute() {
        // 1. 查询超时未处理的预警
        List<AlertRecord> overdueAlerts = alertRepository
            .findOverdueAlerts(AlertStatus.NEW, Duration.ofHours(24));
        
        for (AlertRecord alert : overdueAlerts) {
            // 2. 获取升级规则
            EscalationRule rule = ruleRepository
                .findByCurrentLevel(alert.getCurrentLevel());
            
            if (rule != null && shouldEscalate(alert, rule)) {
                // 3. 执行升级
                AlertLevel newLevel = rule.getTargetLevel();
                alert.setCurrentLevel(newLevel);
                alertRepository.save(alert);
                
                // 4. 记录升级日志
                saveEscalationLog(alert, rule);
                
                // 5. 发送升级事件
                rabbitTemplate.convertAndSend(
                    "alert.escalation.trigger",
                    new EscalationEvent(alert.getAlertId(), newLevel)
                );
            }
        }
        
        return ReturnT.SUCCESS;
    }
    
    private boolean shouldEscalate(AlertRecord alert, EscalationRule rule) {
        Duration overdue = Duration.between(alert.getCreatedAt(), LocalDateTime.now());
        return overdue.compareTo(rule.getOverdueDuration()) >= 0;
    }
}
```

## 八、安全架构

### 8.1 认证鉴权

- **服务间调用**: Service Token，统一由API Gateway验证
- **用户访问**: JWT Token，统一由DSTE平台认证中心颁发
- **权限控制**: RBAC模型，按角色+部门维度控制预警查看/处理权限

### 8.2 数据安全

- 预警数据按组织隔离
- 敏感预警脱敏展示
- 通知内容不包含敏感数据
- 操作日志完整记录

---

## 九、性能优化策略

### 9.1 高并发处理

- 预警事件接收采用异步处理，立即返回确认
- 通知发送采用批量+异步，降低延迟
- 使用Redis缓存去重，避免重复处理

### 9.2 实时推送

- WebSocket连接推送实时预警到前端
- 企业微信采用异步推送，失败重试
- 预警大盘采用增量更新，降低带宽
