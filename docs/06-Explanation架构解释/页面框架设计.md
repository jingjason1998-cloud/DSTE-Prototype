├──────────┬──────────────────────────────────────────────────────────────────┤

### 1.1 顶部导航栏 (64px)

| 元素 | 说明 | 交互 |
|------|------|------|
| **Logo区** | 产品Logo + 名称"DSTE战略管理平台" | 点击返回首页/驾驶舱 |
| **全局搜索** | 搜索框，支持Ctrl+K快捷键 | 搜索KPI、工作、会议、文档 |
| **通知中心** | 消息、预警、待办提醒 | 下拉展示通知列表 |
| **用户菜单** | 头像 + 下拉菜单 | 个人设置、退出登录 |
| **主题切换** | 🌙/☀️ 图标 | 切换light/dark模式 |

### 1.2 左侧导航边栏 (240px / 折叠后 64px)

**分组结构**（基于DSTE四阶段）：

```
📊 战略驾驶舱 (首页)

━━ 战略制定 (SP) ━━
  🗺️ 战略地图
  📋 战略专题

━━ 战略解码 (BP) ━━
  📈 KPI指标体系
  🔧 BEM战略解码

━━ 战略执行 (Execute) ━━
  ✅ 重点工作管理
  📂 业务专题管理
  🎯 经营分析会 ⭐

━━ 战略评估 (Review) ━━
  🏆 绩效与激励
  👔 干部管理

━━ 智能中心 ━━
  🤖 AI战略助手

━━ 系统管理 ━━
  ⚙️ 系统设置
  👥 用户权限
```

**交互细节**:
- 当前页面高亮显示
- 支持拖拽排序（个性化）
- 折叠后仅显示图标
- 悬停显示Tooltip

### 1.3 面包屑 + 页面标题区 (48px)

```
首页 / 战略执行 / 重点工作管理 / 详情
                    [新建工作] [批量导入] [导出]
```

## 四、响应式设计

### 4.1 断点定义

| 断点 | 宽度 | 布局调整 |
|------|------|---------|
| **桌面大屏** | ≥1440px | 左侧边栏展开，多列布局 |
| **桌面** | ≥1280px | 左侧边栏展开，标准布局 |
| **平板** | ≥768px | 左侧边栏折叠，单列布局 |
| **手机** | <768px | 底部导航，单列布局 |

### 4.2 移动端适配

- 左侧导航 → 底部Tab栏
- 表格 → 卡片列表
- 多列布局 → 单列堆叠
- 抽屉面板 → 全屏弹窗
- 复杂操作 → 简化流程

## 五、交互状态定义

### 5.1 页面状态

| 状态 | 说明 | 展示 |
|------|------|------|
| **加载中** | 数据请求中 | 骨架屏 + 加载动画 |
| **空状态** | 无数据 | 插图 + 提示文字 + 操作按钮 |
| **错误状态** | 请求失败 | 错误图标 + 重试按钮 |
| **无权限** | 权限不足 | 锁图标 + 提示文字 |

### 5.2 操作反馈

| 操作 | 反馈方式 |
|------|---------|
| 保存成功 | Toast: "保存成功" |
| 删除确认 | 模态框: "确认删除？" |
| 提交成功 | Toast + 页面跳转 |
| 操作失败 | Toast: 错误信息 |
| 加载中 | 按钮Loading状态 |

## 七、设计原则总结

1. **信息层级清晰**: 通过字号、颜色、间距建立明确的信息层级
2. **数据驱动**: 所有决策都有数据支撑，减少主观描述
3. **效率优先**: 常用操作不超过3步，支持批量操作
4. **一致性**: 相同功能在不同模块保持一致的交互方式
5. **可扩展**: 组件化设计，支持新模块快速接入
6. **沉浸式**: 分析场景支持全屏模式，减少干扰
