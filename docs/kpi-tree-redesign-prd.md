# KPI 指标树 — 卡片式层级视图 PRD（v3 最终版）

> 目标：耳目一新 · 既震撼又适用

---

## 1. 设计哲学

**不是「表格的替代品」，而是「指标体系的数字孪生」。**

用户看到的不应该是一张张卡片，而是一张**活的 KPI 战略地图**——
- 每个节点像神经元一样脉动（状态驱动）
- 连接线像血管一样输送数据流向
- 展开时像思维导图一样自然生长
- 达成时像里程碑一样被点亮

---

## 2. 核心创新点（5个「震撼」元素）

### 2.1 环形进度锚点 — 一眼定生死

每个卡片**左侧**放置一个 **40px 的 SVG 环形进度条**：
- 达成率直接画成圆环，填满角度 = 达成率 × 3.6°
- 颜色随状态变化：绿/橙/红
- 达成 100% 时圆环闭合，带轻微 **发光效果** (`box-shadow: 0 0 8px rgba(82,196,26,0.4)`)

> 💡 为什么震撼：从「读数字」变成「看图形」，大脑处理速度快 6 倍

### 2.2 状态驱动的卡片质感 — 三种「情绪」

卡片不只是颜色标签不同，**整张卡片的气质都不同**：

| 状态 | 卡片质感 | 动效 |
|------|----------|------|
| **达成** | 轻微绿色光晕从底部向上晕染，像被点亮 | 首次出现时有 ✨ 微闪烁（1次） |
| **预警** | 卡片边框有缓慢的橙色呼吸脉冲 | 边框 `opacity` 0.6→1→0.6，3s 循环 |
| **落后** | 红色虚线边框快速闪烁，紧迫感 | 边框颜色 #F5222D ↔ #FF7875，0.8s 循环 |

> ⚠️ 呼吸/脉冲动效只在 Level 0（公司级）卡片上播放，避免满屏乱闪

### 2.3 贝塞尔曲线 + 流动粒子 — 连接线活了

抛弃直线条，改用 **圆角贝塞尔曲线** 连接父子节点：
- 从父节点底部中心出发，先垂直向下，再水平弯折到子节点左侧
- 线上有 **1-2 个微小圆点** 沿曲线缓慢流动（CSS 动画），暗示数据从上级流向下级
- 流动速度 ∝ 达成率（达成快的流得快）

```
    ┌──────┐
    │ 父   │
    └──┬───┘
       │      ← 垂直段
       └─╮    ← 贝塞尔弯折
         │
       ┌─┴──┐
       │ 子 │
       └────┘
```

### 2.4 展开动画 — 像思维导图一样「生长」

点击 ▶ 展开时：
1. 父节点卡片 **轻微缩小到 96%**（聚焦子节点）
2. 子节点卡片从父节点下方 **像扑克牌展开一样扇形出现**
3. 连接线从父节点 **像墨水一样向下晕染绘制**
4. 全部完成时间：**300ms**，ease-out

折叠时反向动画，父节点恢复 100%。

### 2.5 共享指标的「汇聚星光」

当一个子指标支撑多个父指标时：
- 子指标卡片边框变为 **双色渐变**（从两个父节点的颜色渐变而来）
- 多条连接线汇聚到子指标时，交汇点有一个 **旋转的星形标记** ⭐
- hover 星形标记，tooltip 显示：「该指标支撑 N 个上级目标」

## 4. 三级架构 — 阶梯式视觉系统

| 层级 | 卡片宽度 | 左边框 | 标题 | 环形大小 | 连接线 |
|------|----------|--------|------|----------|--------|
| **公司级 L0** | 100% | 4px 主色渐变条 | 15px/700 | 40px | 蓝色 2px 实线 + 粒子 |
| **部门级 L1** | calc(100% - 32px) | 3px 灰色条 | 14px/600 | 32px | 灰色 1.5px 实线 |
| **团队级 L2** | calc(100% - 64px) | 无 | 13px/500 | 24px | 浅灰 1px 虚线 |

> 缩进：L1 左缩进 32px，L2 左缩进 64px，形成明显阶梯

## 5. 默认展开策略

```
首次加载：
  L0（公司级）── 全部展开（展示骨架）
    ├── L1（部门级）── 全部折叠（保持整洁）
    │     └── L2（团队级）── 随 L1 折叠而隐藏
    └── L1（部门级）── 全部折叠
  
用户交互后：
  展开/折叠状态持久化到 kpiExpandedIds
```

> 为什么这样：首屏看到「公司有哪些大目标」，想看细节再逐层展开，避免信息过载

## 6. 颜色规范

### 达成率环形
- 0-79%：`#F5222D`（红）
- 80-94%：`#FAAD14`（橙）
- 95-100%：`#52C41A`（绿）
- 100%+：`#1677FF`（蓝，超额完成）

### 卡片状态光晕
- 达成：`box-shadow: 0 2px 8px rgba(82,196,26,0.15)`
- 预警：`box-shadow: 0 2px 8px rgba(250,173,20,0.15)`
- 落后：`box-shadow: 0 2px 8px rgba(245,34,45,0.15)`

### 连接线
- L0→L1：`stroke: #1677FF; stroke-width: 2;` + 白色流动粒子
- L1→L2：`stroke: #BFBFBF; stroke-width: 1.5;`
- 共享汇聚：`stroke: #722ED1; stroke-dasharray: 4 2;` + 紫色粒子

## 7. 动效清单

| 动效 | 触发 | 时长 | 曲线 |
|------|------|------|------|
| 卡片 hover 上浮 | mouseenter | 150ms | ease-out |
| 子树展开 | 点击 ▶ | 300ms | cubic-bezier(0.34, 1.56, 0.64, 1) |
| 子树折叠 | 点击 ▼ | 200ms | ease-in |
| 连接线绘制 | 展开时 | 250ms | ease-out |
| 达成闪烁 | 首次渲染 | 600ms | 单次 |
| 预警呼吸 | 持续 | 3s | ease-in-out 循环 |
| 落后脉冲 | 持续 | 0.8s | ease-in-out 循环 |
| 粒子流动 | 持续 | 2-4s | linear 循环 |

> ⚠️ 性能保护：使用 `transform` 和 `opacity` 实现动效，避免触发重排

## 8. 共享指标（多父节点）

```
🏢 合同额
│
├─ 📂 销售部
│   └─ 👤 华东区
│
└─ 📂 市场部
    └─ 👤 华东区  ← 同一个「华东区」支撑两个部门
         ↑
      星形汇聚标记 ⭐
```

- 卡片边框：双色渐变（左半来自父A颜色，右半来自父B颜色）
- 连接线：从两个父节点分别引出，在子节点左侧汇聚成一个圆点
- 卡片标签：`🔗 共享` 紫色药丸标签
- 点击汇聚圆点：tooltip 列出所有上级指标名称

沿用现有 `parentId` + `level` 模型，预留 `parentIds` 扩展字段：

```js

```

## 11. 技术实现要点

1. **环形进度**：纯 CSS `conic-gradient` 或内联 SVG `<circle>` 的 `stroke-dasharray`
2. **贝塞尔曲线**：SVG `<path>` 动态计算 d 属性，或使用 CSS border 技巧
3. **流动粒子**：CSS `@keyframes` 沿 SVG path 运动（`offset-path`）
4. **展开动画**：CSS `grid-template-rows` 或 `max-height` 过渡
5. **状态动效**：CSS `animation`（呼吸/脉冲），Level 0 才挂载
6. **性能**：所有动画仅使用 `transform` / `opacity`，`will-change` 预声明
